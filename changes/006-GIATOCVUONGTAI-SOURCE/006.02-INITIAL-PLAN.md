# 006.02 ‚Äî Initial Plan: giatocvuongtai.com scraper source

## 0. Summary & verified ground truth

`giatocvuongtai.com` is an Astro SPA backed by a **clean public JSON API** ‚Äî no login, no cookies, no obfuscation, no observed rate limit. The plain `HttpClient.get_html` (a throttled/retrying `requests.Session` GET) reaches every endpoint and returns 200. This makes the adapter dramatically simpler than medoctruyen: no `BeautifulSoup`, no cookie plumbing, no `AuthRequiredError`/`RateLimitedError`/`DailyLimitError` wiring, no GUI/config/settings changes, no `TranslateWorker` changes. It is a **single new file + a one-line registration edit + tests**.

Confirmed against the repo:
- `SiteAdapter` ABC (abstract `fetch_metadata`, `fetch_chapter_list`, `fetch_chapter`) and the shared `HttpClient` live in `src/noveltrans/scrapers/base.py`. `HttpClient.get_html(url, **kwargs)` returns decoded text and already does throttle + retry/backoff + encoding fix. There is **no** `get_json` for GET (only `get_json_post` for POST form data).
- Registry in `src/noveltrans/scrapers/__init__.py`: classes self-register via `@register`; `adapter_for_url(url, client)` iterates `ADAPTERS` and returns the first whose `matches()` (inherited, uses `re.search` over `url_patterns`) hits; `_import_adapters()` currently does `from noveltrans.scrapers import ixdzs, medoctruyen, xbanxia`.
- `NovelMeta(url, site, title, author="", description="", cover_url="", source_lang="zh", ‚Ä¶)` and `ChapterRef(index: int, title: str, url: str)` in `src/noveltrans/models.py`. `index` is 0-based TOC order.
- Errors in `src/noveltrans/errors.py`: `ScrapeError(message, url="")`, `ObfuscatedContentError(ScrapeError)`. No new error subclass is required for this site.
- The **medoctruyen adapter already sets `source_lang="vi"`** for Vietnamese-edit content ‚Äî we mirror that exactly. Downstream, the pipeline treats a `vi` source like any other source language (the identity/passthrough translate path from feature 003 already handles vi‚Üívi export/TTS); nothing new is needed here.
- Tests use `responses` + `load_fixture(site, name)` (reads `tests/fixtures/<site>/<name>`) with `HttpClient(delay_seconds=0)`; `@pytest.mark.live` marks network tests. See `tests/test_scrapers_medoctruyen.py`.

### Live-API findings (provided reconnaissance, treated as ground truth)

Example story: `https://giatocvuongtai.com/stories/dm-edit-chao-mung-den-voi-phong-livestream-ac-mong/`

1. **Story + full TOC in one request:** `GET https://giatocvuongtai.com/api/public/story/<slug>.json`, where `<slug>` is the last path segment of the story URL. Response `{"data": {...}, "error": ...}`. `data` carries `id` (story UUID), `slug`, `title`, `summary` (multiline description blob embedding "T√°c gi·∫£: ‚Ä¶", "Th·ªÉ lo·∫°i: ‚Ä¶", "ƒê·ªô d√Ýi: ‚Ä¶"), `cover_url`, `chapter_count`, `completion_status`, `author.name`, and `chapters`: a list of `{"id": <uuid>, "title", "order_number": <int, 1-based>, "is_published": <bool>, ‚Ä¶}` **already in reading order**. Also `requiresPassword`/`is_password_protected` (both `false` here; present for password-gated stories). This story returns all 745 chapters in one response ‚Äî **no pagination**.
2. **Chapter body, one request each:** `GET https://giatocvuongtai.com/api/public/chapter/<chapterId>.json` ‚Üí `{"data": {"id", "title", "order_number", "content", ‚Ä¶}}`. `content` is a nested block document `{"blocks": [{"type": "paragraph", "inline": [{"text": "‚Ä¶", "marks": []}, ‚Ä¶]}, ‚Ä¶]}`. `content` MAY arrive as a real dict OR as a JSON-encoded string depending on serialization.
3. Human reader URL `https://giatocvuongtai.com/reader/?storyId=<storyId>&chapter=<chapterId>` is informational only ‚Äî no browser needed.

## DECISION #1 ‚Äî How to fetch JSON (no base.py change)

**Recommendation: parse in the adapter with `json.loads(self.client.get_html(url))`; do NOT add a `get_json` to `HttpClient`.**

Rationale (lighter option, justified):
- `get_html` already returns correctly-decoded text (the API serves UTF-8; `get_html`'s encoding fallback sniffs UTF-8 when charset is absent, which is correct for Vietnamese). Wrapping in `json.loads` is one line.
- Adding a `get_json` GET helper touches shared `base.py` (blast radius across all adapters, needs its own test) for zero reuse benefit ‚Äî only this one adapter needs GET-JSON today. `medoctruyen.py` already does `json.loads(...)` locally for JSON-LD, so an in-adapter parse is an established pattern.
- Provide a tiny private helper on the adapter, `_get_json(url) -> dict`, that does the GET + `json.loads` + raises `ScrapeError("Expected JSON response", url)` on `ValueError`, plus unwraps and validates the `{"data", "error"}` envelope in one place.

If a second JSON-GET site appears later, promote `_get_json` to `HttpClient` then ‚Äî not now.

## DECISION #2 ‚Äî What to store in `ChapterRef.url`

**Recommendation: store the chapter-content API URL directly**, i.e. `https://giatocvuongtai.com/api/public/chapter/<chapterId>.json`.

Rationale: `fetch_chapter(ref)` then simply GETs `ref.url` and parses ‚Äî no id juggling, no storyId needed at download time, and `ChapterRef.url` stays a single self-contained actionable string (consistent with how medoctruyen/xbanxia store a directly-fetchable URL). The human reader URL and the raw `chapterId` are not stored because `fetch_chapter` never needs them. Trade-off: the stored URL is an API endpoint rather than a human-visitable page, so it is not meaningful to open in a browser ‚Äî acceptable, since `ChapterRef.url` exists to drive `fetch_chapter`, not for display. (If a human-clickable link were ever desired, it can be reconstructed from the story + chapter ids; not worth storing now.)

## 1. New file: `src/noveltrans/scrapers/giatocvuongtai.py`

New `@register`ed `GiatocvuongtaiAdapter(SiteAdapter)`. No `BeautifulSoup` import ‚Äî pure JSON.

Module constants:
- `_ORIGIN = "https://giatocvuongtai.com"`
- `_STORY_API = _ORIGIN + "/api/public/story/{slug}.json"`
- `_CHAPTER_API = _ORIGIN + "/api/public/chapter/{cid}.json"`
- `_SLUG_RE = re.compile(r"giatocvuongtai\.com/stories/([a-z0-9-]+)")`

Class attributes:
- `name = "giatocvuongtai"`
- `display_name = "Gia T·ªôc V∆∞∆°ng T√Ýi (giatocvuongtai.com)"`
- `url_patterns = [r"giatocvuongtai\.com/stories/[a-z0-9-]+"]` ‚Äî matches the story landing URL (with or without trailing slash). Reader URLs (`/reader/?storyId=‚Ä¶`) are intentionally **not** matched: the adapter is entered via the story slug, and a reader URL carries no slug. Document that users should paste the `/stories/<slug>` URL. (`matches()` is inherited and uses `re.search`, so a trailing slash or query string does not break the match.)

### Helpers

`@staticmethod _slug(url) -> str`
- `m = _SLUG_RE.search(url)`; if none ‚Üí `raise ScrapeError("Could not extract story slug from URL", url)`; return `m.group(1)`.
- This naturally handles the trailing slash: the regex captures the path segment after `/stories/` up to (but not including) the next `/`, so `.../stories/<slug>/` and `.../stories/<slug>` both yield `<slug>`. As a belt-and-suspenders fallback if the regex is ever relaxed, `strip("/").split("/")[-1]` on the path also works ‚Äî but the anchored regex is preferred and is the single source of truth (mirrors medoctruyen's `_slug`).

`def _get_json(self, url) -> dict`
- `raw = self.client.get_html(url)`; `try: payload = json.loads(raw) except ValueError as exc: raise ScrapeError("Expected JSON response", url) from exc`.
- If `payload.get("error")` is truthy ‚Üí `raise ScrapeError(f"API returned error: {payload['error']}", url)`.
- `data = payload.get("data")`; if `data is None` ‚Üí `raise ScrapeError("API response missing 'data'", url)`; return `data`.

### `fetch_metadata(self, url) -> NovelMeta`
- `slug = self._slug(url)`; `data = self._get_json(_STORY_API.format(slug=slug))`.
- `title = (data.get("title") or "").strip()`; if empty ‚Üí `raise ScrapeError("Story title not found", url)`.
- `author = (data.get("author") or {}).get("name", "").strip()` (guard `author` may be `None` or missing ‚Üí `""`).
- `description = (data.get("summary") or "").strip()` (the multiline blob; store verbatim ‚Äî downstream translation/export handle multiline).
- `cover_url = data.get("cover_url") or ""`.
- Return `NovelMeta(url=f"{_ORIGIN}/stories/{slug}/", site=self.name, title=title, author=author, description=description, cover_url=cover_url, source_lang="vi")`.
  - Normalize `url` to a canonical `/stories/<slug>/` form so re-scans/dedup are stable regardless of the pasted trailing slash.
  - `source_lang="vi"` and `site=self.name` are load-bearing (mirror medoctruyen).
- Optional (note only): `data["id"]` (story UUID) is available here but not needed given Decision #2; do not store it.

### `fetch_chapter_list(self, url) -> list[ChapterRef]`
- `slug = self._slug(url)`; `data = self._get_json(_STORY_API.format(slug=slug))`.
- `chapters = data.get("chapters") or []`; if empty ‚Üí `raise ScrapeError("Chapter list not found", url)`.
- Sort defensively by `order_number` (`sorted(chapters, key=lambda c: c.get("order_number", 0))`) even though the API returns them ordered ‚Äî cheap insurance against ordering drift.
- Filter: optionally skip `is_published is False` entries. **Decision:** include only published chapters (`if c.get("is_published", True)`), because unpublished chapter bodies may 404 or be empty; note this in a comment. Keep it simple ‚Äî do not fail if the field is missing (default `True`).
- Build refs with **sequential 0-based index** (decoupled from `order_number`, matching how medoctruyen assigns `index`):
  ```
  return [
      ChapterRef(index=i, title=(c.get("title") or f"Ch∆∞∆°ng {c.get('order_number','')}").strip(),
                 url=_CHAPTER_API.format(cid=c["id"]))
      for i, c in enumerate(published_sorted)
  ]
  ```
- Guard: a chapter dict missing `id` ‚Üí skip it (can't build a fetch URL); if that empties the list, raise `ScrapeError`.
- **No pagination** ‚Äî the single story endpoint returns the full list (745 for the sample). Note this explicitly in a comment so a future maintainer doesn't add a pager.

### `fetch_chapter(self, ref) -> str`
- `data = self._get_json(ref.url)` ‚Äî `ref.url` is already the chapter-content API URL (Decision #2).
- `content = data.get("content")`; if `content is None` ‚Üí `raise ScrapeError("Chapter content missing", ref.url)`.
- **String-vs-dict handling:** `if isinstance(content, str): content = json.loads(content)` (wrap in try/except ‚Üí `ScrapeError` on bad JSON).
- Extract via the block algorithm below ‚Üí `text`.
- If `not text.strip()` ‚Üí `raise ScrapeError("Chapter content is empty", ref.url)`.
- Return `text` (paragraphs joined by `\n\n`, per the `fetch_chapter` contract).

## 2. Block ‚Üí text extraction algorithm (defensive)

Given the parsed `content` dict:

```
blocks = content.get("blocks") or []
paragraphs: list[str] = []
for block in blocks:
    btype = block.get("type")
    if btype == "paragraph" or btype == "heading":
        text = "".join(seg.get("text", "") for seg in (block.get("inline") or []))
        text = text.strip()
        if text:
            paragraphs.append(text)
    # else: skip non-text blocks (divider/image/‚Ä¶) ‚Äî do not crash on unknown types
return "\n\n".join(paragraphs)
```

Design notes:
- Every block in the sample is `type: "paragraph"`, but the loop is written to **skip unknown types** (`divider`, `image`, `code`, ‚Ä¶) rather than assume ‚Äî this satisfies the defensiveness requirement. `heading` is folded in as text (a reasonable inclusion for other stories); anything else is silently skipped. Keep the explicit allow-list (`paragraph`, `heading`) so unexpected block types never leak raw JSON into the output.
- Each block's text is the concatenation of its `inline[].text` values (marks like bold/italic are ignored ‚Äî we produce plain text, matching the contract). `inline` may be missing/`None` ‚Üí default to `[]`.
- Empty-after-strip paragraphs are dropped (prevents doubled `\n\n`).
- No `ObfuscatedContentError` is warranted here (content is clean JSON, not obfuscated). `ScrapeError` for structural failures (missing `content`, empty result) is sufficient.

## 5. Open questions / risks (with mitigations)

- **Password-protected stories:** the API exposes `requiresPassword` / `is_password_protected`. The sample story is open, so this plan does **not** implement password entry. Mitigation: in `fetch_metadata`/`fetch_chapter_list`, if `data.get("is_password_protected")` is true (and no chapter bodies are returnable), raise a clear `ScrapeError("This story is password-protected; unsupported")` rather than silently producing an empty TOC. Full password support is deferred as a follow-up (would need a config field + an auth param on the request, analogous to medoctruyen cookies) ‚Äî noted, not built.
- **Non-paragraph block types in other stories:** handled by the allow-list extraction (skip unknowns). If a future story is mostly `image`/`divider`, `fetch_chapter` may return empty and raise `ScrapeError("Chapter content is empty")` ‚Äî acceptable and visible, not a silent bad save.
- **`content` serialization variance (dict vs string):** explicitly handled by the `isinstance(content, str)` branch, covered by a dedicated fixture.
- **No pagination today:** the story endpoint returns all chapters in one shot. Risk: a much larger story could make that one response big, but it is still a single GET and well within `requests` limits; documented so no one adds a needless pager.
- **Rate limits:** none observed. All traffic still flows through `HttpClient`'s throttle (`request_delay`) + retry/backoff, so 745 sequential chapter GETs are polite by construction. No `RateLimitedError`/`DailyLimitError` handling added (justified: the site imposes none).
- **API drift:** the adapter depends on the `{"data": {...}}` envelope and the `blocks/inline/text` shape. The `_get_json` envelope check and the `@pytest.mark.live` test catch drift early. Fixtures pin the current shape for offline regression.
- **UUID `id` vs numeric order:** `index` is derived from enumeration (0-based) and is independent of `order_number` and of the chapter UUID ‚Äî robust to any id scheme.
