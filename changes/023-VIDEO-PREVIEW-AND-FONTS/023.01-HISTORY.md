## [STEP 1] Implemented — fonts + preview + GUI — 2026-07-19
Plan 023.01, all parts.

### Fonts (bundled + verified before trusting)
Downloaded 5 OFL fonts from the Google Fonts repo. **Verification caught a real trap:**
Nunito and Montserrat are *variable* fonts whose DEFAULT instance is ExtraLight (wght 200)
and Thin (wght 100) — bundling them raw would render titles too thin. Fixed by pinning both
to Regular via `fontTools.varLib.instancer` (`wght=400`, `updateFontNames=True`,
`usWeightClass=400`) → clean static Regular. Lora/Playfair defaulted to 400 already; Be
Vietnam Pro is static. Verified every file: `ImageFont.getname()[0]` == the registry family
AND all of `ệữỗậọằẽỹđĐơư` render (no tofu). Rendered a sample sheet to eyeball weights.
Bundled `assets/{BeVietnamPro,Nunito,Montserrat,Lora,PlayfairDisplay}-Regular.ttf` + each
font's `*-OFL.txt` license (the `*.txt` package-data glob distributes them).

### Code
- **`video.py`** — `VIDEO_FONTS` (6) + `DEFAULT_VIDEO_FONT` + `video_font()`. `render_video`
  gained `font_name` and now actually *passes* it to `build_ass_subtitles` (the param was
  there but had been dropped). New `render_preview_frame(...)`: reuses the exact `_filtergraph`
  (WYSIWYG), bakes skin/vinyl/knob + a 2-event ASS, feeds 2s pink-noise `anoisesrc` so bars
  look alive, grabs ONE frame at t=1s (`-ss 1 -frames:v 1 -update 1`) with a synthetic 3s
  total so the playhead sits ~1/3 along. No chapter audio needed.
- **`config.py`** — validated `video_font` property (lazy import to avoid a config→tts cycle).
- **`workers.py`** — `VideoWorker` gained `font` (→ `font_name`); new `VideoPreviewWorker`.
- **`tab_audio.py`** — "Phông chữ" dropdown (persists) + "Xem trước" button (both gated on
  ffmpeg); `_start_preview` needs only a valid image (no project/audio), result shown in an
  in-app `QDialog` scaled to fit; the font is threaded into the real `_start_video` render too;
  `_preview_worker` joined in `shutdown()`.
