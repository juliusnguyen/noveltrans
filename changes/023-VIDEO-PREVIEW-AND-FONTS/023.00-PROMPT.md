# 023 — Video preview frame + selectable Vietnamese fonts

Two related improvements to the video export ("music player" video, feature 020–022).

## 1. Preview the first frame before rendering
Rendering a full video is slow (hours for a long novel). Before committing to that, let the
user **preview a single still frame** of how the video will look — the player skin with the
chosen background image, novel title, chapter title, vinyl label, bars, and progress track —
so they can check the layout/photo/title/font *before* starting the long render.

- A "Xem trước" (Preview) button in the video box that renders **one frame** (fast, no
  encode) and shows it (open the PNG, or a small preview dialog).
- Uses the currently-selected background image, quality preset (resolution), and font.
- Should work even with no chapter audio yet (use the novel title + a sample chapter title),
  so the user can preview purely for design.

## 2. Choose a nicer Vietnamese-capable font
Right now titles use the single bundled Noto Sans. Let the user pick from **several
beautiful fonts that fully support Vietnamese** (all tone marks / diacritics), so the video
matches their channel's style.

- Bundle a small set of OFL/free fonts that cover Vietnamese well (e.g. a clean sans, a
  friendly rounded, an elegant serif, a display face) — each verified to render Vietnamese
  diacritics without tofu.
- A font dropdown in the video box; persist the choice; thread it through to the ASS
  subtitle styles (`build_ass_subtitles` `font_name` + `fontsdir`).
- The preview (part 1) reflects the selected font so the two features work together.

## Out of scope
- CJK glyphs (the titles are Vietnamese; a font need only cover Latin + Vietnamese).
- Per-chapter font changes; one font per video is enough.
