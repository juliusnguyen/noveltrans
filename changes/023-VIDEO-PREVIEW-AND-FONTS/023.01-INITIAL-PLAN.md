# Implementation Plan — Feature 023: Video preview frame + selectable Vietnamese fonts

## Context grounded from the code
- `render_video` (video.py) calls `build_ass_subtitles(...)` **without** `font_name`, so the
  existing `font_name=FONT_NAME` param is dead — the chosen font must be threaded
  `tab → VideoWorker → render_video → build_ass_subtitles`.
- `_filtergraph` draws bars from `[1:a]showfreqs` (audio is input 1) → a preview frame needs
  *some* audio input; silence gives empty bars, so feed a synthetic `lavfi` source.
- Knob x is `track_x + (t/total)*track_w - knob_half`; at `t=0` it sits at the left end, so
  the preview grabs a frame at `t>0` with a small synthetic `total`.
- `font_dir_context()` yields the real `assets/` dir; libass resolves styles by **family
  name** among all TTFs in `fontsdir`. Bundling more TTFs there is automatic for resolution.
- Packaging already covered: `pyproject.toml` globs `tts/assets/*.ttf` + `*.txt`, and the
  spec uses `collect_data_files("noveltrans")` → new TTFs/licenses bundle with no build-file
  edits, only an `.app` rebuild.

## Part A — Font registry + threading the font through (do first)
### A1 — Bundle fonts into `assets/` (manual download + verify)
OFL-1.1 TTFs, full precomposed Vietnamese coverage, sourced from the Google Fonts repo:

| key | family | file | source |
|---|---|---|---|
| `noto_sans` | Noto Sans | `NotoSans-Regular.ttf` (existing) | bundled |
| `be_vietnam` | Be Vietnam Pro | `BeVietnamPro-Regular.ttf` | ofl/bevietnampro |
| `nunito` | Nunito | `Nunito-Regular.ttf` | ofl/nunito |
| `montserrat` | Montserrat | `Montserrat-Regular.ttf` | ofl/montserrat |
| `lora` | Lora | `Lora-Regular.ttf` | ofl/lora |
| `playfair` | Playfair Display | `PlayfairDisplay-Regular.ttf` | ofl/playfairdisplay |

Spans clean sans / Vietnamese-designed sans / rounded / geometric / elegant serif / display
serif. **Verify each at download time (and as test D4):** `ImageFont.truetype(path).getname()[0]`
== registry `family` (libass falls back silently otherwise), and each Vietnamese codepoint in
`"ệữỗậọằẽỹđĐơư"` renders a non-empty glyph (`getmask(ch).getbbox() is not None`). Prefer a
static Regular instance if a Google variable file misbehaves. Drop each font's license as
`<Font>-OFL.txt` (the `*.txt` glob bundles them).

## Part B — Preview frame renderer (`video.py`)
### B1 — `render_preview_frame(image_path, out_png, font_dir, novel_title, sample_chapter_title, *, width=1920, height=1080, spin_vinyl=True, font_name=FONT_NAME, cancelled=None) -> Path`
One fast ffmpeg call, no encode; temp dir + `finally`-cleanup like `render_video`. Bakes
skin/vinyl/knob (`PlayerLayout.of`), writes a 2-event ASS (novel + one sample chapter,
`total = _PREVIEW_TOTAL = 3.0`), then:
```
ffmpeg -y -loop 1 -framerate 25 -i skin.png \
  -f lavfi -t 2 -i anoisesrc=color=pink:amplitude=0.5:sample_rate=48000 \
  -loop 1 -i vinyl.png -loop 1 -i knob.png \
  -filter_complex <_filtergraph(width,height,subs,font_dir,3.0,spin_vinyl)> \
  -map [v] -ss 1 -frames:v 1 -update 1 out.png
```
Input order (skin0/audio1/vinyl2/knob3) matches `_filtergraph`. Pink noise → lively bars
(silence looks empty); `-ss 1 -frames:v 1` grabs the frame at 1 s (rotate/bars/knob correct,
knob ~1/3 along), finishes in ~1–2 s. Reuses `_filtergraph` verbatim → WYSIWYG with the real
render. Short deadline (~60 s) via `_run_ffmpeg`.

## Part C — GUI wiring (`tab_audio.py` + `workers.py`)
### C1 — `VideoPreviewWorker(QThread)` in `workers.py`
Signals `done(str png_path)` / `failed(str)`; `run()` calls `render_preview_frame` inside
`font_dir_context()`, writes to a single reused temp path, emits result. Keeps the UI
responsive (a 1080p bake+ffmpeg is a couple seconds).

### C2 — Font dropdown + "Xem trước" button in `_build_video_box()`
`self.video_font` combo populated from `VIDEO_FONTS` (current from `config.video_font`),
`currentIndexChanged → _on_video_font_changed`. A `QPushButton("Xem trước")`
`self.video_preview_button → _start_preview`, gated on `ffmpeg_available()`. Add
`QLabel("Phông chữ:")` + combo + preview button to the row (near quality, before "Tạo video").

### C3 — `_start_preview`
Requires a valid image (warn otherwise); does **not** require a project/chapter audio (uses
`novel_title` from the project meta if present else "Tên truyện", sample title "Chương 1:
Chương mẫu"). Resolves the quality preset (resolution/spin) + selected font family, starts
`VideoPreviewWorker`, disables the button + status "🖼️ Đang tạo ảnh xem trước…".

### C4 — Show result in an in-app `QDialog` (recommended; matches `_preview_text`)
`_on_preview_ready(png)`: load `QPixmap`, scale to fit (~900×520, KeepAspectRatio, Smooth),
show in a dialog with a Close button. `_on_preview_failed(msg)`: re-enable + `QMessageBox`.
Imports: `QPixmap` (QtGui), `Qt` (QtCore), `VideoPreviewWorker`.

### C5 — Thread the font into the real render (`_start_video`)
Resolve `family = video_font(self.video_font.currentData())["family"]` and pass `font=family`
to `VideoWorker(...)` so the chosen font applies to the final video too.

### C6 — Lifecycle
`self._preview_worker` in `__init__`; `shutdown()` waits on it if running (`wait(60_000)`).

## Risks
- **Family-name contract**: registry `family` must equal the TTF name-table family or libass
  falls back / tofus. D4 guards it — don't skip.
- **Variable fonts**: Google ships `Lora[wght].ttf` etc.; verify `getname()` + rendering per
  file, else fetch a static Regular. Slightly larger app.
- **Licensing**: OFL only; bundle each `OFL.txt` (renamed) — the `*.txt` glob distributes it.
- **Preview bars on silence**: feed pink-noise lavfi so bars look alive; real render unaffected.
- **Knob at t=0**: grab at `t=1s` with synthetic `total=3s` so the playhead sits ~1/3 along.
- **App rebuild**: new TTFs only appear in the frozen `.app` after a PyInstaller rebuild.
