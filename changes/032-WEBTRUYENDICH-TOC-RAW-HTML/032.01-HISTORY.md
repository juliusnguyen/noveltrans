- The "(HTML Sitemap)" button links to `/danh-sach-chuong-day-du` — the **same**

  (server-rendered). The infinite-scroll is JS *virtualizing* the live DOM down to
  ~135 rows; the raw document has them all. So we can read the raw response and
  skip scrolling entirely (~1.4s vs ~10s).

## [IMPL] — 2026-07-24

### The twist — malformed raw HTML broke the HTML parsers
First live run: 1725 chapter URLs but **1590 blank titles**. Root cause: the raw
server HTML is malformed enough that **both lxml and html.parser** drop the inner
title text of every chapter after the first ~135 (a browser normalises it — which
is exactly why the 031 scroll path, reading `page.content()`, had 0 blanks). A
targeted regex over the raw markup recovered all 1725 titles.
→ Rewrote `parse_chapter_list` to extract `(href, title)` pairs with a regex
(`_CH_ANCHOR_RE`) instead of BeautifulSoup. It works on both the malformed raw
document and the well-formed browser DOM (the fallback). Dedup-by-URL and
sort-by-number logic unchanged. (`parse_metadata`/`parse_chapter` still use
BeautifulSoup — their pages are well-formed.)

## [VERIFY] — 2026-07-24
- Live (3 runs): **1725 chapters, 0 blank titles, ~1.4s** each (was ~10s scrolling).
- Full suite **664 passed**; ruff clean. Fixture-based parser tests still pass

## Net effect
