# 032 — webtruyendich: read the TOC's raw HTML instead of scrolling

## Motivation

The user pointed at the "Xem danh sách chương đầy đủ (HTML Sitemap)" button and
asked if we can use that page instead of scrolling the whole list.

## Findings

- That button links to `/truyen/<slug>/danh-sach-chuong-day-du` — the **same**
  page the scraper already uses. "(HTML Sitemap)" is only a label; there is no
  separate static page, and the site's `sitemap.xml` is novel-level, not chapter-
  level.
- **But** the page's *raw server HTML* already contains **all 1692 chapters**.
  The infinite-scroll is client-side JS *virtualizing* the live DOM down to ~135
  rows for display; the initial document response holds the complete list (that's
  what makes it an HTML sitemap). Verified with a Playwright probe:
  - raw navigation response body: **1692** distinct chapters
  - `page.content()` @domcontentloaded: 1692
  - live DOM after JS settles: ~135 (why 031 had to scroll)

## Fix (better than the 031 scroll)

Parse the TOC from the **raw navigation response** rather than the post-JS DOM —
instant and complete, no scrolling, timing-independent.

- `webtruyendich_browser.py` — `_goto` returns the navigation `Response`.
  `get_html(url, *, prefer_document=False, scroll_item_selector=None)`: when
  `prefer_document`, return the response body if it isn't a Cloudflare challenge;
  otherwise fall back to the DOM path (which may still scroll). Keep
  `_scroll_until_stable` as the fallback so a challenged first response still
  resolves.
- `scrapers/webtruyendich.py` — `fetch_chapter_list` fetches with
  `prefer_document=True` (raw HTML fast path) and keeps `scroll_item_selector`
  as the fallback. `parse_chapter_list` unchanged.
- Tests: `_FakeSession.get_html` accepts `prefer_document`; assert the TOC is
  fetched with it.

## Out of scope

- Landing/metadata and chapter pages are unchanged (chapters need the live AI
  render; the landing page is small and static).
