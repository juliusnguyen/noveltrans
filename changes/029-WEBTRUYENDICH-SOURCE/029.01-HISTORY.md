## [INVESTIGATION] Live chapter page in Chrome — 2026-07-23
Extension reconnected. Opened chuong-1760 in a real browser; Cloudflare JS
challenge passed automatically. **Full mechanism now mapped:**

### The translation control
- `<select id="translator">` — default option **"Vietphrase"** (rule-based
  Sino-Vietnamese transliteration, low quality). Each option's `value` **equals
  its visible label**, so the target value is the literal string
  `"AI Gemini FLASH 3.5 - Memory - No Apikey"`.
- Full option list: Gemini FLASH 3.6 / 3.5 / 3.5 LITE / 3.1 LITE (each in
  "No Apikey", "Memory", and "Custom Prompt" flavours) + Gemma 4 31B.
- A second `<select id="target-lang">` picks output language (default
  Vietnamese; also English/Korean/etc.).
- A **"Dịch lại"** (re-translate) button (`refresh Dịch lại`) triggers generation.

### What selecting Gemini + "Dịch lại" does (network trace via performance API)
1. **Cloudflare Turnstile** verification widget renders ("Verifying…") — a bot
   check that must pass before the AI endpoint responds.
2. `POST /api/getChapterStream` — **streams** the Gemini translation
   ("AI Loading…" shown for ~10–15 s per chapter).
3. `POST /api/updateMemory` — the **"Memory"** feature: persists a
   glossary/context so names & terms stay consistent across chapters.
4. (default/non-AI path uses `/api/getChapter`; `/api/views-count` is a counter.)
- "**No Apikey**" = uses the site's own Gemini quota; no user key required.

### Output
- Renders into `#chapter-content-body` as `<p class="fade-in-paragraph
  has-anchor">`, **each paragraph prefixed with `#\n`** (an anchor marker —
  strip the leading `#`). After stripping: clean, well-formed Vietnamese prose
  (~7.5 KB for this chapter). Quality good; occasional stray CJK char slips
  through (saw one `绝`) — AI is not 100% clean.
- **Content is already Vietnamese** → import as `translated`; our own
  translators are not needed for this source.

### The key obstacle (make-or-break for automation)
- The AI translation is **gated by Cloudflare Turnstile on every generation**.
  Solving/bypassing Turnstile is off-limits. A genuine logged-in browser
  (Playwright driving real Chrome, like the cf_browser path) *may* pass it as a
  managed/invisible challenge — but if it presents an interactive challenge,
  automated bulk scraping of all 1762 chapters is not feasible without a human
  in the loop. **This must be validated before committing to the approach.**
- Even if Turnstile passes: ~10–15 s AI generation per chapter × 1762 chapters
  is hours of runtime and 1762 quota hits on the free "No Apikey" tier — expect
  rate limiting. Worth checking whether the site caches a generated AI
  translation so a re-fetch is instant.

### Open design decision (for the user)
- Import the Gemini output as the chapter's `translated` text (skip NovelTrans
  translators). Store the site's chapter title too. Confirmed content is VI.

### Findings that corrected the plan against real markup
- The `#` is an `<a class="citation-anchor">#</a>` element inside each `<p>`, not a
  text prefix → `parse_chapter` decomposes `a.citation-anchor` (not a regex strip).
- `og:title` is decorated (`[Mới nhất] … | webtruyendich.com`); the `<h1>` "Truyện
  <title>" is cleaner → title prefers h1, falls back to a cleaned og:title.
- No `og:novel:author`; author is an `<a href="/tac-gia/…">` link.
- TOC has 1722 distinct chapter URLs; **32 chapter NUMBERS carry two different
  slugs** → dedup by URL, not by number (the draft would have dropped 32 chapters).

### Caveat for real use
First-ever download of many un-cached chapters is the slow/challenge-prone case
(live AI generation + CF sensitivity). Sequential-only, polite delay, per-chapter
error isolation, one internal retry, and managed-challenge waiting all mitigate;
a hard CF/AI-quota block still surfaces as a clear retry-able error.
