# 029.02 — Initial Plan — webtruyendich.com source

A browser-driven `SiteAdapter` for **webtruyendich.com** that fetches each
chapter's on-site AI-Gemini translation and lands it directly as the chapter's
`translated` text (skipping NovelTrans's own translators). Modeled on the
69shuba adapter, with one genuinely new capability: **interactive** browser
choreography (select a model, click a button, wait hands-off for streamed
content) versus 69shuba's navigate→HTML.

## Architecture at a glance

Three layers, matching the 69shuba split:

- **Browser lifecycle + CF clearance + the mechanical Playwright calls** → new
  module `webtruyendich_browser.py` (sibling of `cf_browser.py`), reusing
  `browser.py`'s launch helpers.
- **Site knowledge (URLs, selectors, model string, cleaning) + wiring** → new
  adapter `scrapers/webtruyendich.py`, with pure `parse_*` functions.
- **DB routing of "pre-translated content"** → a small branch in `DownloadWorker`
  plus three defaulted class attributes on `SiteAdapter`.

## 1. Recommended browser-session design (the hard part)

**Decision: a dedicated new module `src/noveltrans/webtruyendich_browser.py` with
class `WtdBrowserSession`. Do NOT extend `cf_browser.BrowserSession`.**

Rationale:
- `cf_browser.BrowserSession.get_html` is deliberately a pure
  navigate→`page.content()` primitive (cf_browser.py:141-162), shared by
  shuba69. Threading model-selection / button-click / stream-wait into it would
  bloat its single responsibility and risk the one working CF path (69shuba).
  Its docstring and `_BLOCKED_TYPES` comment (cf_browser.py:51-53) are
  69shuba-specific.
- The interaction is site-specific DOM choreography; per the existing convention
  (cf_browser is its own module, each browser caller keeps its own profile dir,
  browser.py:10-13), webtruyendich gets its own module and profile dir.
- Both still share the launch layer: `WtdBrowserSession` imports
  `require_playwright`, `launch_persistent_context`, `close` from
  `noveltrans.browser` exactly as cf_browser does (cf_browser.py:35-36).

**`WtdBrowserSession` — mirror cf_browser's lifecycle, add one interactive primitive:**

```python
# src/noveltrans/webtruyendich_browser.py
from noveltrans.browser import close as _close_browser
from noveltrans.browser import launch_persistent_context, require_playwright
from noveltrans.storage.library import DEFAULT_LIBRARY_DIR

_CHALLENGE_MARKERS = (...same set as cf_browser...)         # reuse verbatim
_BLOCKED_TYPES = {"image", "font", "media", "stylesheet"}   # NEVER block script/xhr — CF + the AI stream are JS/XHR
_LAUNCH_LOCK = threading.Lock()

def profile_dir() -> Path:
    return DEFAULT_LIBRARY_DIR / ".webtruyendich-profile"   # matches the spike's ~/NovelTrans/.webtruyendich-profile

def looks_like_challenge(markup: str) -> bool: ...

class WtdBrowserSessionError(Exception): ...

class WtdBrowserSession:
    def __init__(self, profile=None, *, headless=False, delay_seconds=1.5,
                 timeout_ms=60_000, content_timeout_ms=45_000, sync_playwright=None): ...
    def __enter__/__exit__/close(self): ...      # copy cf_browser lifecycle (lazy _launch, _LAUNCH_LOCK, route once)
    def _launch(self): ...                        # identical to cf_browser._launch
    def _throttle(self): ...                      # identical

    def get_html(self, url: str) -> str:          # plain navigate→content, for landing + TOC (both CF-behaviour-safe)
        ...  # identical to cf_browser.get_html incl. looks_like_challenge guard

    def read_translated_chapter(
        self, url: str, *,
        translator_select: str, translator_value: str,
        retranslate_button: str, content_selector: str, paragraph_selector: str,
    ) -> str:
        """Navigate, choose the model, click re-translate, wait hands-off for the
        streamed paragraphs to stabilize, return the container's innerHTML."""
```

`read_translated_chapter` steps (mechanical Playwright calls live here; selectors
and model string are passed from the adapter, so site knowledge stays in the adapter):
1. `_throttle()`; `page.goto(url, wait_until="domcontentloaded")`;
   `looks_like_challenge(page.content())` → raise `WtdBrowserSessionError`.
2. `page.select_option(translator_select, value=translator_value)` — option value
   equals label, so select by the literal model string.
3. Click re-translate: `page.get_by_role("button", name=retranslate_button).click()`
   (fallback `page.click(f'button:has-text("{retranslate_button}")')`).
4. `_wait_for_stable_content(paragraph_selector, content_selector, content_timeout_ms)`:
   wait for the first `paragraph_selector` (`p.fade-in-paragraph.has-anchor`) to
   appear, then poll `inner_text()` length every ~400 ms; done when non-empty and
   unchanged across two consecutive polls, or raise on timeout. Absorbs both the
   cached ~1.7 s case and the streamed ~10–15 s first-generation case, and
   **never touches the Turnstile widget**; if an interactive challenge ever blocks
   generation, content never stabilizes → timeout → `WtdBrowserSessionError`
   (adapter turns this into a clear ScrapeError).
5. Return `page.locator(content_selector).inner_html()`.

Waiting for the `fade-in-paragraph` class specifically (not just any `<p>`) avoids
reading the stale default "Vietphrase" body that may already be in
`#chapter-content-body` before the click.

Testability boundary preserved: the session returns raw container innerHTML; all
cleaning is in the pure `parse_chapter` (fixture-tested, no browser), like shuba69.

## 2. The key integration — "import as translated"

**How the pipeline currently sets `translated`:** `DownloadWorker.run` calls
`adapter.fetch_chapter(ref)` then `project.save_content(chapter.index, text)`
(workers.py:1277-1278), which writes only `content` and sets status `downloaded`
(project.py:289-295). `translated`/`target_lang`/`translator` are set later by
`TranslateWorker` via `project.save_translation(...)` (project.py:297-312).

**Existing Vietnamese-source precedent (medoctruyen):** it sets `source_lang="vi"`
(medoctruyen.py:112) and stores the Vietnamese body in `content`. On "Dịch" with
target `vi`, `TranslateWorker` detects `meta.source_lang == target_lang` and takes
the **identity path** `_run_identity` (workers.py:178-179, 136-162), copying
`content`→`translated` with translator label `"(nguyên bản)"`. So today VI sources
still require a second (no-op) "translate" click and get a generic label.

**Product decision here is stronger:** import Gemini output *directly* as
`translated`, skip translators, label it `"webtruyendich (Gemini Flash 3.5)"`.

**(a) Add three defaulted class attributes to `SiteAdapter`** (base.py, near 93-95):
```python
content_is_translated: bool = False  # True → fetch_chapter returns a finished translation, not source text
translated_lang: str = ""            # language of that translation (e.g. "vi")
translator_label: str = ""           # recorded in Chapter.translator
```

**(b) Branch in `DownloadWorker.run`** (workers.py:1277-1279), reusing existing
storage methods — no schema change:
```python
text = self._fetch_with_backoff(adapter, chapter, done, total)
if adapter.content_is_translated:
    # Pre-translated source (Vietnamese MTL). Land it as the translation directly and
    # skip NovelTrans's own translators. Write content too, else pending_download
    # (content == '') would re-fetch this chapter forever — and it lets "voice the
    # original" / original-text export still work.
    project.save_content(chapter.index, text)
    project.save_translation(
        chapter.index, chapter.title, text,
        adapter.translated_lang, adapter.translator_label,
    )
else:
    project.save_content(chapter.index, text)
```
After this a chapter is simultaneously "downloaded" and "translated" in one pass.

**Why write both columns:** `pending_download` keys off `content == ''`
(project.py:215-219); empty `content` would re-queue every scan. The Vietnamese
text *is* the source NovelTrans fetched, so original-text TTS/export keep working.

**(c) Set `source_lang="vi"` in `parse_metadata`** (like medoctruyen): a later
manual "Dịch → vi" hits the identity path with an **empty** pending list
(`pending_translation` excludes rows already `target_lang == vi`,
project.py:222-236), so our `translator` label is never clobbered; and a
different-language target treats the Vietnamese as source.

**Meta front-matter:** no `save_meta_translation` needed — exporters/video fall
back via `meta.translated_title or meta.title` (workers.py:843, 992); title is
already Vietnamese.

Production code touched: base.py attributes + workers.py branch (two files), plus
the new adapter/session modules.

## 3. `parse_metadata` (landing page)

Landing `.../truyen/<slug>` is open (HTTP 200, server-rendered), fetched through
the session's `get_html`. Defensive design (selectors to confirm at fixture time):

```python
def parse_metadata(markup: str, url: str, site: str) -> NovelMeta:
    soup = BeautifulSoup(markup, "lxml")
    def og(prop): ...  # meta[property='og:...'] content, like shuba69
    title = og("og:title")
    if not title:
        h1 = soup.select_one("h1.text-2xl.sm\\:text-3xl.font-bold") or soup.select_one("h1")
        title = h1.get_text(strip=True) if h1 else ""
    title = re.sub(r"^\s*Truyện\s+", "", title).strip()   # the h1 is "Truyện <title>"
    if not title:
        t = soup.select_one("title"); title = t.get_text(strip=True) if t else ""
    if not title:
        raise ScrapeError("Novel title not found — page layout may have changed", url)
    return NovelMeta(
        url=url, site=site, title=title,
        author=og("og:novel:author"),          # may be absent → ""
        description=og("og:description"),
        cover_url=og("og:image"),
        source_lang="vi",
    )
```
`url` echoed unchanged (see §5).

## 4. `parse_chapter_list` (full TOC)

`/danh-sach-chuong-day-du` is CF-gated → fetch via `session.get_html`. Links
follow `/truyen/<slug>/fanqie/chuong-<N>-<slug>`, listed newest-first.

```python
SEL_TOC_LINKS = 'a[href*="/fanqie/chuong-"]'
_CH_NUM_RE = re.compile(r"/chuong-(\d+)(?:[-/]|$)")

def parse_chapter_list(markup: str, base_url: str) -> list[ChapterRef]:
    soup = BeautifulSoup(markup, "lxml")
    by_num: dict[int, tuple[str, str]] = {}      # N -> (title, absolute_url); first seen wins
    for a in soup.select(SEL_TOC_LINKS):
        m = _CH_NUM_RE.search(a.get("href") or "")
        if not m: continue
        n = int(m.group(1))
        by_num.setdefault(n, (a.get_text(" ", strip=True), urljoin(base_url, a["href"])))
    if not by_num:
        raise ScrapeError("Chapter list not found — page layout may have changed", base_url)
    return [ChapterRef(index=i, title=t, url=u)
            for i, (n, (t, u)) in enumerate((n, by_num[n]) for n in sorted(by_num))]
```
Sorting by `N` converts newest-first → reading order, `index` ascending 0-based.
Dedup by `N` handles the 3 latest chapters also appearing on the landing area.

## 7. The adapter class + `parse_chapter`

```python
_ANCHOR_RE = re.compile(r"^#\s*")   # every AI paragraph is prefixed "#\n" (anchor marker)

def parse_chapter(container_html: str, url: str) -> str:
    soup = BeautifulSoup(container_html, "lxml")
    ps = soup.select(SEL_PARAGRAPH) or soup.find_all("p")
    lines = []
    for p in ps:
        text = _ANCHOR_RE.sub("", p.get_text(" ", strip=True)).strip()
        if text and text != "#":
            lines.append(text)
    if not lines:
        raise ScrapeError("Chapter content is empty", url)
    return "\n\n".join(lines)   # stray CJK left as-is — tolerated, not fatal

@register
class WebtruyendichAdapter(SiteAdapter):
    name = "webtruyendich"
    display_name = "Webtruyendich (webtruyendich.com)"
    url_patterns = [r"webtruyendich\.com/truyen/[a-z0-9-]+"]
    content_is_translated = True
    translated_lang = "vi"
    translator_label = TRANSLATOR_LABEL

    def __init__(self, client, *, headless=False):
        super().__init__(client)
        self._headless = headless
        self._session: WtdBrowserSession | None = None

    def _ensure_session(self):   # lazy, mirrors shuba69._get_html status message + error mapping
        if self._session is None:
            self._status("🌐 Đang mở trình duyệt để đọc webtruyendich (Cloudflare + dịch AI)…")
            self._session = WtdBrowserSession(headless=self._headless,
                                              delay_seconds=self.client.delay_seconds)
        return self._session

    def _get_html(self, url): ...        # ensure_session().get_html(url) wrapped in the shuba69 try/except
    def close(self): ...                 # idempotent session teardown (shuba69.py:215-219)

    def fetch_metadata(self, url):       return parse_metadata(self._get_html(landing_url(url)), url, self.name)
    def fetch_chapter_list(self, url):   return parse_chapter_list(self._get_html(toc_url(url)), url)
    def fetch_chapter(self, ref):
        try:
            html = self._ensure_session().read_translated_chapter(
                ref.url, translator_select=SEL_TRANSLATOR, translator_value=TRANSLATOR_MODEL,
                retranslate_button=RETRANSLATE_BUTTON, content_selector=SEL_CONTENT,
                paragraph_selector=SEL_PARAGRAPH)
        except BrowserUnavailableError as exc:
            raise ScrapeError("Cần trình duyệt để đọc webtruyendich … pip install 'noveltrans[browser]' && playwright install chromium", ref.url) from exc
        except WtdBrowserSessionError as exc:
            raise ScrapeError("Không đọc/dịch được chương webtruyendich — có thể bị Cloudflare Turnstile chặn hoặc hết hạn mức AI. Thử lại. (" + str(exc) + ")", ref.url) from exc
        return parse_chapter(html, ref.url)
```
Error strings mirror shuba69.py:200-213. `fetch_metadata`/`fetch_chapter_list`
reuse the same session (one Chrome for the whole scan+download).

**Register it:** add `webtruyendich` to the import tuple in `scrapers/__init__.py:26-31`.

## 8. Politeness / robustness

- One Chrome, sequential only. Session `_throttle()` (delay from
  `client.delay_seconds`) is the sole rate limiter — same reasoning as
  cf_browser.py:17-23. **Do not parallelise** (CF + shared AI quota → block risk).
- Per-chapter `content_timeout_ms≈45 s`; on timeout → `WtdBrowserSessionError` →
  ScrapeError. `DownloadWorker` keeps the batch going on `NovelTransError`
  (workers.py:1289-1292), so a stuck chapter is errored, not fatal.
- Runtime caveats to note: first-ever (un-cached) read is the slow,
  rate-limit-prone case (~10–15 s vs cached ~1.7 s); a hard AI-quota block
  surfaces as repeated content timeouts. Interactive Turnstile didn't appear in
  the spike; the timeout path is the safety net.
- `DownloadWorker`/`ScanWorker` already call `adapter.close()` in `finally`
  (workers.py:82-83, 1301-1303) — no browser leak.

## Risks / caveats
- **Selector drift** (`#translator` option value, `p.fade-in-paragraph`,
  `#chapter-content-body`, the "Dịch lại" button text) — re-confirm at
  fixture-capture time; all wrapped in clear ScrapeErrors if they break.
- **Un-cached chapters are slow / rate-limit-prone**; the 45 s timeout + per-chapter
  error isolation contain it, but a 1762-chapter first download may hit AI-quota
  throttling — sequential-only is deliberate.
- **Turnstile**: invisible in the spike; stabilize-or-timeout is the fallback,
  never interacting with the widget.
- **Two-column write** (`content` + `translated`) is intentional; "optimising" it
  to translated-only would resurrect infinite re-downloads via `pending_download`.
