# 029 — webtruyendich.com source

## Request

Add support for scraping from **webtruyendich.com**, starting with this novel:

    https://webtruyendich.com/truyen/ta-manh-nhat-doc-si-nu-de-goi-thang-nguoi-gian-ac

**In each chapter, use the site's `AI Gemini FLASH 3.5 - Memory - No Apikey`
option** to obtain the (translated) chapter content.

## What we know (verified against the codebase)

- Existing scrapers live in `src/noveltrans/scrapers/`:
  `giatocvuongtai, ixdzs, medoctruyen, shuba69, xbanxia`. Add a new
  `webtruyendich.py` following the same `Scraper` base (`scrapers/base.py`)
  and register it (`scrapers/__init__.py`). Naming matches prior features
  (e.g. 015-69SHUBA-SOURCE, 003-MEDOCTRUYEN-SOURCE).
- **`AI Gemini FLASH 3.5 - Memory - No Apikey` is NOT an existing translator
  in this app** — there is no Gemini/Flash code anywhere, and there's no public
  "Gemini Flash 3.5" model. This is almost certainly an **on-page option on
  webtruyendich.com** (the site offers per-chapter AI translation with a model
  dropdown), not one of our `translators/`. So the scraper likely has to select
  that option and read the site's rendered translation, rather than us
  translating the raw text ourselves.

## Open questions (resolve during /plan — likely needs looking at the live site)

1. **Is the content Chinese-raw or already Vietnamese?** webtruyendich = "web
   truyện dịch" (translated-stories site). If the "Gemini Flash" option produces
   the Vietnamese text on the site, does this novel skip our own translation
   step entirely (import as already-translated), or do we still store raw +
   translate in-app?
2. **Does selecting that AI option require a browser?** If the translation is
   generated client-side / behind JS or a button, this needs Playwright/Chrome
   like `medoctruyen` (Discord unlock) and `shuba69` (Cloudflare) do —
   see `browser.py` / `cf_browser.py`. Plain `requests` may not suffice.
3. **Login / rate limits / Cloudflare?** Check whether the site gates chapters.
4. **Where does the "No Apikey" option live in the DOM**, and is it sticky per
   session or must be re-selected each chapter (the request says "in each
   chapter", implying per-chapter selection)?
5. **TOC + chapter URL structure** for `/truyen/<slug>` and its chapter pages.

## Scope

- New `scrapers/webtruyendich.py` + registration + tests.
- Whatever mechanism is needed to trigger and read the `AI Gemini FLASH 3.5 -
  Memory - No Apikey` translation for each chapter.
- Fit the existing scraper interface (TOC listing, chapter fetch) so the novel
  flows through the normal download → (translate?) → audio/export pipeline.
