# 011 — medoctruyen title badges — History

## 2026-07-16 — Fix + robustness

**Bug:** medoctruyen chapter titles ended with "… Đã đọc" (and would with other badges).
`fetch_chapter_list` used `a.get_text()`, which flattens the title span AND the status-
badge spans the TOC renders inside each chapter `<a>`. Only "Miễn phí" was stripped.

**Fix (`scrapers/medoctruyen.py`):** new `_chapter_title(a)` — takes the title span (the
child `<span>` starting with "Chương") and ignores every other span, so *any* badge is
dropped structurally. Fallback for the compact "latest chapter" links (title as plain
`<a>` text, no spans): strip a set of known trailing badges (`_BADGE_TAGS` = Đã đọc,
Miễn phí, VIP, Trả phí, Mới), repeatedly for multi-badge links.

**Tests:** TOC fixture updated to the real span markup (ch.1 with a "Đã đọc" badge span,
ch.2 "Miễn phí" span) + a plain-text multi-badge case (ch.3 "… Miễn phí Đã đọc");
assertions that no badge leaks into any title.

**Verification:** ruff clean; 237 tests pass. Live-checked against
`tu-bao-tien-bon` (1954 chapters) — 0 titles with leaked badges; ch.1 now
"Chương 1: Nhặt được chậu rách" (was "… Đã đọc").

**Note:** already-scanned projects keep their old titles until re-scanned (the bad title
was stored at scan time). Only medoctruyen — giatocvuongtai titles come from a JSON API.
