# 011 — medoctruyen: strip status badges from chapter titles

## Request

Chapter titles from medoctruyen.vn end with "… Đã đọc" (and similar). Fix it, and make
it robust to other badges (VIP, Mới, Miễn phí, …).

## Root cause (verified against the live site)

medoctruyen's TOC wraps the title and per-chapter status badges in **separate `<span>`s**
inside the chapter `<a>`:

```html
<a href="/…/chuong-1">
  <span class="flex-1 truncate …">Chương 1: Nhặt được chậu rách</span>   <!-- title -->
  <span class="… text-text-muted">Đã đọc</span>                          <!-- badge -->
</a>
```

`fetch_chapter_list` extracts the title with `a.get_text(" ", strip=True)`
(medoctruyen.py:134), which flattens *all* inner text — so the badge ("Đã đọc" for
already-read chapters, "Miễn phí" for free ones) lands in the title. Only "Miễn phí" was
stripped (`_FREE_TAG`, line 142); "Đã đọc" and any other badge leak through. The bad
title is stored at scan time, so it also flows into exports and audiobook chapter markers.

medoctruyen-only — giatocvuongtai titles come from a clean JSON API.

## Fix

- Take the **title span** (the child `<span>` whose text starts with "Chương") and ignore
  every other span — robust to *any* badge, present or future.
- Fallback for the plainer link variant (the "latest chapter" links render the title as
  plain `<a>` text with no spans): strip a set of known trailing badges
  ("Đã đọc", "Miễn phí", "VIP", "Mới", "Trả phí"), repeatedly, in case of several.
- Update the TOC fixture to the real span markup (with a badge) + keep a plain-text badge
  case; assert extracted titles carry no badge text.
- Note: existing scanned projects keep their bad titles until re-scanned (document).
