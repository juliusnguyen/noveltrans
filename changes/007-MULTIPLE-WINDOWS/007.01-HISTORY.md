# 007 — Multiple windows (workspace tabs) — History

## [PLAN] 007.02-INITIAL-PLAN — 2026-07-15

Generated plan: `007.02-INITIAL-PLAN.md`

Clarified with the user: "multiple windows" = an outer bar of **workspace tabs**, each
an independent crawl/translate/TTS flow (not separate OS windows).

Design: extract `MainWindow`'s inner four-tab stack + cross-tab wiring into a new
`Workspace(QWidget)`; `MainWindow` becomes an outer `QTabWidget` host keeping only
app-global concerns. Key decisions recorded in the plan:
- Single shared `AppState` injected into all workspaces (avoids JSON clobber).
- Same-project-open guard: registry in `MainWindow`, **warn-and-focus** on duplicate
  (hard lockfile guard deferred as follow-up).
- Dock badge stays app-global (option A); accurate per-workspace counter deferred.
- One workspace on launch reopening last project (session restore deferred).
- One small tab tweak: add `ScrapeTab.current_title()` for the outer tab label.

## 2026-07-15 — Implementation

Built the plan. New `gui/workspace.py` (`Workspace(QWidget)` = inner four-tab stack +
wiring + `shutdown`/`has_running_workers`/`current_project_path`/`current_title`/
`reopen_last_project`, signals `title_changed`/`project_opened`); rewrote
`gui/main_window.py` as the outer `QTabWidget` host (shared `AppState`, `_add_workspace`/
`_close_workspace`, guard registry, New/Close actions Cmd+T/Cmd+W, Settings→left corner
& "＋"→right corner, rewritten `closeEvent`). Added `current_title()` +
`has_running_workers()` to the four tabs. New `tests/test_main_window.py` (9 tests,
offscreen).

**Three deviations from the plan, all discovered during implementation:**
1. **Guard registers only on `scrape_tab.project_changed`, not on picker selections.**
   Routing the pickers into the fan-out handler recursed infinitely: `ProjectPicker.refresh()`
   re-emits `project_selected` even on programmatic re-select (widgets.py:82), so
   refresh→emit→refresh looped. Reverted the pickers to the old direct
   `→ state.touch_project` wiring; only the scrape tab (the primary SQLite writer)
   drives the guard + title. Residual risk: selecting a duplicate project in a
   translate/audio tab of another workspace isn't guarded — consistent with the
   "warn-only, hard lock is a follow-up" stance.
2. **Dropped the `currentChanged → refresh_pickers` auto-refresh** (plan §5): the
   picker re-emit made it re-touch/reload on every tab switch. Cross-workspace
   recent-list refresh deferred as a follow-up; core parallelism unaffected.
3. **`has_running_workers()` excludes the models/voices background fetches.** Those
   start at tab construction; counting them made closing a fresh workspace pop a false
   "đang chạy?" confirm (and hung headless tests on the modal). `shutdown()` still joins
   those threads.

**Verification:** ruff clean; 204 tests pass (195 + 9 new). Offscreen smoke test:
window shows, add/close workspaces works, first tab labels itself with the loaded
novel title. Real parallel downloads = manual (see plan §Manual).

## 2026-07-15 — Fixes from user feedback (empty picker, clean guard, UI polish)

User reported: (1) in a 2nd workspace, couldn't choose recent novels; (2) UI not good
enough (＋/⚙ buttons, nested tabs confusing, tab labels/close, spacing).

**Empty-picker bug (the real cause of #1):** a new workspace never populated its
pickers (`reopen_last_project` ran only for workspace #1), so "Truyện gần đây" was
blank. Added `ProjectPicker.refresh(..., default_to_first=False)` (populate the list
but select nothing) and `Workspace.populate_lists()`, called for every workspace. New
tabs now list the whole library, unselected.

**Clean same-project guard (user chose "keep blocking"):** the guard used to load the
novel in the 2nd tab, then warn + focus the owner — leaving it half-open in two tabs.
Reworked to a pre-open veto: `ScrapeTab.can_open_project` callback → `Workspace._can_open_project`
→ `MainWindow._claim_project(ws, path) -> bool`. On conflict it warns, focuses the owner,
and the picker reverts without opening any SQLite. `_register_project`/`_unregister_workspace`
→ `_claim_project`/`_release_workspace`. Residual: translate/export/audio pickers still
open a project without the veto (documented; scrape — the download writer — is guarded).

**UI polish:** outer tabs restyled browser-like and made visually distinct from the
inner step-tabs (`#workspaceTabs` QSS: document mode, rounded-top tabs on their own
surface, no stretch, elide). ＋/⚙ became flat `#cornerButton` icon buttons. Added
explicit `#tabCloseButton` ✕ per tab (Qt's default close icon is invisible on the dark
theme) with a red hover. Verified via offscreen screenshots.

**Verification:** ruff clean; 205 tests pass (+1 guard test). Screenshot-checked the
tab bar renders as intended.
