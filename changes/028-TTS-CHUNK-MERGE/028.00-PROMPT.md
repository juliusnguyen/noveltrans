# 028 — TTS chunk merge (fix garbled audio on short fragments)

## Request

Fix the occasional "weird sound" in VieNeu-TTS output by merging very short
text chunks into their neighbours before synthesis.

## Why (evidence, from the 027 investigation)

Empirically synthesized 147 representative chunks from the problem project
(`dm-edit-chao-mung-den-voi-phong-livestre-3c512475`) and measured actual audio
duration vs expected (chars ÷ ~15/sec). Drift/babble shows up as a duration
ratio ≫ 1. Result:

| chunk length | drift rate (ratio > 1.6) |
|---|---|
| 0–10 chars | 80% |
| 10–20 | 21% |
| 20–40 | 5% |
| **40+** | **0%** |

All 8 drift cases were ≤26 chars; every chunk ≥40 chars was clean regardless of
digits / Latin text / missing terminal punctuation. This is the classic
autoregressive-TTS runaway: too little context → end-of-speech prediction
misfires → the model babbles until the token cap (ratio up to 4.08).

The reported symptom was "long paragraphs", but the cause is the **opposite** —
tiny fragments. `split_sentences` (tts/base.py) caps the *maximum* chunk length
but has no *minimum*, so a one-word paragraph or a stray number becomes its own
doomed micro-chunk. The `dm-edit` project is worst because its livestream-style
prose has many short lines.

## Fix

A `merge_short_chunks(chunks, min_chars, max_chars)` pass that coalesces any
chunk below a floor (~30 chars) into a neighbour without exceeding max_chars.

Quantified (mergesim.py):
- Problem project: 22.0% sub-floor chunks → **0%** after merge, 0 oversized.
- All 81 chapters: 16.0% → **0%**, 0 oversized.

## Scope

- `merge_short_chunks` in `tts/base.py`; `TtsEngine.min_chunk_chars = 30`.
- Apply in `synthesize_chapter` after `split_sentences`.
- Keep `split_sentences` pure (its unit tests unchanged).
- Unit tests for the merge pass.

## Also in this change

- Audio tab "Ký tự" (character-count) column (started during the investigation).

## Out of scope (deferred / optional follow-ups)

- Surfacing vieneu `repetition_penalty` (default 1.2) as a guard.
- Duration-sanity retry for the rare stochastic tail.
- GPU/MPS support — that is feature 027.
