# 028 — TTS chunk merge — History

## [FEATURE] 028-TTS-CHUNK-MERGE — 2026-07-23
Branch `changes/028-TTS-CHUNK-MERGE`. Prompt: `028.00-PROMPT.md`.
Root-cause investigation was done under 027 (see `027.01-HISTORY.md`): garbled
audio correlates with *short* chunks, not long paragraphs — drift rate 80% under
10 chars, 0% at 40+. Fix is a minimum-length merge pass.

## [IMPL] merge_short_chunks — 2026-07-23
`src/noveltrans/tts/base.py`:
- New pure function `merge_short_chunks(chunks, min_chars, max_chars)`: joins any
  sub-floor fragment into the previous chunk (space-joined) when it fits under
  max_chars; leaves an un-mergeable fragment as-is rather than create an
  oversized chunk (whose failure mode is worse). `min_chars <= 0` is a no-op.
- New class attr `TtsEngine.min_chunk_chars = 30`.
- `synthesize_chapter` applies the merge pass right after `split_sentences`.
- `split_sentences` left pure/untouched → all its existing unit tests unchanged.

Cross-paragraph merging is allowed (the flat chunk list is merged after the
per-paragraph split). A tiny dangling fragment therefore loses its 0.4 s
paragraph gap when absorbed — intended: that fragment was the drift source.
