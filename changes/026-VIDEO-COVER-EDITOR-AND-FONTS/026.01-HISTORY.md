# 026 — Per-video folders, remembered mode, a live cover editor & more fonts — History

## [FEATURE] 026-VIDEO-COVER-EDITOR-AND-FONTS — 2026-07-21
Follow-up to 025, driven by a series of user requests on the Video tab. Branch
`changes/026-VIDEO-COVER-EDITOR-AND-FONTS`. No YouTube upload (still parked). Delivered in
four increments:

### 1. Per-video folders + remembered mode + a separate thumbnail font
- **One folder per video.** Each part now renders to its own subfolder
  `exports/video/<stem>/<stem>.mp4`, with its sidecars (`.title.txt` / `.txt` / `.tags.txt`
  / `.jpg`) beside it, so a single part can be uploaded without hunting among the others.
  `video.py` gains `video_part_dir_name`; `VideoWorker` writes into the subfolder and
  `tab_video._part_output_path` resolves it, with a **legacy fallback** so older flat renders
  still count as "đã tạo" and keep opening. The **Chi tiết → Mở thư mục video** button now
  opens that part's own folder (`_open_part_dir`); the top-level button still opens the whole
  `video/` dir.
- **Remembered "Chế độ".** `config.video_mode` + `config.video_batch_size` persist the mode
  (Toàn bộ / Theo lô / …) and batch size across restarts (were hard-coded to batch/10).
- **Thumbnail font.** `config.video_thumbnail_font` + a "Phông bìa" picker in the thumbnail
  box, chosen independently of the video font (defaults to Nunito — soft, rounded, Vietnamese).
  The cover render uses it via `VideoWorker.thumb_font_key`.

### 2. Regenerate the cover only (no re-render)
`render_thumbnail` is pure Pillow, so the cover can be redone without the hours-long video
encode. Added `tab_video._render_thumbnail_now` + `_regen_part_thumbnail` (single, in the
Chi tiết dialog) and `_regen_all_thumbnails` (bulk, "Tạo lại tất cả ảnh bìa" in the thumbnail
box, wait-cursor + per-part status). Base image = the dedicated "Ảnh bìa" or the video "Ảnh
nền" fallback.

### 3. Live cover editor (`gui/thumbnail_editor.py`)
`ThumbnailEditorDialog` — a WYSIWYG editor opened via "Tùy chỉnh ảnh bìa…". Drag the title or
"PHẦN N" block on the preview (or nudge with X/Y sliders), switch fonts and see the result
live, then **Lưu** (persist) or **Lưu & áp dụng cho tất cả** (persist + regenerate every
part's cover). To support it, `thumbnail.py` was split into a disk-free `compose_thumbnail`
(returns a PIL image, used for the live preview) + a thin `render_thumbnail`, and both gained
`title_pos` / `part_pos` — (x, y) fractions of the frame (title = top-left anchor, part =
top-centre) — persisted as `config.video_thumbnail_title_pos` / `_part_pos` (clamped 0–1) and
threaded through the render/regen paths. Defaults reproduce the original fixed layout exactly.

### 4. Seven more bundled fonts
Added Roboto, MuseoModerno, Noto Serif, Pacifico, Dancing Script, Sedgwick Ave and Amatic SC
to `VIDEO_FONTS` (both the video-title and thumbnail pickers). Sourced from google/fonts (OFL);
the four shipped-as-variable families (Roboto, MuseoModerno, Noto Serif, Dancing Script) were
instanced to a static Regular (wght=400, wdth=100) with fontTools to match the existing
static-Regular convention. Each was verified to render the full Vietnamese diacritic set
(`ệữỗậọằẽỹđĐơư`, incl. đ Đ ơ ư) and to carry a family name matching its registry entry — the
family-match + Vietnamese-coverage guard test now parametrizes over all 13 fonts.
