# Implementation Plan — Video Export (feature 019)

## Summary

Add a "Xuất video" (Export video) capability to Tab 4 that renders per-chapter audio into one or more MP4s: a user-chosen background image scaled to 1920×1080 with a blurred fill, the concatenated chapter audio as the soundtrack, and a burned-in ASS subtitle track showing the persistent novel title plus a per-chapter title that changes at each chapter boundary. It is a structural clone of `tts/merge.py` + `MergeWorker` + the merge QGroupBox, reusing `plan_merge_windows`, `MergeSegment`, `build_concat_list`, and the cumulative-ms timing math verbatim.

The work splits into five concrete pieces, each mirroring an existing counterpart:

| New | Mirrors | File |
|---|---|---|
| `tts/video.py` (pure builders + `render_video`) | `tts/merge.py` | new file |
| bundled font asset | `translators/data/*.json` | `tts/assets/` |
| `VideoWorker` | `MergeWorker` | `gui/workers.py` |
| "Xuất video" QGroupBox | `_build_merge_box` | `gui/tab_audio.py` |
| `test_video.py` | `test_merge.py` | `tests/` |

The pure builders (window selection reuse, ASS document, ASS escaping, ffmpeg argv construction) are unit-tested with no ffmpeg; only the actual shell-out is `@pytest.mark.skipif(ffmpeg absent)`, exactly as `test_merge.py` does.

### 1a. ASS escaping — the load-bearing pure function

This is the analogue of `_escape_ffmetadata` and MUST be tested. ASS is a comma-delimited, brace-delimited format; a chapter title like `Chương 3: {Bí mật}, phần 2` or one containing a newline would otherwise corrupt the `Dialogue:` line or inject an override tag.

```python
def _escape_ass(text: str) -> str:
    """Make arbitrary title text safe inside an ASS Dialogue Text field.

    - '{' / '}' start/end ASS override blocks -> replace with fullwidth or strip.
    - literal newlines end the event -> convert to the ASS soft break '\\N'.
    - a trailing backslash could swallow the terminator -> neutralize.
    Commas are safe in the LAST field (Text) because ASS only splits the first 9.
    """
```

Concrete rules to implement and assert:
- Replace `{` → `(` and `}` → `)` (simplest robust choice; braces are ASS's only in-text metacharacter and titles never legitimately contain override tags). Alternative: escape as `\{`/`\}`, but libass brace handling is finicky, so substitution is safer.
- `\r\n`, `\n`, `\r` → `\N` (ASS hard line break) so multi-line titles wrap instead of truncating.
- Collapse a title ending in `\` (would escape nothing meaningful but is defensive).
- Do NOT escape commas — the Text field is the 10th and final field, and libass treats the remainder of the line as text. Add a test asserting a comma title survives verbatim.

Because Text is the final field, the escaping surface is small and well-defined; the test file should cover braces, newline, comma, and a CJK+Vietnamese mixed title.

### 1b. ASS document builder

```python
def build_ass_subtitles(
    segments: list[MergeSegment],
    novel_title: str,
    *,
    play_res_x: int = 1920,
    play_res_y: int = 1080,
    font_name: str = "Noto Sans",
    novel_font_px: int = 56,
    chapter_font_px: int = 72,
) -> str:
    """An ASS document: a persistent novel-title event + one per-chapter event.

    Chapter events are timed by the cumulative sum of MergeSegment.seconds — the
    SAME offset math as build_chapter_metadata, but formatted as ASS H:MM:SS.cs.
    """
```

Timing helper (mirror `build_chapter_metadata`'s cumulative loop, but centiseconds because ASS timestamps are `H:MM:SS.cc`):

```python
def _ass_time(total_seconds: float) -> str:
    cs = int(round(total_seconds * 100))
    h, cs = divmod(cs, 360000)
    m, cs = divmod(cs, 6000)
    s, cs = divmod(cs, 100)
    return f"{h}:{m:02d}:{s:02d}.{cs:02d}"  # ASS uses H:MM:SS.cc (dot before centiseconds)
```
Accumulate `start` and `end` per segment identically to the merge metadata loop so timings line up exactly with the audio boundaries; the total end for the last chapter is the sum of all durations (and `-shortest` at encode time guarantees the video ends with the audio regardless of rounding).

Document structure the builder emits:

```
[Script Info]
ScriptType: v4.00+
PlayResX: 1920
PlayResY: 1080
WrapStyle: 2
ScaledBorderAndShadow: yes

[V4+ Styles]
Format: Name, Fontname, Fontsize, PrimaryColour, ... , BorderStyle, Outline, Shadow, Alignment, MarginL, MarginR, MarginV, Encoding
Style: Novel,   Noto Sans, 56, &H00FFFFFF, ..., 4, 0, 0, 8, 60, 60, 60, 1
Style: Chapter, Noto Sans, 72, &H00FFFFFF, ..., 4, 0, 0, 2, 60, 60, 120, 1

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:00.00,<total>,Novel,,0,0,0,,<escaped novel_title>
Dialogue: 0,<c1 start>,<c1 end>,Chapter,,0,0,0,,<escaped chapter 1 title>
Dialogue: 0,<c2 start>,<c2 end>,Chapter,,0,0,0,,<escaped chapter 2 title>
...
```

ASS specifics to get right:
- **Scrim for legibility (the decided requirement):** use `BorderStyle=4` (opaque box) in both styles, with a semi-transparent `BackColour` (e.g. `&H80000000` — the `80` is alpha) so a mid alpha gives a dark translucent box hugging the text. This gives the "subtle dark scrim behind the text" without compositing a separate rectangle.
- **Alignment (`\an` numpad convention):** Novel style `Alignment: 8` (top-center, persistent), Chapter style `Alignment: 2` (bottom-center) with a larger `MarginV` so it sits clear of the frame edge. This keeps the two elements from overlapping.
- **PlayResX/Y = 1920/1080** so font pixel sizes are absolute to the output frame (libass scales the burn to the video resolution).
- **`Encoding: 1`** (default) — Unicode handling is by the font, not the codepage.
- Colours are ASS BGR hex with a leading alpha byte (`&HAABBGGRR`). White text = `&H00FFFFFF`.

The whole document is one `str`, written to a temp `.ass` file by `render_video` (like `merge.py` writes `list.txt`/`chapters.txt`).

Return the document as a plain string so the test can assert: header present, `[Events]` has `len(segments)+1` `Dialogue:` lines, the first chapter's Start is `0:00:00.00`, the second chapter's Start equals the first chapter's duration, the novel title spans `0` → total, and an injected malicious title is escaped.

## 2. The ffmpeg call — `render_video`

Signature and structure mirror `merge_chapters` exactly (Popen + poll-for-cancel + stderr→file + deadline + temp-file cleanup in `finally`):

```python
def render_video(
    segments: list[MergeSegment],
    image_path: Path,
    out_path: Path,
    font_dir: Path,          # dir containing the bundled font (libass fontsdir)
    *,
    width: int = 1920,
    height: int = 1080,
    fps: int = 12,
    cancelled: Callable[[], bool] | None = None,
) -> Path:
```

Steps inside:

1. Guard: empty segments → `TtsError`; early cancel → `MergeCancelled` (copy from `merge_chapters`).
2. Temp dir `noveltrans-video-`. Write three temp files: `list.txt` (`build_concat_list([s.path for s in segments])`), `subs.ass` (`build_ass_subtitles(...)`), `err.txt`. **Key decision: concatenate the audio to a temp file first**, then feed that single audio as `-i`. This makes `-shortest` behave predictably and decouples audio concat from the video filtergraph. Reuse the existing concat demuxer for that step:

   - Audio concat command (re-encode to AAC so the MP4 mux is clean and durations are exact):
     ```
     ffmpeg -y -f concat -safe 0 -i list.txt -c:a aac -b:a 96k <tmp>/audio.m4a
     ```
     (Could `-c copy` if all inputs are identical-codec AAC, but per-chapter files may be WAV/MP3 mixed; re-encoding to AAC once is simplest and matches the merge module's 96k choice.)

3. **The render command** — background image looped, ASS burned, audio muxed, ends with audio:

   ```
   ffmpeg -y \
     -loop 1 -framerate {fps} -i {image_path} \
     -i {tmp}/audio.m4a \
     -filter_complex "\
       [0:v]scale={W}:{H}:force_original_aspect_ratio=increase,\
            crop={W}:{H},boxblur=20:2,\
            eq=brightness=-0.25[bg];\
       [0:v]scale={W}:{H}:force_original_aspect_ratio=decrease[fg];\
       [bg][fg]overlay=(W-w)/2:(H-h)/2,\
            subtitles={tmp}/subs.ass:fontsdir={font_dir}[v]" \
     -map "[v]" -map 1:a \
     -c:v libx264 -tune stillimage -pix_fmt yuv420p -r {fps} \
     -c:a copy \
     -shortest \
     {out_path}
   ```

   Filtergraph rationale (the decided "scale to cover + blurred/darkened copy behind" approach):
   - `[bg]`: scale the image to **cover** the frame (`force_original_aspect_ratio=increase` then `crop`), blur it (`boxblur`), and darken it (`eq=brightness=-0.25`) — the blurred fill that removes black bars for any aspect ratio.
   - `[fg]`: scale the same image to **fit** inside the frame (`force_original_aspect_ratio=decrease`), preserving the full undistorted image.
   - `overlay` centers the sharp fit image over the blurred cover → any aspect ratio fills 1920×1080 with no black bars and no distortion.
   - `subtitles=...:fontsdir=...` burns the ASS. `fontsdir` points libass at the bundled font directory; the ASS style `Fontname` must match the font's internal family name (§3).
   - `-tune stillimage` optimizes x264 for a static frame (tiny video bitrate; almost all size is audio). `-pix_fmt yuv420p` for player/YouTube compatibility. `-r {fps}` caps output framerate.
   - `-shortest` ends the video exactly when the audio ends.
   - `-c:a copy` because audio was already encoded to AAC in step 2.

   **Framerate / boundary alignment:** because titles come from a subtitle track (not `-r`-tied `drawtext`), transitions land on the exact ASS timestamp regardless of frame rate — libass rasterizes per output frame. A low fps (12) keeps encode fast and file small; a chapter boundary shows at the next rendered frame, i.e. within ≤ 1/12 s of the true boundary, which is imperceptible. This is a genuine advantage of the ASS decision over `drawtext` and is why fps can be low.

4. **Deadline:** copy merge's pattern but bump the floor — a full-novel encode is longer than an audio merge: `deadline = time.monotonic() + max(3600, int(total))`. Note this in a comment.

5. **Popen + poll loop + stderr-to-file + `_terminate` on cancel/deadline** — copy verbatim from `merge_chapters`, including `FileNotFoundError → TtsError("Không tìm thấy ffmpeg…")` and the non-zero-exit `err.txt` tail (`[-300:]`). Two ffmpeg invocations (audio concat, then render) each need the poll/cancel wrapper; factor a small internal `_run_ffmpeg(cmd, err_file, cancelled, deadline)` helper so both share the poll logic. The `finally` unlinks all temp files and rmdirs (mirror merge).

## 3. Font bundling, path, and licensing

**Recommended font: Noto Sans (SIL Open Font License 1.1)** — modern, well-hinted, unambiguous to bundle; ship `OFL.txt` alongside the `.ttf`. (DejaVu Sans is a permissive alternative.) Both fully cover Vietnamese Latin + tone marks.

Placement (mirroring `translators/data/`):
```
src/noveltrans/tts/assets/NotoSans-Regular.ttf
src/noveltrans/tts/assets/OFL.txt
```

Resolve the absolute path at runtime via `importlib.resources.as_file` (in a frozen app the resource may live inside the archive; ffmpeg needs a real filesystem path):

```python
from importlib import resources
def font_dir_context():
    return resources.as_file(resources.files("noveltrans.tts").joinpath("assets"))
# used as: with font_dir_context() as font_dir: render_video(..., font_dir=font_dir)
```

ASS reference: set the style `Fontname` to the font's **internal family name** ("Noto Sans"), and pass `fontsdir={font_dir}` to the `subtitles` filter so libass finds the bundled file without relying on system fontconfig. Do not rely on `/Library/Fonts/Arial Unicode.ttf` — macOS-only and absent in the packaged `.app` on other machines.

**Packaging changes:**
- `pyproject.toml` → extend `[tool.setuptools.package-data]`:
  ```toml
  noveltrans = ["translators/data/*.json", "tts/assets/*.ttf", "tts/assets/*.txt"]
  ```
- `packaging/NovelTrans.spec` → `datas = collect_data_files("noveltrans")` already collects declared package-data, so the font is picked up once it is package-data. Verify; if `collect_data_files` misses it, add an explicit `datas += [("src/noveltrans/tts/assets/NotoSans-Regular.ttf", "noveltrans/tts/assets")]`.

**CJK caveat (flag explicitly):** Noto Sans / DejaVu do NOT include CJK glyphs. When `audio_source == "original"` and the source is Chinese, `chapter_marker_title` returns the Chinese title, which would render as tofu boxes. The dominant, default path is Vietnamese translated titles (fully covered). Options, in order of recommendation:
1. Accept the limitation and document it (translated titles are the intended video use).
2. Additionally bundle **Noto Sans CJK** (~10–16 MB per weight) and set it as the style font when any window's titles contain CJK — detect via a Unicode range check in the builder. Robust but adds size.
This is the single biggest correctness/portability risk and should be surfaced to the user before implementation.

## 9. Testing strategy (ffmpeg-free, mirrors `test_merge.py`)

Pure tests (no ffmpeg, always run):
- **`_escape_ass`**: braces `{}` neutralized; `\n`/`\r\n` → `\N`; comma preserved; CJK+Vietnamese passthrough; trailing backslash handled.

- **`_ass_time`**: `0 → 0:00:00.00`, `12.0 → 0:00:12.00`, `20.5 → 0:00:20.50`, an hour-plus value.
- **`VideoWorker` field-shadowing regression**: assert `callable(w.start)` and `w.start_num/end_num` set (the `self.start` trap is real).
- **argv construction via mocked subprocess**: patch `subprocess.Popen`, assert `render_video` builds the expected argv — `-loop 1`, `-shortest`, `-tune stillimage`, `subtitles=`/`fontsdir=`, `-map "[v]"`, image + out paths present — non-zero return raises `TtsError`, early `cancelled()` raises `MergeCancelled`. Tests the whole pipeline without ffmpeg.

ffmpeg-gated (`@pytest.mark.skipif(shutil.which("ffmpeg") is None)`), mirroring `TestRealMerge`:
- Generate two short `sine` WAVs (reuse the `_tone` helper), a tiny solid-color PNG (via `ffmpeg lavfi color`), pass the bundled font dir, run `render_video`, assert the MP4 exists, is non-empty, and `ffprobe` reports one video + one audio stream and a duration ≈ sum of segment durations (validates `-shortest`).

## 10. Risks (flagged)

- **CJK glyphs in original-language titles** (highest risk): the bundled Vietnamese font renders Chinese titles as tofu. Document the limitation (translated-title videos are the intended path) or additionally bundle Noto Sans CJK (large). Decide before implementation — the one genuinely open product question.
- **Font family-name / fontsdir mismatch:** if the ASS `Fontname` doesn't exactly match the TTF's internal family name, libass silently falls back and tone marks may render wrong. Verify the exact family string and assert it in the ASS-builder test. Confirm `fontsdir` works in the frozen app (resource extracted to a real path via `as_file`).
- **Encode time / file size** (decided to surface, not solve): a full-novel video is hours/multi-GB; the confirm dialog and status must warn, and batch is steered as default. `-shortest` + `-tune stillimage` keep the video stream tiny (size dominated by AAC audio ≈ 0.7 MB/min), but wall-clock encode scales with total audio duration.
- **ASS path/escaping in the `subtitles` filter:** the temp `.ass` path is fed into a filter string where `:`/`\` are metacharacters. The temp dir is app-controlled (known prefix, no user input) so it's safe in practice; use forward slashes / a plain prefix.
- **Framerate/boundary alignment:** low fps means a title can appear up to `1/fps` s late; at fps=12 that's ≤ 83 ms — imperceptible, and no drift because titles are subtitle-timed. Raise fps if ever needed.
- **Two-stage vs inline audio:** concatenating audio to a temp `.m4a` first makes `-shortest`/duration exact and isolates codec issues; cost is transient disk for one full-novel audio file. Acceptable.
