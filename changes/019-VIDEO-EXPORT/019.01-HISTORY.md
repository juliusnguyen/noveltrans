## [DECISIONS] Settled before planning — 2026-07-18
1. **On-screen:** novel title persistent + current chapter title changing per chapter,
   with a subtle dark scrim for legibility over the image.
2. **Splitting:** mirror the existing merge modes — all / range / batch (batch = one
   video per N chapters, the sane default for long novels).
3. **Resolution:** 1920×1080 (16:9), background fitted with a blurred fill so any aspect
   works without black bars.
4. **Overlay technique (my call):** burn an **ASS subtitle track** (libass), not
   per-chapter `drawtext` — scales to hundreds of chapters.

### Verified environment facts
- This ffmpeg has `subtitles`+`ass` filters, **libass + libfreetype + libx264 compiled
  in** — ASS burn + H.264 encode both work.
- `/Library/Fonts/Arial Unicode.ttf` covers Vietnamese tone marks AND CJK. But for the
  **packaged .app** a system font isn't guaranteed → plan should **bundle** a
  redistributable Vietnamese-capable font (Noto/DejaVu) under package-data + the
  PyInstaller spec, referenced by absolute path in the ASS call. Font rendering of tone
  marks is the main correctness/portability risk.

### The feature is a close cousin of `tts/merge.py` — reuse:
- `plan_merge_windows` (all/range/batch selection) — verbatim.
- `MergeSegment(path, seconds, title)`, `build_concat_list`, and the cumulative-ms timing
  from `build_chapter_metadata` (same math the subtitle timing needs).
- `merge_chapters`' ffmpeg **Popen + poll-for-cancel + stderr-to-file + deadline** pattern.
- `MergeWorker` → template for `VideoWorker`; the merge QGroupBox in `tab_audio` →
  template for a "Xuất video" box.

## [PLAN] 019.01-INITIAL-PLAN — 2026-07-18

### [DECISION] CJK titles: Vietnamese font only, document the limit
User chose to bundle just a Vietnamese font (small) rather than also bundling Noto Sans
CJK. Videos of translated (Vietnamese) titles render perfectly — the intended use. A
video from the *original* Chinese audio would show chapter titles as tofu; documented as
a known limit. Reversible later by adding a CJK font + a Unicode-range font pick.

### [DE-RISK] Font acquired + Vietnamese rendering PROVEN before implementation
The #1 risk was whether libass renders Vietnamese tone marks with a bundled font. Fully
de-risked with a real render:
- Fetched the official OFL **Noto Sans** variable font (google/fonts), verified it covers
  the Vietnamese tone-mark codepoints (ộ ữ đ ằ ệ ở), and **instantiated a static Regular**
  (wght=400) via `fontTools.varLib.instancer` → `src/noveltrans/tts/assets/NotoSans-
  Regular.ttf` (646 KB, family **"Noto Sans"** — this is the ASS `Fontname`). Static, not
  variable, for predictable libass behaviour. Shipped `OFL.txt` alongside.
- Ran a real `ffmpeg -vf "subtitles=t.ass:fontsdir=fonts"` burn of a Vietnamese chapter
  title onto a frame: **rendered correctly** — "Chương 1: Diệp Vân mở khóa bí mật ộ ữ đ
  ằ ệ ở", white on the `BorderStyle=4`/`BackColour=&H80000000` scrim, bottom-center. So
  the font + fontsdir + scrim + alignment all work end to end.
- Placed `NotoSans-Regular.ttf` + `OFL.txt` under `src/noveltrans/tts/assets/`.

## [NEW REQUIREMENT] YouTube description with clickable chapter timestamps — 2026-07-18
User added: alongside the video, generate a **YouTube description** where each chapter is
a clickable timestamp (`0:00`, `2:05`, …) so viewers can jump to a chapter. Same cumulative
timing as the subtitle → a second pure builder. YouTube makes chapters only when the first
timestamp is `0:00` and they ascend (≥3), so the first chapter is always `0:00`.

## [STEP 1] `tts/video.py` — pure builders + render_video + tests — 2026-07-18
Plan 019.01 Step 1, extended with the description builder. Added `src/noveltrans/tts/
video.py` and `tests/test_video.py`.

- **Reuses `merge.py` verbatim** (imported, not copied): `plan_merge_windows`,
  `MergeSegment`, `build_concat_list`, `chapter_marker_title`, `MergeCancelled`,
  `_terminate`.
- **`build_ass_subtitles`** — ASS doc: a Novel-style event spanning the whole video +
  one Chapter event per segment, cumulative-timed (same math as `build_chapter_metadata`).
  `BorderStyle=4` + translucent `BackColour` = the legibility scrim; Novel top-center,
  Chapter bottom-center. `Fontname` = "Noto Sans" (the bundled family).
- **`_escape_ass`** — the one fiddly bit: `{`/`}` → `()` (ASS override markers),
  newlines → `\N`, trailing backslash stripped; commas preserved (Text is the final field).
- **`build_youtube_description`** — the new requirement: novel-title header + one
  `M:SS <title>` line per chapter, first always `0:00`.
- **`_ass_time` / `_yt_timestamp`** — the two timestamp formats.
- **`render_video`** — two-stage ffmpeg (concat audio → AAC, then loop image + blurred-fill
  filtergraph + burn ASS + mux audio + `-shortest`), copying merge's Popen/poll/cancel/
  deadline pattern via a factored `_run_ffmpeg`. Writes the `.txt` description next to the
  `.mp4`. `font_dir_context()` resolves the bundled font via `importlib.resources.as_file`
  (works in the frozen app).
