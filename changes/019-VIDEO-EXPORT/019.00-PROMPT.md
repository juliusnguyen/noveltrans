# 019 — Video export from rendered audio

Generate a **video** from the per-chapter audio the app already produces (tab 4), with:

1. **Chapter titles that appear on screen when each chapter starts** — the current
   chapter's name shown as an overlay/title card, synced to that chapter's audio.
2. **The audio as the soundtrack** — the rendered chapter audio plays over the video.
3. **A user-chosen background image** behind everything.

Output: a single video file (e.g. MP4) that plays the whole novel (or a selected range)
as an audio-with-titles video — the kind of thing uploaded to YouTube.

## Context (what already exists)

- Each chapter has `audio_path`, `audio_seconds` (duration), `audio_voice`,
  `audio_source` in the DB (`models.py`). So chapter start times = cumulative sum of
  `audio_seconds`.
- `src/noveltrans/tts/merge.py` already concatenates per-chapter audio via **ffmpeg**
  (`merge_chapters`, `build_concat_list`) and builds **M4B chapter markers** from the
  durations — the same timing math the video needs. `MergeWorker` runs it off-thread
  with progress + cancel. The pure metadata/selection builders are already unit-tested.
- `src/noveltrans/tts/convert.py` has `ffmpeg_available()` and the ffmpeg-subprocess
  pattern. ffmpeg is the obvious tool for this whole feature.
- The Audio tab (`gui/tab_audio.py`) has the "Ghép audio" (merge) box with modes
  all/range/batch — the video export likely sits alongside it.

## Requirements

- Loop/hold a **background image** as the video, mux the concatenated chapter **audio**,
  and burn in **chapter titles** timed to each chapter's start/duration.
- Reuse the existing duration/selection/concat machinery from `merge.py` where possible.
- Runs off-thread with progress + cancel, like the merge and audio workers.

## Open questions for /plan

- **Title overlay technique:** ffmpeg `drawtext` with `enable='between(t,a,b)'` per
  chapter (unwieldy for 199 chapters, and needs a font file) vs. a generated **subtitle
  track (SRT/ASS)** burned in with the `subtitles` filter (scales to any chapter count,
  ASS gives positioning/styling). Recommend one.
- **Vietnamese/CJK text in the overlay:** ffmpeg text rendering needs a font with the
  right glyphs. Which font — a bundled one, or a system font (macOS has Vietnamese-capable
  fonts)? This is the main correctness risk.
- **Resolution / aspect:** 1920×1080 default? How is the background image scaled/padded
  (fit, fill/crop, letterbox)?
- **Title card style:** just the chapter title text, or title + maybe chapter number /
  novel title? Position, size, background scrim for legibility over the image?
- **Scope of chapters:** all / range / batch, mirroring the merge modes? A full novel
  is *hours* long → a multi-GB MP4 and a long encode. Note the size/time cost; maybe
  default to a range.
- **Output format & codec:** MP4 (H.264 + AAC)? Since it's a static image, the video
  stream compresses to almost nothing — most of the size is audio.
- **Where in the GUI:** a new box/button in the Audio tab next to "Ghép audio"?
- **ffmpeg gating:** the whole feature needs ffmpeg (disable/hide if absent, like the
  MP3 and speed controls).
- **Testability:** keep the ffmpeg command + the subtitle/timing builders as pure,
  unit-testable functions (no ffmpeg in tests), following `merge.py`'s split.
