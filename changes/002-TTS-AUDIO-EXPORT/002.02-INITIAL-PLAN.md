# Implementation Plan — 002 TTS Audio Export (VieNeu-TTS)

## Research findings (VieNeu-TTS)

- **Package**: `pip install vieneu` (PyPI, v3.0.11, Python >=3.10, Apache 2.0). Base install is **torch-free** — runs v3 Turbo on ONNX Runtime; `vieneu[gpu]` adds PyTorch (not needed here).
- **Apple Silicon**: README explicitly says the torch-free ONNX CPU path is *faster* than MPS/PyTorch on macOS. Perfect fit for the M4; claimed "real-time CPU inference". No CUDA needed.
- **API**: `from vieneu import Vieneu; tts = Vieneu(); audio = tts.infer(text, voice=..., ref_audio=...); tts.save(audio, "out.wav")`; `tts.list_preset_voices()` returns (label, voice_id) pairs. Preset Vietnamese voices: Ngọc Lan (default), Xuân Vĩnh, Ngọc Linh. Also a `mode='remote'` HTTP-client mode (ignore for v1).
- **Model**: auto-downloads from HF `pnnbao-ump/VieNeu-TTS-v3-Turbo`, **~334 MB** (262 MB safetensors + 2 ONNX files) into `~/.cache/huggingface`. First run blocks for the download.
- **Output**: 48 kHz WAV. No documented max input length → we must chunk chapters ourselves (sentence/paragraph-level) and concatenate.
- **Memory**: ONNX + ~330 MB weights ≈ well under 2 GB RSS — safe in-process alongside LM Studio on 24 GB.

## Key decisions

1. **Integration shape: dedicated Tab "Audio"**, mirroring `tab_translate.py`. TTS of a novel takes hours → needs per-chapter resumability, progress, cancel, per-row re-generate. The exporter contract (one file, no progress signal) doesn't fit. Do NOT add to the exporter registry; instead create a parallel `tts/` engine package (mirrors `translators/` registry so a future engine, e.g. remote server, slots in).
2. **In-process QThread** (like `TranslateWorker`), with `vieneu` lazily imported inside `run()`. Torch-free ONNX makes memory pressure a non-issue; subprocess isolation is deferred as a mitigation if instability appears.
3. **Per-chapter files** in `project.exports_dir / "audio"`, named `0001-<slug(translated_title)>.wav` (or `.mp3` when ffmpeg exists). DB stores the path **relative to the project folder**.
4. **DB**: new columns via existing `_migrate()` ALTER TABLE pattern: `audio_path`, `audio_voice`, `audio_seconds`, `audio_error` (separate from `error`/`status`, which belong to the download/translate lifecycle — audio is a parallel pipeline; audio state is derived from `audio_path` + file existence).
5. **Settings in-tab** (voice combo, format combo) persisted via `AppConfig` (`tts_voice`, `tts_format`) — same pattern as engine/model/URL pickers living in TranslateTab.
6. **Dependency**: optional extra `[project.optional-dependencies] tts = ["vieneu>=3.0"]`. Missing package → friendly `TtsError` ("cài: pip install 'noveltrans[tts]'") emitted via the worker's `failed` signal.

## Milestones

### M1 — TTS engine layer (no GUI)

**New files**
- `src/noveltrans/tts/__init__.py` — registry, mirroring `translators/__init__.py`:
  ```python
  def get_tts_engine(name: str, *, voice: str = "") -> TtsEngine  # lazy import; ImportError -> TtsError with install hint
  TTS_ENGINE_NAMES = {"vieneu": "VieNeu-TTS (local)"}
  ```
- `src/noveltrans/tts/base.py`:
  ```python
  def split_sentences(text: str, max_chars: int = 400) -> list[str]
      # paragraph split on "\n\n" (reuse translators.base.split_paragraph_chunks idea),
      # then sentence split on [.!?…] keeping delimiters; oversized sentence = own chunk

  class TtsEngine(ABC):
      name/display_name: str
      max_chunk_chars: int = 400
      sample_rate: int
      @abstractmethod def load(self) -> None                      # model download/load, called once by worker
      @abstractmethod def list_voices(self) -> list[tuple[str, str]]
      @abstractmethod def synthesize(self, text: str) -> "np.ndarray"   # one chunk
      @abstractmethod def save_wav(self, samples, out_path: Path) -> None
      def synthesize_chapter(self, title: str, body: str, out_path: Path,
                             cancelled=None) -> float:
          # chunk -> synthesize each (checking cancelled() between chunks)
          # -> np.concatenate (short silence gap between paragraphs)
          # -> save_wav; returns audio duration seconds (len(samples)/sample_rate)
  ```
- `src/noveltrans/tts/vieneu.py` — `VieneuEngine(TtsEngine)`: wraps `Vieneu()` (constructed in `load()`), `infer(text, voice=self.voice or None)`, `list_preset_voices()`, `save()`. Verify at implementation time whether `infer` returns a raw numpy array or an object with `.samples`; normalize inside the engine. Fallback voices constant `["Ngọc Lan", "Xuân Vĩnh", "Ngọc Linh"]` for when the model isn't loaded yet.

**Modify**
- `src/noveltrans/errors.py` — add `class TtsError(NovelTransError)`.
- `pyproject.toml` — add `tts` optional extra.

**Tests** (`tests/test_tts.py`) — all offline: `split_sentences` edge cases (empty, oversized sentence, order-preserving rejoin); `FakeTtsEngine(TtsEngine)` exercising `synthesize_chapter` (chunk count, wav written via fake save, duration math); `get_tts_engine` unknown-name raises; missing `vieneu` package raises `TtsError` (patch `sys.modules`); `VieneuEngine` behavior with a mocked `vieneu.Vieneu` injected via `sys.modules` (vieneu is NOT a dev dependency). One `@pytest.mark.live` test: real `Vieneu()` synthesizes one short sentence to `tmp_path` (excluded by default `addopts = "-m 'not live'"`).

### M2 — Storage: migration + queries

**Modify `src/noveltrans/storage/project.py`**
- `_SCHEMA`: add `audio_path TEXT NOT NULL DEFAULT ''`, `audio_voice TEXT NOT NULL DEFAULT ''`, `audio_seconds REAL NOT NULL DEFAULT 0`, `audio_error TEXT NOT NULL DEFAULT ''`; add the same four to the `added` dict in `_migrate()`.
- New methods (follow existing style):
  ```python
  AUDIO_DIR = "audio"  # exports/audio
  @property def audio_dir(self) -> Path: return self.exports_dir / AUDIO_DIR
  def pending_audio(self) -> list[Chapter]      # translated != '' AND audio_path = ''
  def save_audio(self, idx: int, rel_path: str, voice: str, seconds: float) -> None   # clears audio_error
  def mark_audio_error(self, idx: int, message: str) -> None
  def clear_audio(self) -> None                 # reset all audio columns ("Tạo lại từ đầu")
  ```
- `counts()` gains `"audio"` count (`audio_path != ''`).

**Modify `src/noveltrans/models.py`** — `Chapter` gains `audio_path: str = ""`, `audio_voice: str = ""`, `audio_seconds: float = 0.0`, `audio_error: str = ""`; `_row_to_chapter` in project.py maps them. Add `@property has_audio`.

**Tests** — extend `tests/test_storage.py`: `_migrate` upgrades a DB created without audio columns (create raw sqlite with old schema, open NovelProject, assert columns); `pending_audio` excludes chapters with audio; `save_audio`/`mark_audio_error`/`clear_audio` round-trips; counts.

### M3 — AudioWorker

**Modify `src/noveltrans/gui/workers.py`** — `class AudioWorker(QThread)` modeled exactly on `TranslateWorker`:
```python
progress = Signal(int, int, str)      # done, total, chapter title ("" phases: "Đang tải model VieNeu…")
chapter_done = Signal(int)
chapter_error = Signal(int, str)
failed = Signal(str)                  # engine could not be constructed/loaded (incl. missing vieneu)
finished_ok = Signal(int, int)        # ok count, error count

def __init__(self, project_path: Path, voice: str, out_format: str = "wav",
             indices: list[int] | None = None, parent=None)
def cancel(self) -> None              # checked between chapters AND passed into synthesize_chapter
                                      # (engine checks a callback between chunks — chapters take minutes)
```
`run()`: `get_tts_engine("vieneu", voice=voice)` → `engine.load()` (emit progress with the loading message first — covers the 334 MB first-run download) → open own `NovelProject` → iterate `pending_audio()` or `indices` → `project.audio_dir.mkdir(parents=True, exist_ok=True)` → filename `f"{idx+1:04d}-{slugify(translated_title or title)}.{ext}"` → `synthesize_chapter` timed with `time.monotonic()` → optional ffmpeg wav→mp3 (M5) → `save_audio(idx, rel_path, voice, seconds)`. Per-chapter try/except keeps the batch going (`mark_audio_error`).

Also add `class TtsVoicesWorker(QThread)` (mirrors `LmStudioModelsWorker`): emits `voices_listed = Signal(list)`; returns preset voice labels without loading the full model if possible, else the static fallback list.

**Tests** — patch `noveltrans.tts.get_tts_engine` to return a fake engine and drive `AudioWorker.run()` directly (call `run()` synchronously, no thread needed); assert files written, DB rows updated, resumability (second run synthesizes 0), cancel stops early, missing-package path emits `failed`.

### M5 — MP3 conversion + polish

- `src/noveltrans/tts/base.py` (or a small `tts/convert.py`): `def convert_to_mp3(wav_path: Path, mp3_path: Path) -> Path` using `subprocess.run(["ffmpeg", "-y", "-i", wav, "-b:a", "96k", mp3])`; delete the intermediate wav on success; raise `TtsError` with stderr tail on failure. Worker calls it when `out_format == "mp3"`. 48 kHz WAV is ~5.8 MB/min → a 30-min chapter is ~170 MB, so MP3 matters for whole novels; surface a size hint in the tab when WAV is selected.
- Tests: mock `subprocess.run`; ffmpeg-missing → format falls back to wav with a status note.

### M6 (stretch — defer unless asked) — single-file audiobook

"Ghép thành audiobook (.m4b)" button: ffmpeg concat demuxer + `FFMETADATA` chapter marks from `audio_seconds`, output `exports/<slug>.m4b`. Pure ffmpeg, no new Python deps. Ship M1–M5 first.

## Verification checklist

1. `.venv/bin/python -m pytest` — all existing 102 + new tests pass offline (no vieneu installed).
2. `.venv/bin/ruff check src tests` clean (line-length 100).
3. `pip install -e '.[tts]'` in the venv (Python 3.12 OK, vieneu supports 3.10–3.13); confirm no torch pulled in.
4. `pytest -m live tests/test_tts.py` — one real sentence synthesized (verifies model download ~334 MB + Apple Silicon ONNX path + actual `infer` return type).
5. Launch app → Tab 4 → pick a novel with translated chapters → generate 1 chapter via per-row button → file appears in `exports/audio/`, row shows Đã tạo + duration, double-click plays.
6. Cancel mid-run → resumes correctly ("Tạo audio tất cả" skips done chapters). Restart app → state persists (DB).

## Risks & mitigations

- **First-run model download (~334 MB)** blocks inside the worker with no HF progress hook → emit explicit "Đang tải model VieNeu (~330 MB, lần đầu)…" status; document in README.
- **`infer()` return type / exact constructor kwargs** may differ from README (docs partly inferred) → isolate all vieneu-touching code in `tts/vieneu.py`; the live test in M1 validates the real API before GUI work builds on it.
- **Speed** (~real-time CPU): a 2-hour-audio novel ≈ hours of synthesis → per-chapter resumability + mid-chapter cancel are core, not optional; consider v3 Turbo batching later as an optimization.
- **Hidden system deps** (phonemizer/espeak-style): sea-g2p appears bundled, but verify in step 4 before committing; if a system dep exists, add a preflight check with a Vietnamese error message.
- **Disk usage of 48 kHz WAV** → MP3 via ffmpeg (M5) + size hint in UI.
- **Memory with LM Studio loaded**: ONNX path is small; if problems surface, the `TtsEngine` abstraction allows a subprocess or `mode='remote'` engine without touching the tab/worker.
