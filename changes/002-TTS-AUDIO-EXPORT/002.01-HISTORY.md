## [FIX] Voice change didn't regenerate existing audio — 2026-07-05
User changed the voice combo but "Tạo audio tất cả" said "Đã đủ": `pending_audio()`
only checked for missing audio_path, ignoring the voice. Verified engine-level voice
switching works (Ngọc Lan vs Gia Bảo: different lengths, correlation ≈ 0) — the bug
was the queue query. `pending_audio(voice)` now re-pends chapters whose audio_voice
differs (same pattern as pending_translation's language switch); the worker also
deletes the stale old file when the new path differs (e.g. wav→mp3 format change).
E2E on the copied project: chapter 1 re-voiced Ngọc Lan→Gia Bảo as mp3, old 42 MB
wav cleaned up. 134 tests pass.

## [INVESTIGATION] "Voices sound alike" — 2026-07-05
Verified NOT an app bug, three layers deep: (1) each preset is a distinct speaker in
the model (unique reserved_id + own codes in vieneu's voices_v3_turbo.json; wrong
names raise ValueError, no silent fallback), (2) male↔female clearly differ (Gia Bảo
~110 Hz vs Ngọc Lan ~220 Hz), (3) but same-gender presets measure nearly identical
pitch (nữ 210–224 Hz; nam 113–126 Hz except Thái Sơn 148 Hz) — a model
characteristic. Sent A/B comparison mp3s. Offered ref_audio voice cloning as the
real differentiator (pending user decision).
