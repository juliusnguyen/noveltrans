# 009.02 — Initial Plan: audio from original text (not only translation)

## Goal

Let the Audio tab generate TTS from a chapter's **original text** (`chapter.content`)
as well as its **translation** (`chapter.translated`), via a "Bản dịch / Bản gốc"
toggle. This lets Vietnamese sources (medoctruyen, giatocvuongtai — `source_lang="vi"`)
be voiced directly without a translate step.

## Design in one line

Add a source toggle in the Audio tab; thread `use_translation: bool` through
`AudioWorker` (which text/title it synthesizes), `pending_audio` (which column gates
"pending"), and make the table/status/messages source-aware. Track the audio's source
in the DB so switching sources re-pends correctly.

## Decisions (resolving the prompt's open questions)

1. **Track the audio source in the DB (recommended over plain overwrite).** The
   `_migrate()` helper (project.py:103) already adds columns idempotently, so a new
   `audio_source TEXT NOT NULL DEFAULT 'translated'` column costs ~one line. The
   `DEFAULT 'translated'` back-fills existing audio correctly (all current audio *is*
   from the translation). With it, `pending_audio` can re-pend a chapter whose existing
   audio came from the *other* source, so flipping Bản dịch ↔ Bản gốc "just works"
   (regenerates) instead of showing a confusing "Đã đủ".
   - *Lighter alternative (rejected):* no column; the toggle only scopes a run and the
     user must click "Tạo lại từ đầu" to switch an existing project's audio. Cheaper
     but confusing on switch. The migration is trivial enough that tracking is worth it.
2. **Non-Vietnamese originals: allow but warn.** VieNeu is a Vietnamese TTS; a Chinese
   original (ixdzs/xbanxia, `source_lang="zh"`) would voice as gibberish. When the user
   picks Bản gốc and `meta.source_lang != "vi"`, show a one-time `QMessageBox.warning`
   ("Giọng đọc là tiếng Việt; bản gốc không phải tiếng Việt nên audio có thể sai") and
   let them proceed. Don't hard-block — the user may have a reason.
3. **Default = Bản dịch.** Preserves today's behaviour; no surprise for existing users.

## 1. Storage — `src/noveltrans/storage/project.py`

- **Schema + migration:** add `audio_source TEXT NOT NULL DEFAULT 'translated'` to
  `_SCHEMA` (chapters table) and to the `added` dict in `_migrate()`. Values:
  `'translated'` | `'original'`.
- **`_row_to_chapter`** + the `Chapter` model (`models.py`): add `audio_source: str = ""`
  so it round-trips (mirrors `audio_voice`). (Default `""` on the dataclass; the DB
  column defaults to `'translated'` for back-fill — new rows written by save_audio set
  it explicitly.)
- **`pending_audio(voice="", use_translation=True)`:**
  ```sql
  SELECT * FROM chapters
  WHERE {src_col} != ''
    AND (audio_path = '' OR (? != '' AND audio_voice != ?) OR audio_source != ?)
  ORDER BY idx
  ```
  where `src_col = "translated" if use_translation else "content"` and the third `?` is
  `"translated" if use_translation else "original"`. So a chapter is pending if it has
  the requested source text AND (no audio yet, OR the audio was a different voice, OR
  the audio was made from the *other* source).
- **`save_audio(idx, rel_path, voice, seconds, source="translated")`:** persist
  `audio_source = source` alongside voice/seconds. One-line addition to the UPDATE.

## 2. Worker — `src/noveltrans/gui/workers.py` (`AudioWorker`)

- Add `use_translation: bool = True` to `__init__` (store it). Update the docstring
  ("translated **or original** chapters").
- Text/title selection per chapter:
  ```python
  if self.use_translation:
      title, text = (chapter.translated_title or chapter.title), chapter.translated
  else:
      title, text = chapter.title, chapter.content
  ```
  Use `title`/`text` in `synthesize_chapter(title, text, out_path, ...)` and the
  filename slug.
- Pending selection:
  - `indices` path: filter `c is not None and (c.translated if use_translation else c.content)`.
  - `None` path: `project.pending_audio(self.voice, self.use_translation)`.
- `save_audio(..., source="translated" if self.use_translation else "original")`.

## 3. Table model — `src/noveltrans/gui/widgets.py` (`AudioChapterTableModel`)

`_audio_status` currently returns "Chưa dịch" when `not chapter.translated`. In original
mode that's wrong (a downloaded-but-untranslated chapter is voiceable). Make the model
source-aware:
- Add `self._use_translation = True` and a `set_source(use_translation)` that
  `beginResetModel()/endResetModel()`s (or emits `dataChanged`) so the status column
  refreshes.
- In `_audio_status`: the "no source text" branch checks
  `chapter.translated` (translation mode) vs `chapter.content` (original mode), with the
  label "Chưa dịch" vs "Chưa tải". Rest (Lỗi / Đã tạo / Chưa tạo) unchanged.

## 4. UI — `src/noveltrans/gui/tab_audio.py`

- **Toggle:** add a "Nguồn đọc" control to `top_row` (or a small row under it) —
  two `QRadioButton`s "Bản dịch" (default checked) / "Bản gốc", grouped, mirroring the
  Export tab. Store as `self.translated_radio` / `self.original_radio`.
- On toggle (`toggled` signal): call `self.model.set_source(use_translation)` and
  refresh the status line; if switching to Bản gốc and `self.project` and
  `meta.source_lang != "vi"`, warn once (guard a `self._warned_source` flag so it
  doesn't nag on every toggle).
- **`_start_generate`:** read `use_translation = self.translated_radio.isChecked()`.
  - Empty-check: translation mode keeps `counts["translated"] == 0 → "Chưa có bản dịch"`;
    original mode checks `counts["downloaded"] == 0 → "Chưa tải chương nào (Tab 1)"`.
  - `total = len(self.project.pending_audio(voice, use_translation))`.
  - Pass `use_translation=use_translation` to `AudioWorker(...)`.
- **`_regenerate_row`:** the `not chapter.translated` guard becomes
  `not (chapter.translated if use_translation else chapter.content)`.
- **`_on_project_selected` status line:** show `translated` or `downloaded` count based
  on the current toggle (or show both — keep simple: reflect the active source).
- **Persist** the choice (optional): a `config.tts_use_translation` bool, like
  `tts_voice`/`tts_format`. Recommend adding it for consistency; low cost.

## 5. Tests

- **Storage:** `pending_audio(use_translation=False)` returns chapters with `content`
  but no `translated`; `save_audio(..., source="original")` round-trips `audio_source`;
  a chapter with `audio_source='translated'` is **re-pending** when requesting original
  (and vice-versa); migration adds the column with the `'translated'` back-fill.
- **Worker-free logic** where possible: the text/title selection can be unit-tested by
  extracting a tiny helper, or asserted via `pending_audio`.
- **Model:** `set_source(False)` makes an untranslated-but-downloaded chapter report
  "Chưa tải"/voiceable rather than "Chưa dịch". (offscreen `qapp`.)
- Regression: default (translation) path unchanged — existing audio tests still pass.

## 6. Verification

1. `make test` green, `make lint` clean.
2. Manual (a Vietnamese source, e.g. giatocvuongtai): download a couple chapters, do
   NOT translate, Audio tab → "Bản gốc" → generate → confirm audio is produced from the
   Vietnamese original and plays. Toggle back to "Bản dịch" with no translation →
   "Chưa có bản dịch". Switch an already-voiced project's source → confirm it re-pends
   and regenerates (audio_source guard).
3. Non-vi source (ixdzs) → picking Bản gốc warns.

## Risks / notes

- **Blast radius is contained**: `use_translation` defaults True everywhere, so the
  translated flow is byte-for-byte unchanged unless the user flips the toggle.
- **`audio_source` default `'translated'`** is the load-bearing migration choice — it
  keeps existing audio from being needlessly re-pended the first time the new query runs.
- **Filenames** already include the voice slug; they don't encode the source. That's
  fine — one audio file per chapter, replaced on regeneration; `audio_source` records
  which text it came from. (A future "keep both" would need source in the filename +
  schema, out of scope.)
- **VieNeu-only**: no engine change; the warning covers the language mismatch.
