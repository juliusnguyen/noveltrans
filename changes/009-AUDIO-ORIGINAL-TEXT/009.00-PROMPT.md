# 009 — Audio: support reading the original text (not only the translation)

## Request (vi)

> Phần record audio, thêm support cho bản gốc thay vì chỉ support cho bản dịch.

Add an option in the Audio (TTS) tab to generate audio from a chapter's **original
text** (`chapter.content`) as well as its **translation** (`chapter.translated`).
Today TTS only reads the translated text.

## Why it matters

The Vietnamese sources (medoctruyen.vn, giatocvuongtai.com) store `content` that is
**already Vietnamese** (`source_lang="vi"`). For those, a user can skip the translate
step entirely and just listen to the original — but the audio tab currently forces a
translation first. Supporting "bản gốc" removes that unnecessary step.

## Current state (verified)

- `gui/tab_audio.py`: only path is translated. `_start_generate` bails with "Chưa có
  bản dịch" if `counts["translated"] == 0`; the voice/format come from combos.
- `gui/workers.py` `AudioWorker`: synthesizes from `chapter.translated` (workers.py:360)
  and is docstring'd "Generate audio for translated chapters".
- `storage/project.py` `pending_audio(voice)`: filters `WHERE translated != '' AND
  (audio_path = '' OR voice mismatch)` — i.e. only translated chapters are ever pending.
- `save_audio(idx, rel_path, voice, seconds)` stores one audio file per chapter with
  its `audio_voice`; there is no notion of "which text the audio came from".
- TTS engine is **VieNeu-TTS (Vietnamese)** — so reading the original only makes sense
  when the original is Vietnamese (medoctruyen / giatocvuongtai). Chinese-source
  originals (ixdzs/xbanxia) won't voice sensibly with a Vietnamese TTS.

## Scope (to confirm during /plan)

- A UI toggle in the Audio tab, mirroring the Export tab's "Bản dịch / Bản gốc"
  radios — choose which text to voice.
- Thread a `use_translation` (or `source`) flag: `tab_audio` → `AudioWorker` → pick
  `chapter.translated` vs `chapter.content`; adjust the empty-checks
  (`translated` vs `content`/`downloaded`).
- `pending_audio` needs an "original" variant (filter on `content != ''`).
- Regenerate-row / regenerate-all and the `counts` messaging follow the chosen source.

## Open questions / risks

- **One audio file per chapter today.** If a chapter already has translated-audio and
  the user switches to original, does the original overwrite it? Options: (a) overwrite
  (simplest, one file per chapter — the toggle picks the source for this run), or
  (b) track the audio's source text so both can coexist. Recommend (a) for v1; note the
  DB/`save_audio` change (b) would need.
- **Voice/language:** VieNeu is Vietnamese. For Chinese originals the output is
  gibberish — should the "bản gốc" option be disabled/warned when `meta.source_lang`
  isn't `vi`? Recommend: allow but warn (or hide) for non-vi sources.
- **`pending_audio` semantics** when mixing sources across runs — keep it simple: the
  toggle scopes a generation run; audio_voice still tracks the voice.
