# 030 — Download from a specific chapter — History

## [FEATURE] 030-DOWNLOAD-FROM-CHAPTER — 2026-07-23
Branch `changes/030-DOWNLOAD-FROM-CHAPTER`. Prompt: `030.00-PROMPT.md`.

## [DECISIONS] From the user — 2026-07-23
- **UX**: keep the "download all" button, plus (1) right-click a chapter row →
  "Tải từ chương N" / "Chỉ tải lại chương N", and (2) a From/To range box.
- **Scope**: bulk actions (from-here, range) skip already-downloaded chapters;
  the single-chapter "Chỉ tải lại" force re-fetches even if it has content.

## [IMPL] — 2026-07-23
### Backend
- `storage/project.py`:
  - `pending_download(start_idx=0, end_idx=None)` — optional 0-based inclusive
    bounds (default = whole novel, so existing callers are unchanged).
  - new `chapters_in_range(start_idx, end_idx=None)` — all chapters in range
    regardless of status, for the forced re-download path.
- `gui/workers.py` — `DownloadWorker` gains keyword-only `start_index`,
  `end_index`, `force` (defaulted). `_select_chapters()` picks
  `chapters_in_range` when `force` else the bounded `pending_download`.

### UI (`gui/tab_scrape.py`)
- Refactored the download entry points: `_start_download` → `_begin_download(start,
  end, force)` (records scope) + `_launch_download()` (core, reads the stored
  scope). The button → `_download_all` (0, None, False); the **post-unlock resume
  timer now calls `_launch_download`**, so a resumed batch reuses the SAME scope
  it was interrupted with (not "all pending").
- Right-click menu: added via a new `extra_actions` hook on `enable_cell_copy`
  (`gui/widgets.py`) so copy + the two download actions share ONE context menu
  (a second `customContextMenuRequested` connection would pop two menus).
- From/To range box (`QSpinBox` ×2 + "Tải khoảng"); 1-based chapter numbers →
  0-based idx; reversed spans tolerated; bounds synced to the chapter count in
  `_reload_table` via `_sync_range_bounds`. `range_button` tracks `download_button`
  enable/disable across load / scan / download / daily-limit / finish.

### Tests
- `test_storage.py`: `pending_download` start-index + range bounds;
  `chapters_in_range` ignores status.
- `test_workers_adapter_close.py`: worker downloads from a start index; range
  skips already-downloaded; `force` overwrites a downloaded chapter and leaves
  neighbours; ordinary path unchanged.
- `test_tab_scrape.py`: range controls start disabled; loading a project sets
  spin-box bounds + enables the button; `_download_all` / `_download_range`
  (incl. reversed) / context-menu actions map to the right (start, end, force)
  scope.
- Full suite **663 passed**; ruff clean.
