# 030 — Download from a specific chapter

## Request

Today the Scrape/Download tab only offers "download all pending from the
beginning" (`Tải các chương` → `pending_download()`). Add the ability to start
downloading from a specific chapter, and to download a chapter range.

## Decisions (from the user)

**UX — two entry points, both keeping the existing "download all" button:**
1. **Right-click a chapter row** in the table:
   - `Tải từ chương này` — download from this chapter to the end.
   - `Chỉ tải chương này` — download only this one chapter.
2. **From/To range box** near the download row: `Từ [x] đến [y]` + a
   `Tải khoảng` button — download chapters x..y (inclusive, by displayed chapter
   number = idx + 1).

**Scope — how already-downloaded chapters in the selection are treated:**
- Bulk actions (from-this-chapter, From/To range) **skip already-downloaded**
  chapters (only fetch those still missing content) — resumable, no wasted AI
  re-runs. Same semantics as the existing button.
- `Chỉ tải chương này` (single) **force re-fetches** that one chapter, even if it
  already has content (a refresh, like the audio tab's per-row regenerate).

## Implementation notes / scope

- `storage/project.py` — `pending_download()` gains optional `start_idx` /
  `end_idx` bounds (0-based, inclusive). Default = today's behaviour.
- `gui/workers.py` — `DownloadWorker` gains `start_index` / `end_index` / `force`
  (keyword, defaulted so existing callers are unchanged). `force` fetches every
  chapter in range regardless of content; otherwise it uses the bounded
  `pending_download`.
- `gui/tab_scrape.py` — refactor `_start_download` so the button + post-unlock
  resume keep "all pending" semantics, add a right-click context menu on the
  table and the From/To range controls. The post-unlock **resume must reuse the
  same scope** as the interrupted download (store the chosen start/end/force).
- Numbers in the UI are 1-based (the `#` column shows `idx + 1`); convert to
  0-based idx for the backend. Validate/clamp the range inputs.
- Tests: `pending_download` bounds; `DownloadWorker` start/end/force selection
  (incl. force overwriting an already-downloaded chapter, and range skipping
  downloaded ones).

## Out of scope

  follow-up).
