# 010 — Audio: merge all chapters into one file

## Request (vi)

> Ở phần nghe audio, sau khi render toàn bộ chương, có cơ chế để merge toàn bộ chương
> thành 1 file audio duy nhất không?

Add a way, in the Audio tab, to merge every chapter's rendered audio into a **single
audio file** (a whole-novel audiobook), after the per-chapter files have been generated.

## Current state (verified)

- Per-chapter audio lives in `exports/audio/` (`project.audio_dir`), one WAV/MP3 per
  chapter. Each chapter row stores `audio_path` (relative), `audio_voice`,
  `audio_source`, and `audio_seconds` (duration) — so chapter offsets for markers are
  already known without probing files.
- `tts/convert.py` shells out to **ffmpeg** (`ffmpeg_available()`, `convert_to_mp3`).
  ffmpeg is the natural tool for concatenation + chapter metadata. `pydub` is also a
  dependency.
- Audio tab (`gui/tab_audio.py`) drives generation via `AudioWorker` (off the GUI
  thread); the table shows per-chapter audio status.

## Scope (to confirm during /plan)

- A **"Ghép thành 1 file"** button in the Audio tab, enabled once chapters have audio.
- Runs off-thread (a `MergeWorker`, like the other workers — ffmpeg can take a while
  over hundreds of chapters).
- Merge in **reading order** (by chapter index), including only chapters that have
  audio. Output written to `exports/` (e.g. `<novel-slug>.m4b` / `.mp3`).
- **Format (decision for /plan):** default to an **M4B audiobook with per-chapter
  markers** (each chapter a seekable bookmark, titles from the chapter titles, offsets
  from `audio_seconds`) — the right format for a long novel. Provide a plain
  **joined MP3** as the simpler alternative.
- Progress + open-file/open-folder on completion; needs ffmpeg (warn if missing, like
  the MP3 path does).

## Open questions

- **M4B-with-chapters vs joined MP3** — user leaned toward the app defaulting to M4B
  (chapter markers) with joined MP3 as fallback. Confirm.
- **Voice consistency:** merge only chapters sharing one voice, or whatever audio
  exists (a novel voiced in two voices would sound inconsistent)? Recommend: merge the
  chapters matching the tab's currently-selected voice; warn if some chapters are
  missing/other-voice.
- **Chapter titles for markers:** translated title vs original — follow the audio's
  `audio_source` (translated audio → translated title; original audio → original
  title).
- **Very long novels:** one file could be huge. v1 = a single file; optional
  split-every-N-chapters is a possible follow-up (note, don't build).
- **Format of the per-chapter inputs** (WAV vs MP3) — concat handles both; M4B encodes
  to AAC. Confirm ffmpeg concat approach (concat demuxer + `-f ffmetadata` for chapters).
