# 010.02 — Initial Plan: merge all chapters into one audio file

## Goal

After per-chapter audio is rendered, let the user merge it into **one or more files**,
choosing which chapters go together — an **M4B audiobook with per-chapter markers**
(default) or a plain **joined MP3**. Three selection modes:

- **Toàn bộ** — all voiced chapters → one file.
- **Từ chương X đến Y** — a chapter-number range → one file.
- **Theo lô N chương** — split into consecutive batches of N chapters → many files
  (`1–10`, `11–20`, …), each an M4B/MP3 with its own chapter markers.

Triggered from the Audio tab, run off-thread, needs ffmpeg.

## Design in one line

A new `tts/merge.py` builds an ffmpeg concat list + an ffmetadata chapter file (offsets
from the stored `audio_seconds`) and shells out to ffmpeg; a `MergeWorker` runs it off
the GUI thread; the Audio tab gets a "Ghép thành 1 file" button + a format choice.

## Decisions (resolving the prompt's open questions)

1. **Default M4B (chapters), MP3 as the alternative.** ffmpeg has the `aac` encoder
   (verified), so M4B = AAC audio in an `.m4b` container with `[CHAPTER]` metadata — each
   chapter a seekable bookmark titled from the chapter title. A whole-novel single track
   is far more usable this way. A "MP3 gộp" option produces one flat MP3 for players
   that don't grok M4B.
2. **Merge only chapters matching the selected voice.** A novel voiced in two voices
   would sound jarring mid-book. Include chapters where `audio_path != '' AND audio_voice
   == <the tab's current voice>`, in `index` order. If some chapters in the range have no
   audio (or another voice), **warn** with the count and let the user proceed (the merge
   just skips them) — don't silently drop or hard-block.
3. **Marker titles follow the audio's source.** Per chapter, if `audio_source ==
   'translated'` use `translated_title or title`, else `title` — so the bookmark label
   matches what was actually voiced.
4. **Selection modes (all / range / batch).** Ranges and batches use **chapter numbers**
   (1-based `index + 1`), not "the Kth chapter that happens to have audio" — so file
   boundaries are predictable (batch 2 is always chapters 11–20 regardless of gaps). A
   batch/range file includes whatever chapters in that number-window have audio in the
   selected voice; empty windows produce no file. This directly answers the "one huge
   file" worry — a 700-chapter novel can be `batch of 50` → 14 files.

## Output naming

- **Toàn bộ:** `exports/<slug>.m4b|.mp3`.
- **Range / each batch:** `exports/<slug>-<AAAA>-<BBBB>.m4b|.mp3` where AAAA/BBBB are the
  zero-padded first/last chapter *number* of the window (e.g. `mo-dau-0001-0010.m4b`).

## 1. Core — new `src/noveltrans/tts/merge.py`

`merge_chapters(project, chapters, out_path, fmt, cancelled=None, progress=None) -> Path`

- `chapters`: the ordered list to include (already filtered to have audio + matching
  voice by the caller); each carries `audio_path`, `audio_seconds`, `audio_source`,
  `title`, `translated_title`.
- Build a **concat list** temp file — lines `file '<abs path>'` (single-quotes escaped:
  `p.replace("'", "'\\''")`), absolute paths, run ffmpeg with `-safe 0`.
- Build an **ffmetadata** temp file for M4B: `;FFMETADATA1` header, then per chapter a
  `[CHAPTER]\nTIMEBASE=1/1000\nSTART=<ms>\nEND=<ms>\ntitle=<label>` block, where
  START/END are the cumulative `audio_seconds` (× 1000, integer ms). Chapter label per
  decision 3.
- **M4B command:**
  ```
  ffmpeg -y -f concat -safe 0 -i list.txt -i chapters.txt
         -map 0:a -map_metadata 1 -c:a aac -b:a 96k <out>.m4b
  ```
  (`-map 0:a` so the audio comes from the concat input, metadata from input 1.)
- **MP3 command:**
  ```
  ffmpeg -y -f concat -safe 0 -i list.txt -c:a libmp3lame -b:a 96k <out>.mp3
  ```
  (no chapters — flat file.)
- Reuse the error/timeout handling shape from `convert.py::convert_to_mp3` (FileNotFound
  → "cài ffmpeg", non-zero → stderr tail as `TtsError`). Longer timeout (a full novel
  re-encode is minutes, not seconds — e.g. no hard cap, or a generous one; poll
  `cancelled()` by running ffmpeg with `Popen` and terminating if cancelled — or accept
  no mid-run cancel for v1 and document it).
- Clean up temp files in `finally`.
- Verify the `aac` encoder exists once (like `ffmpeg_available()`); if not, raise a
  clear `TtsError` suggesting MP3 instead.

Helper `ffmpeg_has_encoder(name)` (small `ffmpeg -encoders` check) so the UI can hide
M4B when AAC isn't built in.

## 2. Selection → windows (pure helper, in `merge.py`)

`plan_merge_windows(chapters, voice, mode, *, start=None, end=None, batch=None)
-> list[MergeWindow]` where each `MergeWindow` = `(first_num, last_num, [chapters])`:

- Filter to chapters with `audio_path` and `audio_voice == voice`, sorted by index.
- **all:** one window spanning the min..max chapter number of the filtered set.
- **range:** one window; keep chapters whose number is in `[start, end]`.
- **batch(N):** group by number-window `[k*N+1 .. (k+1)*N]`; emit a window only if it
  contains ≥1 audio chapter (skip empty windows). Windows are numbered by their
  number-range, not by count, so gaps don't shift boundaries.
- Returns `[]` if nothing matches (caller shows "chưa có audio").

This is the unit-testable heart of the feature — no ffmpeg needed.

## 3. Worker — `src/noveltrans/gui/workers.py` (`MergeWorker`)

Mirror `ExportWorker`, but it can emit **several** files (batch mode):
- Signals: `progress = Signal(int, int, str)` (done-windows, total-windows, label),
  `file_done = Signal(str)` (each output path as it finishes), `finished_ok = Signal(int)`
  (count of files written), `failed = Signal(str)`.
- `__init__(project_path, voice, fmt, mode, start, end, batch, parent=None)`.
- `run()`: open project, `windows = plan_merge_windows(project.chapters(), voice, mode,
  …)`; if empty → `failed`. For each window: resolve absolute paths (skip missing files),
  compute `out_path` per the naming rules, `merge_chapters(...)`, `file_done.emit`,
  `progress.emit`. Support `cancel()` between windows (and best-effort terminate the
  running ffmpeg `Popen`). `finished_ok.emit(len(done))`.
- One ffmpeg invocation per window, so batch = N ffmpeg passes (progress is per-window).

## 4. UI — `src/noveltrans/gui/tab_audio.py`

A "Ghép audio" row (below the generate row, or a small group box), left→right:
- **Chế độ** `QComboBox`: "Toàn bộ" / "Từ chương … đến …" / "Theo lô". On change, show/
  hide the relevant inputs.
- **Range inputs:** two `QSpinBox` (from / to), 1..chapter-count, default 1..count.
- **Batch input:** one `QSpinBox` (N chương mỗi lô), default 10, min 1.
- **Định dạng ghép** `QComboBox`: "M4B (có mục lục chương)" / "MP3 (gộp phẳng)" — hide the
  M4B item if `ffmpeg_has_encoder("aac")` is false.
- **`QPushButton` "Ghép audio"** — disabled (tooltip "cần ffmpeg") when
  `not ffmpeg_available()`.

`_start_merge`:
- guard `self.project`; `voice = self.voice_combo.currentData()`. Preview the windows via
  `plan_merge_windows(...)` (cheap, no ffmpeg): if 0 windows → info "Chưa có chương nào có
  audio giọng {voice} trong phạm vi đã chọn".
- Confirm summary: `QMessageBox.question` — "Sẽ tạo {len(windows)} file từ {k} chương (giọng
  {voice}). Tiếp tục?" (so batch mode's file count is explicit before a long encode).
- disable controls, status "Đang ghép… (có thể mất vài phút)", start `MergeWorker` with the
  mode/params.

`_on_merge_file_done(path)`: append to a small "đã tạo" list / status. `_on_merge_finished(n)`:
re-enable, status "Đã ghép {n} file → mở thư mục", offer open (reuse `_open_audio_dir`, but
point at `exports_dir`). `_on_merge_failed(msg)`: warn. Register `MergeWorker` in
`has_running_workers()` / `shutdown()`.

## 4. Tests

ffmpeg-dependent, so gate the real-encode test behind availability (skip if no ffmpeg),
but unit-test the **pure** parts (no ffmpeg needed):
- **Chapter-metadata builder** (extract a pure `build_chapter_metadata(chapters) -> str`):
  cumulative START/END in ms from `audio_seconds`; titles follow `audio_source`
  (translated vs original); ordering by index. Assert exact ffmetadata text for a small
  list (e.g. two chapters 12.0s + 8.5s → START=0/END=12000, START=12000/END=20500).
- **Concat-list builder** (`build_concat_list(paths) -> str`): correct `file '...'` lines,
  single-quote escaping.
- **`plan_merge_windows`** (the selection heart — most important tests, no ffmpeg):
  - *all* → one window min..max of voiced chapters.
  - *range* 3..7 → only those chapter numbers; excludes out-of-range and wrong-voice.
  - *batch(10)* over chapters 1..25 → windows 1–10, 11–20, 21–30 (last holds 21–25);
    an empty number-window (no audio) yields no file; a gap inside a batch doesn't shift
    later batch boundaries.
  - voice filter: chapters voiced differently are excluded.
  - output-name derivation matches the naming rules (`<slug>-0001-0010`).
- **`@pytest.mark.live`-style / skip-if-no-ffmpeg:** synthesize two tiny WAVs (or use
  fixtures), merge to `.m4b`, assert the output exists and `ffprobe` reports 2 chapters.
  Keep it optional so CI without ffmpeg still passes.

## 5. Verification

1. `make test` green (pure builders always run; ffmpeg merge test skips w/o ffmpeg),
   `make lint` clean.
2. Manual: a project with a few voiced chapters → Audio tab → "Ghép thành 1 file" (M4B)
   → open in a player, confirm one file with per-chapter bookmarks and correct titles;
   scrub between chapters. Then MP3 mode → one flat file. Try with a missing-audio gap →
   the warn dialog appears and the merge skips the gap.

## Risks / notes

- **Duration drift:** chapter offsets come from stored `audio_seconds` (the synth's
  reported duration). If that ever diverges from the actual file, markers could drift over
  a long book. Mitigation/alt: probe each file with `ffprobe` for exact durations when
  building metadata — more correct but adds N ffprobe calls. Recommend stored seconds for
  v1 (fast, and the values are the real synth output), note ffprobe as the precise option.
- **Mixed input formats** (some WAV, some MP3): the concat demuxer + re-encode to AAC/MP3
  handles both; no need to normalise first.
- **Big output / long encode:** one ffmpeg pass over a whole novel can take minutes; the
  worker keeps the UI responsive. Mid-run cancel is best-effort (terminate Popen) or
  deferred — document whichever v1 ships.
- **No schema change** — purely reads existing audio fields.
