# 013 — Keep the Mac awake during long jobs

## Request

The user runs long jobs (download / translate / TTS / merge) and macOS idle-sleeps
mid-run. Make the app keep the Mac awake **while it's actually working**, and let it
sleep normally when idle.

## Approach (to confirm during /plan)

- **Mechanism:** hold a wake-lock via macOS `caffeinate -i` (prevents idle *system*
  sleep; works on battery; lets the display sleep — the run continues). Spawn it as a
  child process and terminate it when work ends. macOS-only; a no-op on other platforms
  (there is no platform check in the codebase today — add a `sys.platform == "darwin"`
  guard).
- **Scope = work-scoped, ref-counted.** A small `KeepAwake` manager: `acquire()` on the
  0→1 edge spawns `caffeinate`; `release()` on the 1→0 edge terminates it. The
  long-running workers (DownloadWorker, TranslateWorker, AudioWorker, MergeWorker) take a
  reference for the duration of a run, across all workspaces, so several parallel jobs
  share one lock and it's released only when the last finishes. (ScanWorker / voices /
  models are short — skip.)
- **Where to hook:** cleanest is the tabs that own the workers — acquire when a worker
  starts, release on its finished/failed signal (mirrors how the tabs already
  enable/disable buttons). Or a QThread base/mixin. Decide in /plan.
- **Setting:** a toggle in Settings — "Giữ máy thức khi đang chạy" (default on) — so the
  user can opt out. `config.keep_awake_enabled`.
- **Robustness:** the caffeinate child must die with the app (use `-w <app_pid>` as a
  backstop, and terminate on release / on window close) so a crash can't leave the Mac
  permanently awake. Never leak the process.

## Open questions

- Ref-count across workspaces: a single app-global `KeepAwake` (all workspaces share it)
  vs per-window. App-global is right (sleep is a machine-level concern).
- Whole-session alternative (stay awake the entire time the app is open) — simpler but
  blocks sleep when idle. Prompt leans work-scoped; confirm.
- Optional: a subtle indicator (status bar / menu) that the wake-lock is active, so the
  user knows why the Mac isn't sleeping.
- Verify `caffeinate` is always present (it's a stock macOS binary) — if missing, degrade
  to no-op silently.
