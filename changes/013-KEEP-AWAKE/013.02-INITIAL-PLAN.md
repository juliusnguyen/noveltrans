# 013.02 — Initial Plan: keep the Mac awake during long jobs

## Goal

While the app is actively working (download / translate / TTS / merge), prevent macOS
from idle-sleeping; let it sleep normally when idle. App-global, ref-counted, opt-out via
a setting, and impossible to leak past app exit.

## Design in one line

A module-level, ref-counted `KeepAwake` singleton spawns `caffeinate -i -w <app_pid>` on
the 0→1 edge and terminates it on the 1→0 edge; the long-running workers take a reference
for the duration of their run via `track_worker(worker)` (wired to QThread's own
`finished` signal), and Settings toggles it.

## Decisions (resolving the prompt's open questions)

1. **Work-scoped, app-global, ref-counted** (confirmed). One shared lock across all
   workspaces; the Mac stays awake only while ≥1 tracked worker is running, and is free to
   sleep the instant the last finishes. Sleep is a machine concern → app-global, not
   per-window.
2. **`caffeinate -i -w <app_pid>` + explicit terminate.** `-i` blocks idle *system* sleep
   (works on battery; display may still sleep — the run continues). `-w <app_pid>` is the
   crash/leak backstop: even if we never call terminate, caffeinate exits when the app
   process does, so it can never leave the Mac awake forever. Normal path still calls
   `terminate()` on the 1→0 edge to release promptly.
3. **No UI indicator in v1** (deferred). The tabs already show "Đang tải…/Đang dịch…"
   during work, which is the visible signal; a dedicated wake-lock indicator (status bar /
   menu) is a clean follow-up, noted not built, to keep this tight.
4. **Setting default ON.** `config.keep_awake_enabled` (default True). Disabling it mid-run
   also stops any active caffeinate.

## 1. Core — new `src/noveltrans/gui/keep_awake.py`

A single module-global manager (mirrors how `gui/notify.py` exposes app-global helpers):

```python
import os, shutil, subprocess, sys

class _KeepAwake:
    def __init__(self):
        self._count = 0
        self._proc: subprocess.Popen | None = None
        self._enabled = True

    def set_enabled(self, enabled: bool) -> None:
        self._enabled = enabled
        if not enabled:
            self._stop()          # release immediately if the user opts out mid-run

    def acquire(self) -> None:
        self._count += 1
        if self._count == 1:
            self._start()

    def release(self) -> None:
        if self._count == 0:
            return
        self._count -= 1
        if self._count == 0:
            self._stop()

    def shutdown(self) -> None:    # called on app quit — force-release
        self._count = 0
        self._stop()

    def _start(self) -> None:
        if not self._enabled or self._proc is not None:
            return
        if sys.platform != "darwin":            # macOS-only; silent no-op elsewhere
            return
        if shutil.which("caffeinate") is None:  # stock on macOS, but degrade safely
            return
        self._proc = subprocess.Popen(
            ["caffeinate", "-i", "-w", str(os.getpid())],
            stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL,
        )

    def _stop(self) -> None:
        if self._proc is not None:
            try:
                self._proc.terminate()
            except Exception:
                pass
            self._proc = None


_manager = _KeepAwake()

def set_enabled(enabled: bool) -> None: _manager.set_enabled(enabled)
def shutdown() -> None: _manager.shutdown()

def track_worker(worker) -> None:
    """Hold the wake-lock for one worker run. Call right before worker.start()."""
    _manager.acquire()
    worker.finished.connect(_manager.release)   # QThread.finished fires once per run
```

Notes:
- `track_worker` acquires immediately and releases on the QThread's built-in `finished`
  signal (emitted exactly once when `run()` returns, delivered to the GUI thread) — no
  dependency on `started` timing, and balances even on early error/cancel (run() still
  returns → finished fires).
- If `set_enabled(False)` is toggled while a job runs, `_stop()` kills caffeinate now; the
  count keeps ticking so a later `set_enabled(True)` won't spuriously re-lock — `_start`
  is only reached on the next 0→1 edge. (Acceptable: re-enabling mid-run waits for the
  next job. Document.)

## 2. Hook points — the four long workers

Call `track_worker(w)` immediately before `w.start()` where each long job launches:
- `gui/tab_scrape.py` `_start_download` → the `DownloadWorker`.
- `gui/tab_translate.py` → the `TranslateWorker` (`_worker`).
- `gui/tab_audio.py` `_start_generate` → `AudioWorker`; `_start_merge` → `MergeWorker`.

Skip the short ones (Scan, voices, models, unlock/login) — they finish in seconds and
don't need it. (UnlockWorker during an auto-unlock+resume is brief; the resumed
DownloadWorker re-locks anyway. Note as an easy add if wanted.)

Each site is a one-liner (`from noveltrans.gui.keep_awake import track_worker` +
`track_worker(self._worker)` before `.start()`), so no worker internals change.

## 3. Setting — `config.py` + `settings_dialog.py`

- `AppConfig.keep_awake_enabled` (bool, default **True**) — same pattern as
  `discord_autounlock_enabled` (`self._s.value("keep_awake_enabled", True, type=bool)`).
- `SettingsDialog`: a `QCheckBox` "Giữ máy thức khi đang chạy (tải/dịch/tạo audio)"
  under an existing section; on `accept()`, write the config AND call
  `keep_awake.set_enabled(value)` so the change takes effect live.

## 4. Wiring lifecycle — `app.py`

- In `main()`, after building `AppConfig`: `keep_awake.set_enabled(config.keep_awake_enabled)`
  so the startup state matches the saved setting.
- Ensure release on quit: connect `app.aboutToQuit` → `keep_awake.shutdown()` (belt-and-
  suspenders; the `-w <pid>` backstop already covers a hard crash).

## 5. Tests — `tests/test_keep_awake.py`

Pure logic, `subprocess.Popen`/`shutil.which`/`sys.platform` monkeypatched — no real
caffeinate, no Qt needed for the manager:
- **ref-counting:** acquire → spawns once; a second acquire does NOT spawn again; release
  down to 0 terminates; extra release is a no-op (no negative count).
- **overlap:** acquire, acquire, release (still 1 → process alive), release (0 → stopped).
- **platform guard:** `sys.platform != "darwin"` → never spawns.
- **missing binary:** `shutil.which` → None → no spawn, no error.
- **disabled:** `set_enabled(False)` → acquire never spawns; toggling False mid-lock stops
  the process.
- **shutdown:** force-releases and terminates regardless of count.
- `track_worker`: with a fake object exposing a `finished` signal-like connect, assert
  acquire on call + release when `finished` fires (can use a tiny `QObject` with a Signal
  under the offscreen `qapp`, or a stub that records the connected callback).

## Risks / notes

- **Leak safety** is the main risk and is covered three ways: `finished`-driven release,
  `aboutToQuit` shutdown, and the `-w <app_pid>` backstop. Manual check: no lingering
  `caffeinate` after quit.
- **Display still sleeps** with `-i` (intended — screen off, work continues). If a user
  wants the screen on too, `-d` is a one-word add; out of scope.
- **Clamshell / battery:** closing the lid still sleeps regardless (macOS clamshell) —
  document; nothing the app can do without power-adapter + external display.
- **No new heavy deps** — `caffeinate` is a stock macOS binary; pure stdlib subprocess.
