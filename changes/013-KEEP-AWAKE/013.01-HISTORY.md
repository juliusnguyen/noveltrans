# 013 — Keep the Mac awake during long jobs — History

## [PLAN] 013.02-INITIAL-PLAN — 2026-07-16

Generated plan: `013.02-INITIAL-PLAN.md`

App-global, ref-counted `KeepAwake` singleton (new `gui/keep_awake.py`) that spawns
`caffeinate -i -w <app_pid>` on the 0→1 edge and terminates it on 1→0. The four long
workers (Download/Translate/Audio/Merge) call `track_worker(w)` before `.start()`, wiring
release to QThread's `finished`. Setting `keep_awake_enabled` (default on) + live apply;
`app.py` sets startup state and force-releases on `aboutToQuit`.

Decisions: work-scoped + app-global; `-w <app_pid>` as the crash/leak backstop; `-i`
(idle system sleep, battery-safe, display may sleep); no UI indicator in v1 (deferred).
Tests mock subprocess so no real caffeinate runs.

## 2026-07-16 — Implementation

Built the plan.

**Changes:**
- `gui/keep_awake.py` (new): ref-counted `_KeepAwake` singleton; `caffeinate -i -w
  <app_pid>` on 0→1, `terminate()` on 1→0; `set_enabled`/`shutdown`/`track_worker`
  (acquire now, release on QThread `finished`). darwin-only, degrades to no-op.
- Hooked the four long workers via `track_worker(w)` before `.start()`:
  DownloadWorker (tab_scrape), TranslateWorker (tab_translate), AudioWorker + MergeWorker
  (tab_audio).
- `config.keep_awake_enabled` (default True); Settings checkbox "Giữ máy thức khi đang
  chạy" with live `keep_awake.set_enabled` on save.
- `app.py`: `set_enabled(config…)` at startup + `aboutToQuit → shutdown`.
- `tests/test_keep_awake.py`: ref-counting, guards (non-macOS / missing binary /
  disabled / disable-mid-lock), shutdown, and `track_worker` acquire+release-on-finished.

**Verification:** ruff clean; 247 tests pass. Live macOS check: acquire spawns exactly
one `caffeinate -i -w`, release terminates it, no lingering process (the pre-existing
`caffeinate -i -t 300` in the shell is the harness's, not ours).
