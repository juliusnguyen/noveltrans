# 017 — Strip special characters before TTS

Add a text-cleaning step to the audio pipeline that removes special characters from
chapter text **before** it is synthesized, so the TTS engine renders more smoothly
(fewer odd pauses, mispronounced symbols, or artifacts from stray glyphs).

## Context

- The natural insertion point is `TtsEngine.synthesize_chapter` in
  `src/noveltrans/tts/base.py`, which builds `text = f"{title}\n\n{body}"` and then
  calls `split_sentences(...)`. Cleaning would happen just before the split.
- The audio is generated from the **translated** (Vietnamese) text, so cleaning must
  preserve Vietnamese — diacritics/tone marks are letters, not "special characters".

## Requirements

- A function that strips characters that don't belong in spoken text.
- Applied before synthesis so the audio renders more smoothly.

## Open questions for /plan

- **What exactly is a "special character"?** Must KEEP: Vietnamese letters (with tone
  marks), digits, and sentence punctuation that shapes prosody (. , ! ? … ; : and
  quotes). Candidates to REMOVE/normalise: emoji, decorative symbols (★ ◆ ※ 【】),
  stray CJK left over from source, zero-width/control chars, markdown leftovers (*, #,
  _, `), and unusual brackets. Whitelist (keep-known-good) vs blacklist (strip-known-
  bad) — which is safer?
- **Punctuation for prosody:** removing ALL punctuation would hurt the audio, not help
  it. The cleaner should keep sentence-ending and clause punctuation. Confirm the set.
- **On/off and scope:** always-on, or a Settings toggle (default on/off)? Per-render or
  global? (Compare feature 014's `tts_workers` and 013's keep-awake settings.)
- **Does it touch the stored text or only the copy sent to TTS?** Cleaning the copy fed
  to the engine (leaving `translated` in the DB intact) is least destructive and needs
  no re-translation. Confirm.
- **Numbers/symbols read aloud:** should `%`, `&`, `$`, degrees, etc. be spoken as
  words, stripped, or left? (Expansion is a bigger feature than stripping — likely out
  of scope, but name it.)
- **Testability:** the cleaner should be a pure function (like `find_replace`,
  `translators/names`) so it's unit-tested without the TTS engine.
