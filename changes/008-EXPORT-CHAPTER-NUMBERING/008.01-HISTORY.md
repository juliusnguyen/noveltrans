# 008 — Export chapter auto-numbering — History

## [PLAN] 008.02-INITIAL-PLAN — 2026-07-15

Generated plan: `008.02-INITIAL-PLAN.md`

Small, contained feature: an export option to title chapters "Chương N" instead of the
parsed `Tên chương`. Traced that the heading is chosen in one place —
`chapter_text()` in `exporters/base.py` — so a `number_chapters: bool` flag threads
from an Export-tab checkbox → `ExportWorker` → `Exporter.export` → `split_available`
→ `chapter_text`, and all three formats (Markdown/DOCX/EPUB incl. EPUB TOC) pick it up
for free. Decisions: number by `chapter.index + 1` (gap-preserving, matches the "#"
column); label "Chương {n}"; pure number (no title). English "Chapter {n}" deferred.

## 2026-07-15 — Implementation

Built the plan as written.

**Changes:**
- `exporters/base.py`: `chapter_text(..., number_chapters=False)` overrides the heading
  with `_NUMBER_LABEL.format(n=chapter.index + 1)`; `split_available` + the `Exporter.export`
  ABC gained the flag.
- `exporters/markdown.py` / `docx.py` / `epub.py`: accept `number_chapters` and forward
  it to `split_available` (headings/TOC unchanged otherwise — they read `text.title`).
- `gui/workers.py` `ExportWorker`: new `number_chapters` field, passed to `export`.
- `gui/tab_export.py`: "Đánh số chương tự động (bỏ tên chương)" checkbox in the options
  box, wired into the worker. Default off.
- `tests/test_exporters.py`: `TestChapterNumbering` — unit (translated + original),
  markdown end-to-end, gap-preserving (skipped index not renumbered), EPUB TOC, and a
  regression that default output keeps the parsed title.

**Verification:** ruff clean; 211 tests pass. Real export contrast confirmed: with the
flag, garbled titles ("On Jianyong…", "Tên chương lỗi ###") become "Chương 1 / Chương
2"; without it, titles are preserved.

## 2026-07-15 — Corrected format per user

User clarified they want the number **prepended to** the title, not replacing it:
`Chương {n}: {Tên chương}` (e.g. "Chương 1: On Jianyong xuất hiện"). Changed
`_NUMBER_LABEL` (replace) → `_NUMBER_PREFIX = "Chương {n}: "` prepended to the parsed
title. Checkbox relabelled "Thêm số chương vào tiêu đề" (no longer "bỏ tên chương").
Tests updated for the "Chương N: <title>" form. 211 pass, ruff clean.

## 2026-07-15 — Doubled-number guard

Added `_ALREADY_NUMBERED_RE` in `chapter_text`: if the parsed title already opens with
its own chapter marker — Vietnamese ("Chương 3", "Chuong 3"), English ("Chapter 3"), or
Chinese ("第三章", "第3章") — the "Chương {n}: " prefix is skipped so exports don't read
"Chương 1: Chương 1: …". Plain titles still get the prefix. Tests: guard cases for all
three marker styles + updated e2e (plain titles get prefixed; marked titles untouched).
213 pass, ruff clean.
