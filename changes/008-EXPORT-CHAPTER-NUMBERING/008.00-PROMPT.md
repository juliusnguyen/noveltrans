# 008 — Export: auto-number chapters instead of parsed titles

## Request

> In the exported file, add an option so that instead of using the `Tên chương`
> (chapter title parsed by the app), it auto-names each chapter by its chapter
> number.

When exporting (DOCX / Markdown / EPUB), offer an option to title each chapter as a
plain sequential number (e.g. "Chương 1", "Chương 2", …) rather than the scraped/
translated chapter title. Motivation: parsed titles are often noisy or contain
garbled proper nouns (see the name-romanization issue), so clean numbering is
sometimes preferable.

## Current state (verified)

- `src/noveltrans/exporters/base.py`:
  - `chapter_text(chapter, use_translation) -> ExportText | None` picks the title as
    `chapter.translated_title or chapter.title` (translated path) or `chapter.title`
    (original path). This is the single place the chapter heading text is chosen.
  - `Exporter.export(meta, chapters, out_path, use_translation=True)` is the ABC;
    `split_available(chapters, use_translation)` builds the (chapter, ExportText)
    list used by all three exporters.
- `Chapter` has a 0-based `index` (models.py), so a number label is `index + 1`.
- Exporters `markdown.py`, `docx.py`, `epub.py` all consume `ExportText.title` for
  their headings.
- `src/noveltrans/gui/tab_export.py` has the export UI (format combo, translated/
  original radios) and builds an `ExportWorker(project.path, exporter.name, out_path,
  use_translation)`; `ExportWorker` (gui/workers.py) calls the exporter.

## Scope (to confirm during /plan)

- A UI toggle in the Export tab, e.g. checkbox "Đánh số chương tự động (bỏ tên chương)".
- Plumb a `number_chapters: bool` option from the tab → `ExportWorker` →
  `Exporter.export(...)` → `chapter_text` / `split_available`, so when on, each
  chapter's heading becomes a numbered label instead of its title.
- Numbering source: use `chapter.index + 1` (stable, matches the in-app "#" column) —
  NOT the export position, so skipped/missing chapters don't renumber the rest.
  Confirm during /plan.
- Keep the body text unchanged; only the heading/title is replaced.

## Open questions

- Label wording: "Chương {n}" always, or localize to the export language
  ("Chapter {n}" when exporting English)? Default: "Chương {n}" (Vietnamese UI).
- Should the numbered label optionally *keep* the title too (e.g. "Chương 1 — <title>")?
  Or pure number only? Request says "auto name by the number" → pure number by default.
- EPUB nav/TOC entries + DOCX heading + Markdown "##" should all use the numbered
  label consistently.
- Numbering base: `index + 1` (per-chapter, gap-preserving) vs. sequential over the
  exported/included chapters. Prompt leans `index + 1`.
