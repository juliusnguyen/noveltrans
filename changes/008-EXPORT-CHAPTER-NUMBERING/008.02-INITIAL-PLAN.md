# 008.02 — Initial Plan: export chapter auto-numbering

## Goal

Add an export option that titles each chapter by its number ("Chương 1", "Chương 2",
…) instead of the scraped/translated `Tên chương`. Applies to all three formats
(Markdown, DOCX, EPUB) — headings **and** the EPUB TOC/nav.

## Design in one line

Thread a `number_chapters: bool` from an Export-tab checkbox → `ExportWorker` →
`Exporter.export(...)` → `split_available` → `chapter_text`, where — the single place
the heading text is chosen — a numbered label replaces the title. Only the heading
changes; body text is untouched.

## Decisions (resolving the prompt's open questions)

1. **Numbering source: `chapter.index + 1`.** Matches the in-app "#" column and is
   gap-preserving — if chapter 7 is skipped (not downloaded/translated), chapters 6
   and 8 keep their numbers instead of the export re-sequencing to hide the gap. This
   is the honest, predictable choice for a reader comparing to the site.
2. **Label wording: "Chương {n}" always.** The UI is Vietnamese and the existing
   front-matter strings ("Bỏ qua … chương", "Tác giả") are Vietnamese regardless of
   export language, so "Chương {n}" is consistent with the current output. English
   ("Chapter {n}") is noted as a trivial follow-up if wanted, keyed off the export
   language — deliberately out of scope to keep this change minimal.
3. **Pure number, no title.** The request is "auto name by the number"; the numbered
   label fully replaces the title. (A "Chương 1 — <title>" hybrid is a possible future
   option but not built now.)

## Central change — `src/noveltrans/exporters/base.py`

`chapter_text` is the one function that decides the heading. Add the flag here and it
propagates to every exporter for free.

```python
_NUMBER_LABEL = "Chương {n}"  # heading when number_chapters is on

def chapter_text(
    chapter: Chapter, use_translation: bool, number_chapters: bool = False
) -> ExportText | None:
    """Pick translated or original text; None if that text isn't available.

    number_chapters replaces the heading with a plain "Chương {n}" label (n =
    chapter.index + 1), leaving the body untouched.
    """
    if use_translation:
        if not chapter.translated:
            return None
        title = chapter.translated_title or chapter.title
        body = chapter.translated
    else:
        if not chapter.content:
            return None
        title = chapter.title
        body = chapter.content
    if number_chapters:
        title = _NUMBER_LABEL.format(n=chapter.index + 1)
    return ExportText(title, body)
```

And thread it through the availability splitter (used by all three exporters):

```python
@staticmethod
def split_available(
    chapters, use_translation, number_chapters=False
) -> tuple[list[tuple[Chapter, ExportText]], list[Chapter]]:
    ...
    text = chapter_text(chapter, use_translation, number_chapters)
    ...
```

And the ABC signature:

```python
@abstractmethod
def export(self, meta, chapters, out_path, use_translation=True, number_chapters=False) -> Path: ...
```

Note: the "skipped chapters" front-matter lists `c.title` (the real titles) — leave
that as-is (it's an informational note about what was omitted; numbering the omitted
list would be less useful). Only the *included* chapter headings get numbered.

## The three exporters (mechanical)

Each already loops `for … (_chapter, text) in included` and uses `text.title`. The
only change is accepting `number_chapters` and passing it to `split_available`:

- **`markdown.py`**: `export(..., number_chapters=False)`; call
  `self.split_available(chapters, use_translation, number_chapters)`. `## {text.title}`
  is unchanged — it now shows "Chương N".
- **`docx.py`**: same signature + `split_available` call; `add_heading(text.title, 1)`
  unchanged.
- **`epub.py`**: same; `item.title` / `<h1>{text.title}</h1>` / `book.toc = items[1:]`
  all key off `text.title`, so the TOC entries and nav become "Chương N" automatically.
  The file name stays `chap_{i+1:04d}.xhtml` (export position — fine for filenames;
  the *visible* label is `text.title`).

## Worker — `src/noveltrans/gui/workers.py` (`ExportWorker`)

- Add `number_chapters: bool = False` to `__init__` (store `self.number_chapters`).
- Pass it in the `exporter.export(..., number_chapters=self.number_chapters)` call.

## UI — `src/noveltrans/gui/tab_export.py`

- Add a checkbox under the existing "Nội dung" (translated/original) row in the
  `options_box` form, e.g.
  `self.number_checkbox = QCheckBox("Đánh số chương tự động (bỏ tên chương)")`
  with a tooltip explaining it replaces titles with "Chương 1, 2, …". Default
  unchecked (preserve current behaviour).
- In `_start_export`, pass `number_chapters=self.number_checkbox.isChecked()` when
  constructing `ExportWorker`.
- No change to `default_export_name` (file name still slugs the novel title).

## Tests — extend `tests/test_exporters*.py` (or add one)

- **base/unit:** `chapter_text(chapter, use_translation=True, number_chapters=True)`
  returns `ExportText(title="Chương {index+1}", body=<translated>)`; body unchanged;
  original-text path likewise; `index+1` used (not export position).
- **gap-preserving:** a chapter list where index 1 is skipped (no content) →
  included headings are "Chương 1" and "Chương 3" (not renumbered to 1,2).
- **markdown end-to-end:** export with `number_chapters=True` → output contains
  `## Chương 1` and not the original `## <title>`.
- **epub:** exported book's chapter `EpubHtml.title` / TOC entries are "Chương N"
  (parse via ebooklib or assert on the item titles before write).
- Confirm default (`number_chapters=False`) output is byte-identical to today
  (regression guard).

## Risks / notes

- Small blast radius: the flag defaults False everywhere, so existing exports are
  unchanged unless the box is ticked.
- `split_available` is a shared `@staticmethod`; adding a defaulted param is
  backward-compatible with any other caller.
- English label ("Chapter {n}") intentionally deferred — one-line follow-up keyed on
  export language if desired.
