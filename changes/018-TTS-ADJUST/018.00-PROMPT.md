# 018 — Adjustable TTS settings

Add settings that let the user adjust how the TTS audio is produced — beyond the
existing voice, format, workers, and clean-text controls.

## What's actually adjustable (verified against the code + the VieNeu library)

**App-side (easy, safe, no quality risk):**
- **Pause length between paragraphs/chunks** — `TtsEngine.paragraph_gap_seconds`
  (currently hardcoded 0.4s in `tts/base.py`). Silence inserted between chunks. Some
  listeners want tighter, some want more breathing room. Trivial to expose.
- **Chunk size** — `TtsEngine.max_chunk_chars` (400). More of an advanced/rare knob;
  affects how text is batched, not obviously audible. Probably leave out.
- **Volume / gain** — a scalar multiply on the samples (or ffmpeg). Easy, some value.

**Engine-side (VieNeu `infer` params, thread through `vieneu.py::synthesize`):**
- **`temperature`** (default 1.0) — expressiveness/variability. Higher = more varied,
  lower = flatter/steadier. Subtle; can make output less consistent.
- **`top_k`** (default 50), **`silence_p`** (0.15) — niche; likely not worth surfacing.

**Speaking speed / rate — the one users most want, but NOT native:**
- VieNeu's `infer` has **no speed/rate/length_scale parameter** (verified — the "speed"
  strings in its source are comments). Adjusting pace requires **post-processing** the
  rendered audio, e.g. ffmpeg `atempo` (which preserves pitch). ffmpeg is already used
  for mp3 conversion (`tts/convert.py`), so the plumbing exists, but this is a real
  audio-processing step with its own quality/robustness considerations, and it must
  also run for WAV output (where ffmpeg isn't currently required).

## Open questions for /plan

- **Which knobs?** Recommend starting with the safe, high-value ones (pause length;
  maybe volume) and deciding speed separately given its cost.
- **Speed via ffmpeg `atempo`:** worth it? It's the most-requested control but the most
  complex (post-process every chapter, handle WAV-without-ffmpeg, atempo's 0.5–2.0 range
  needing chaining beyond that). Could be a follow-up.
- **Where the values live:** `AppConfig` props + Settings controls (sliders/spinboxes),
  threaded through `AudioWorker` → `synthesize_chapter` → the engine, exactly like the
  clean-text toggle from feature 017.
- **Preview/preview-listen:** hard to preview audio changes without rendering. The
  existing "Xem trước văn bản" is text-only. Out of scope unless asked.
- **Defaults must reproduce today's output** (gap 0.4, temperature 1.0) so existing
  users hear no change unless they opt in.
