# 003-MEDOCTRUYEN-SOURCE — History

## [PLAN] 003.02-INITIAL-PLAN — 2026-07-14
Generated plan: `003.02-INITIAL-PLAN.md`

Probed the live site (`https://medoctruyen.vn/tu-bao-tien-bon`) before planning:
- Next.js App Router (server-rendered HTML, parseable by requests+BeautifulSoup).
- Metadata + paginated TOC (`?ch=N`) readable anonymously; content is Vietnamese.
- **Key constraint:** full chapter bodies are login-gated (`isPreview:true`,
  `data-preview-gate`), even for free chapters. Requires an authenticated session.

Plan raises two DECISION-NEEDED items: (1) auth strategy — recommends
cookie-injection over programmatic login; (2) language — recommends
identity/passthrough translation when `source_lang == target_lang` so TTS/export
work unchanged for vi-source novels.

## [FIX] Preview-gate false positive — 2026-07-14
With a valid session cookie, downloads still failed with "cần đăng nhập". Root cause:
`fetch_chapter` treated the mere presence of the string `previewLoginHref` as a gate,
but that string is emitted on EVERY page (including authenticated, full-content ones),
so full chapters were wrongly rejected. Fixed by dropping the `previewLoginHref` check;
gating now relies only on the rendered `[data-preview-gate]` element and
`"isPreview":true`, which appear only when the body is actually truncated. Verified
against the real site with the user's stored cookie: chuong-1 now returns the full
8.5k-char / 86-paragraph body instead of raising. Existing preview-gate test still
passes (its fixture carries both real gate markers). 156 tests pass, ruff clean.

## [FIX] Rate-limit handling + copyable errors — 2026-07-14
Downloads failed from ~chapter 4 with "Chapter content not found". Root cause:
medoctruyen.vn throttles reads — after ~3 chapters read quickly it serves the page
WITHOUT the `<article>` body and sets `"isRateLimited":true` + the notice "Bạn đọc
quá nhanh…". Measured recovery ≈60s. The adapter was misreading this as a broken
layout.
- `errors.py`: added `RateLimitedError(ScrapeError)`.
- `scrapers/medoctruyen.py`: when the body is absent, distinguish the throttle
  (`"isRateLimited":true` or the "Bạn đọc quá nhanh" notice → `RateLimitedError`,
  retryable) from a genuine layout break (→ `ScrapeError`). Verified live: chapters
  4–6 download, chuong-7 correctly raises `RateLimitedError`.
- `gui/workers.py`: `DownloadWorker._fetch_with_backoff` catches `RateLimitedError`,
  waits `_RATE_LIMIT_WAIT_SECONDS` (60s) with an interruptible per-second countdown
  shown in the status bar, retries up to `_RATE_LIMIT_MAX_RETRIES` (8) before giving
  up. So bulk downloads now proceed unattended (~1 chapter / ~20s effective for this
  site) instead of erroring out.

Also, per user request, made error text easy to copy:
- `gui/widgets.py`: full error as `ToolTipRole` on the error column of both chapter
  models (hover shows the untruncated message); new `enable_cell_copy(table)` helper
  adds Ctrl+C and a right-click "Sao chép" menu that copies a cell (its full tooltip
  text when longer).
- Wired `enable_cell_copy` onto the scrape/translate/audio tables; made the scrape
  status label selectable (`TextSelectableByMouse`).
Tests: added rate-limit fixture + `TestRateLimit` (throttle → retryable, missing
article without throttle → hard error). 158 tests pass, ruff clean.

## [FEATURE] Daily-limit detection (declined Discord automation) — 2026-07-14
Chapter 201 showed a hard "GIỚI HẠN ĐỌC TRONG NGÀY — đã đọc đủ 50 chương hôm nay"
page (`"dailyLimit":{"limit":50,"used":50,"resetAt":...}`, `isRateLimited:false`).
The site's unlock path is a Discord slash command `/mochuong <CODE>` in `#mở-khoá`.

Decision: did NOT automate the Discord unlock — it would require self-botting the
user's Discord account (against Discord ToS, account-ban risk) and would circumvent a
deliberate anti-leak access control ("Do tình trạng leak truyện nhiều"). Instead made
the manual unlock frictionless:
- `errors.py`: added `DailyLimitError(ScrapeError)` (distinct from the retryable
  `RateLimitedError`).
- `scrapers/medoctruyen.py`: detect the daily-limit page (notice "GIỚI HẠN ĐỌC TRONG
  NGÀY") before the reading-too-fast check; `_daily_limit_message()` extracts and
  surfaces the site's own unlock code, Discord invite, and reset time as an actionable
  Vietnamese message.
- `gui/workers.py`: `DownloadWorker` catches `DailyLimitError`, marks the row, shows
  the message in the status bar, and STOPS the batch (every remaining chapter is
  capped too) — no pointless 60s retries. Verified live against chuong-201: raises the
  message with code URFME4PA + discord.gg link + reset date.
- Test + fixture: `TestRateLimit.test_daily_limit_raises_with_unlock_instructions`.
159 tests pass, ruff clean.
