# 003.02 — Initial Plan: medoctruyen.vn scraper source

## 0. Summary & verified ground truth

Confirmed against the repo:
- `SiteAdapter` ABC + `HttpClient` in `src/noveltrans/scrapers/base.py`; `HttpClient.__init__(self, delay_seconds=1.5, max_retries=3, timeout=20.0)` has **no** cookie support (session headers set only `User-Agent`).
- Registry in `scrapers/__init__.py`: `@register`, `adapter_for_url(url, client)`, `_import_adapters()` imports `ixdzs, xbanxia`.
- `NovelMeta` (models.py) already has `source_lang: str = "zh"`; `NovelMeta.from_dict` filters to known fields, so adding nothing new is needed for language. `meta.json` round-trips it via `to_dict`/`from_dict`.
- `ScanWorker` and `DownloadWorker` (gui/workers.py) each construct their own `HttpClient(delay_seconds=...)` then `adapter_for_url(...)`. Neither currently receives cookies. `DownloadWorker.__init__(project_path, delay)`.
- Translation source flows via `translator.translate_chapter(..., source=project.meta.source_lang, target=self.target_lang)` in `TranslateWorker.run`.
- **Critical downstream fact:** the pipeline keys off DB columns. Export `use_translation=True` requires `chapter.translated`; `AudioWorker`/`pending_audio(voice)` require `translated != ''` and synthesize `chapter.translated_title, chapter.translated`. So for a Vietnamese-source novel, TTS and "export translated" only work if `translated` is populated. Export "Bản gốc" already works off `content`.

### Live-site findings (probed the real site)

Example novel: `https://medoctruyen.vn/tu-bao-tien-bon`

1. **Framework**: Next.js App Router (RSC). Content is server-rendered into HTML, so `requests` + BeautifulSoup can parse it. Pages carry `__next_f.push(...)` streaming payloads too.
2. **Metadata** (anonymous): fully available via OpenGraph meta tags on the novel landing page — `og:title`, `og:description`, `og:image` (cover on cdn.medoctruyen.vn), `og:url`.
3. **Language**: Content is **Vietnamese** (already Hán-Việt translated from a Chinese original). NOT Chinese.
4. **Chapter list / TOC**: Paginated on the landing page via `?ch=N` (e.g. `?ch=1` … `?ch=82` for 1951 chapters, ~24/page). Each TOC page server-renders anchors `href="/tu-bao-tien-bon/chuong-<N>"` with visible text like "Chương 25: Hỏa Oa". Some entries carry a "Miễn phí" (Free) tag; others are paid. Pager exposes first/last page numbers. Chapter URL pattern: `https://medoctruyen.vn/<novel-slug>/chuong-<N>`.
5. **CHAPTER CONTENT IS LOGIN-GATED — THE CRITICAL CONSTRAINT**: An anonymous GET of `/tu-bao-tien-bon/chuong-1` returns an `<article>` with only the first ~5 paragraphs (89 words), then `<div data-preview-gate="true">` "Đăng nhập để đọc tiếp". The RSC payload contains `"isPreview":true`, `"previewLoginHref":"/auth/login?next=..."`, and `"isLocked":false`. So even *free* chapters require an authenticated session to fetch the full body; `isLocked` (paid/coin-gated) is a separate stronger gate. `<article>` + `<p data-line="N">` is the content structure when full text is available.
6. **Login**: `/auth/login` is a client-side React form (email/password + Google/Facebook social); no plain HTML form action — auth is via a JSON/XHR API that sets a session cookie. Programmatic login (social + likely captcha) is fragile.

---

## DECISION NEEDED #1 — Authentication strategy

**Recommendation: cookie-injection.** The user logs into medoctruyen.vn in their own browser, copies their session cookie string, and pastes it into a NovelTrans settings field. The scraper attaches it to the `requests.Session` for medoctruyen requests only.

Rationale vs. programmatic email/password login:
- `/auth/login` is a client-side React form posting to a JSON/XHR API (no HTML `form action`), plus Google/Facebook social buttons and likely captcha. Reimplementing that is high-fragility and breaks whenever the auth API changes.
- Cookie-injection works uniformly for free-but-gated chapters *and* paid chapters the user's own account can access, with near-zero site-coupling.
- Trade-off: cookies expire → the plan includes explicit expiry detection with an actionable error (§4/§5). This is the only real downside and it is handled.

If the user prefers programmatic login later, it can be added as a second code path behind the same config field without disturbing this design. **Proceeding with cookie-injection below.**

## DECISION NEEDED #2 — Language handling for vi-source

**Recommendation: identity/passthrough "translation" when `meta.source_lang == target_lang`.** In `TranslateWorker.run`, when source == target, skip the engine and copy `content → translated`, `title → translated_title`, `target_lang = source_lang`. This populates the DB so **export-translated and TTS work with zero changes** to those tabs/exporters. It is the smallest-blast-radius option (one worker branch) versus making `pending_audio`, `AudioWorker`, and export all fall back to `content` (touches 3+ code paths and changes their invariants).

Downstream effect: user still opens the Translate tab and clicks "Dịch tất cả", but it runs instantly (a copy), giving a uniform mental model (every novel is "translated" before TTS/export). **Proceeding with identity-translate below.**

## 1. New file: `src/noveltrans/scrapers/medoctruyen.py`

New `@register`ed `MedoctruyenAdapter(SiteAdapter)`.

Class attrs:
- `name = "medoctruyen"`
- `display_name = "Mê Đọc Truyện (medoctruyen.vn)"`
- `url_patterns = [r"medoctruyen\.vn/[a-z0-9-]+"]` — matches both the novel landing (`/tu-bao-tien-bon`) and chapter URLs (`/tu-bao-tien-bon/chuong-1`). `matches()` (inherited, uses `re.search`) is fine.
- Selectors as class constants so tests/fixtures pin them:
  - `SEL_ARTICLE = "article"`
  - `SEL_PARAGRAPHS = "article p"` (site uses `<p data-line="N">`; select all `<p>` under article)
  - `SEL_PREVIEW_GATE = "[data-preview-gate]"`
  - `SEL_TOC_LINKS = 'a[href*="/chuong-"]'`

Helper methods / signatures:
- `def _soup(self, url: str, **kwargs) -> BeautifulSoup` → `BeautifulSoup(self.client.get_html(url, **kwargs), "lxml")` (mirrors xbanxia).
- `@staticmethod def _slug(url: str) -> str` — first path segment: `re.search(r"medoctruyen\.vn/([a-z0-9-]+)", url)`; raise `ScrapeError("Could not extract novel slug from URL", url)` if none.
- `def _novel_url(self, url: str) -> str` — normalize any URL (novel or chapter) to the landing page: `f"https://medoctruyen.vn/{self._slug(url)}"`. Used so pasting a chapter URL still scans the novel.

`fetch_metadata(self, url) -> NovelMeta`:
- `soup = self._soup(self._novel_url(url))`.
- `og(prop)` helper reading `meta[property='og:...']` (copy the closure from ixdzs.py).
- `title = og("og:title")` (strip the trailing " - Truyện Huyền Huyễn" category suffix if present — split on `" - "` and keep the head; keep it simple, only strip a trailing `- Truyện ...` segment). Raise `ScrapeError("Novel title not found …", url)` if empty.
- `description = og("og:description")`; `cover_url = og("og:image")`.
- author: og tags may not carry it; attempt a stable DOM lookup for an author link/label and fall back to `""` (author is non-critical; xbanxia already tolerates empty). **Do not** parse `__next_f`.
- Return `NovelMeta(url=self._novel_url(url), site=self.name, title=title, author=author, description=description, cover_url=cover_url, source_lang="vi")`. **Setting `source_lang="vi"` here is the single point that drives all language handling.**

`fetch_chapter_list(self, url) -> list[ChapterRef]`:
- `base = self._novel_url(url)`.
- Fetch page 1 (`base` or `base?ch=1`).
- Discover last page from the pager: find anchors/params matching `[?&]ch=(\d+)` in the page; `last = max(...)` (fall back to 1 if none). Guard against an absurd value (cap at a sane ceiling, e.g. 500 pages, and log/`ScrapeError` if exceeded — protects against selector drift).
- Iterate `n` in `1..last`: `page = self._soup(f"{base}?ch={n}")`; collect `SEL_TOC_LINKS` anchors. For each, parse chapter number from href `re.search(r"/chuong-(\d+)", href)`; keep visible text as title (e.g. "Chương 25: Hỏa Oa"). Build absolute URL via `urljoin(base, href)`.
- **Robustness:** dedupe by chapter number (pages can overlap); if a page yields zero new links, stop early (defensive against pager over-count). Sort by chapter number ascending; assign `index = 0..n-1` sequentially (index is TOC order, decoupled from the chapter number).
- Raise `ScrapeError("Chapter list not found …", url)` if the final list is empty.
- **Politeness note in code comment:** ~82 TOC pages at the configured `request_delay` (default 1.5s) is ~2 min; this is acceptable and already throttled by `HttpClient`.

`fetch_chapter(self, ref) -> str`:
- `soup = self._soup(ref.url)`.
- **Preview-gate detection FIRST** (before trusting paragraphs): if `soup.select_one(SEL_PREVIEW_GATE)` is present, OR the raw html contains `'"isPreview":true'` / `data-preview-gate="true"` / `previewLoginHref`, raise `AuthRequiredError` (see §3) with a message like: *"Chương này cần đăng nhập. Cookie phiên đăng nhập bị thiếu hoặc đã hết hạn — vào Cài đặt và dán lại cookie medoctruyen.vn."* Include `ref.url`.
- Detect a hard paywall separately: if payload indicates `"isLocked":true`, raise `AuthRequiredError` with a distinct message (*"Chương bị khoá (trả phí) — tài khoản của bạn chưa mở khoá chương này."*) so the user can distinguish a coin-gate from an expired cookie.
- `container = soup.select_one(SEL_ARTICLE)`; if `None` → `ScrapeError("Chapter content not found …", ref.url)`.
- `paragraphs = [p.get_text(strip=True) for p in container.select("p")]`; `lines = [p for p in paragraphs if p]`.
- Strip the first line if it duplicates the chapter title (mirror ixdzs/xbanxia: `if lines and lines[0] == ref.title: lines = lines[1:]`; also strip if it equals the "Chương N" portion).
- If `not lines` → `ScrapeError("Chapter content is empty", ref.url)`.
- Return `"\n\n".join(lines)`.

> Implementation caveat for the implementer: exact author/pager/gate DOM selectors must be confirmed against saved fixtures during implementation. Prefer stable DOM/OpenGraph selectors; only fall back to a narrow regex on the html string (`isPreview`/`isLocked`/`ch=`) for gate & pager discovery where DOM is unstable. **Never** parse the full `__next_f` RSC stream.

## 2. `HttpClient` cookie support — `src/noveltrans/scrapers/base.py`

Backward-compatible, cookies default off:
- Add optional param: `def __init__(self, delay_seconds=1.5, max_retries=3, timeout=20.0, cookies: str = "")`. If `cookies`: call `self.set_cookies(cookies)`.
- New method `def set_cookies(self, cookie_string: str) -> None`: if the string is non-empty, set `self._session.headers["Cookie"] = cookie_string.strip()`. (Raw-header approach is simplest and robust for a pasted browser cookie string; each worker builds a fresh client scoped to one site per run, so there is no cross-request bleed within a client.)
- **Cookie scoping (avoid leaking the medoctruyen cookie to other domains):** do **not** pass cookies unconditionally into every `HttpClient`. Instead, workers attach cookies only after confirming the matched adapter is medoctruyen (see §4). This keeps `HttpClient` generic and prevents sending a medoctruyen session cookie to xbanxia/ixdzs.

No changes to `get_html`/`get`/`_request` needed; the header rides along automatically.

## 3. Errors — `src/noveltrans/errors.py`

Add one subclass for a clear, catchable auth signal (still a `ScrapeError`, so existing `except NovelTransError` handlers in workers surface its message unchanged):

```python
class AuthRequiredError(ScrapeError):
    """Full chapter body needs an authenticated session (missing/expired cookie),
    or the chapter is paywalled for this account."""
```

Used by `fetch_chapter` (§1). Because `DownloadWorker` already catches `NovelTransError` and calls `project.mark_error(idx, str(exc))`, the actionable message lands in the chapter's `error` column and the tab status — the user is told to re-supply cookies rather than silently saving a 89-word preview.

## 4. Worker wiring — `src/noveltrans/gui/workers.py`

`ScanWorker`:
- Add param `cookies: str = ""` to `__init__`; store it.
- In `run()`, after `adapter = adapter_for_url(self.url, client)`, add: `if adapter is not None and adapter.name == "medoctruyen": client.set_cookies(self.cookies)`. (Metadata + TOC are anonymous, but attaching early is harmless and future-proofs a gated TOC.)

`DownloadWorker`:
- Change `__init__(self, project_path, delay, cookies: str = "", parent=None)`; store `self.cookies`.
- In `run()`, after `adapter = adapter_for_url(project.meta.url, client)`: `if adapter is not None and adapter.name == "medoctruyen": client.set_cookies(self.cookies)`. This is the load-bearing attach — chapter bodies are the gated resource.

`TranslateWorker` (identity-translate branch, DECISION #2):
- In `run()`, immediately before the per-chapter loop, compute `identity = project.meta.source_lang == self.target_lang`.
- If `identity`: for each `chapter in pending`, call `project.save_translation(chapter.index, chapter.title, chapter.content, self.target_lang, translator="(nguyên bản)", seconds=0.0)` and emit `chapter_done`; skip glossary building and the engine entirely. Also short-circuit the meta-translation block: call `project.save_meta_translation(project.meta.title, project.meta.description, self.target_lang)` so export front-matter and default filenames work.
- Guard: only enter identity mode when `self.target_lang == project.meta.source_lang` (i.e. user is exporting/TTS-ing in Vietnamese, the natural case). If the user picks `en` as target for a `vi` source, fall through to the normal engine path (a real vi→en translation) — the `google_free`/`cli` engines already accept arbitrary source/target.
- This branch means `get_translator(...)` should be constructed lazily/after the identity check so a vi→vi run needs no engine (avoids e.g. GoogleTranslator rejecting source==target). Reorder so identity mode returns before/without touching `get_translator`.

`AudioWorker` / exporters: **no changes** — once `translated` is populated by the identity branch, `pending_audio`, `synthesize_chapter`, and export-translated all work as-is.

## 7. Edge cases & risks (with mitigations)

- **Preview vs full**: gate-check runs before trusting paragraphs → never silently save an 89-word preview. (§1, §3)
- **Paid/locked chapters the account can't access**: distinct `isLocked` → `AuthRequiredError` message; row marked error, batch continues (existing `DownloadWorker` behavior).
- **Cookie expiry**: surfaced as `AuthRequiredError`; user re-pastes in Settings and re-runs download; errored rows (empty content) are retried.
- **TOC pagination bounds**: derive `last` from `?ch=N`, cap at a ceiling, early-stop on empty page, dedupe by chapter number → resilient to pager drift/over-count.
- **Slug-with-chapter-suffix URLs**: `_novel_url` normalizes any pasted URL (novel or `/chuong-N`) to the landing page for metadata/TOC.
- **Rate-limiting/ban over 1900+ requests**: all traffic goes through `HttpClient`'s throttle + retry/backoff; honor the user's `request_delay`; add a comment recommending a higher delay for large novels. No parallelism.
- **RSC fragility**: rely on OpenGraph meta + stable DOM (`article`, `a[href*="/chuong-"]`, `[data-preview-gate]`); use narrow string checks only for gate/pager flags; never parse `__next_f`.
- **Cookie leakage**: cookies attached only when the matched adapter is medoctruyen (§4), never sent to other sites.
- **vi→en still possible**: identity branch only triggers when target == source; picking `en` performs a real translation via existing engines.
