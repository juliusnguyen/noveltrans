# 016 — Find & Replace in the Dịch tab

Add find-and-replace to the **Dịch** (translate) tab, with two scopes:

- **Per chapter** — find/replace within a single chapter.
- **All chapters** — replace across every chapter in the project in one go.

## Context

The tab (`src/noveltrans/gui/tab_translate.py`) already has a chapter table, a
"Dịch tất cả" button, "Dịch lại từ đầu", and a per-row retranslate button delegate.
Find/replace should sit alongside these.

The motivating use case is fixing translation output in bulk — a character name or
term rendered inconsistently across a whole novel — without re-running the translator.

## Open questions for /plan

- **Which text?** Translated text only, or the original source text too? (Editing the
  original would desync it from what was scraped.)
- **Case sensitivity / whole word / regex** — how much matching power is warranted?
- **Preview before applying?** A blind replace-all across 199 chapters is
  unrecoverable if wrong. Is a match count, a preview, or an undo needed?
- **Interaction with re-translation** — a later "Dịch lại từ đầu" would overwrite
  replacements. Should that be warned about, or should replacements be re-applied?
  (Compare `translators/names.py`, which already carries a name-mapping concept.)
- **Where in the UI** — toolbar row, dialog, or a panel? Does per-chapter replace act
  on the selected row, or the chapter open in the preview pane?
