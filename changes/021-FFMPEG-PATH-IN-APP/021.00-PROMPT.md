# 021 — ffmpeg not found in the packaged .app (Tạo video button disabled)

In the packaged macOS `.app`, the "Tạo video" button is greyed out / unusable, even though
video export works fine when running from a terminal.

Cause: a Finder/Launchpad/Dock-launched app inherits a minimal PATH
(`/usr/bin:/bin:/usr/sbin:/sbin`) that excludes Homebrew, MacPorts, and `~/.local/bin`
(where the user's ffmpeg lives). `ffmpeg_available()` uses `shutil.which("ffmpeg")`, which
returns None → the button is disabled; every ffmpeg/ffprobe subprocess would also fail.

Fix: prepend the standard tool directories to `os.environ["PATH"]` once at app startup, so
`shutil.which` and all subprocesses find ffmpeg the way a terminal does.
