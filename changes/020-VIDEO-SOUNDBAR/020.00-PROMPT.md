# 020 — Nicer video: audio visualizer ("sound bar") + smooth chapter title

Enhance the video export (feature 019) so the generated video looks nicer:

1. **A "sound bar" (audio visualizer)** that moves with the voice — an animated
   waveform / frequency-bar visual driven by the audio, so the video isn't a static
   image with sound.
2. **The chapter name sits right below the sound bar** and changes as each chapter plays.
3. **The chapter name changes smoothly** (fade / cross-transition) when the chapter
   changes — not a hard cut.

(This replaces the earlier read-along-captions idea, which was dropped.)

## What already exists (feature 019, shipped)

- `src/noveltrans/tts/video.py` — `render_video(segments, image_path, out_path, font_dir,
  novel_title, *, width, height, fps, cancelled)`: builds a concat audio track, loops the
  background image with a **blurred-fill filtergraph**, burns an **ASS** subtitle track
  (persistent novel title + per-chapter chapter title on a scrim, timed by cumulative
  durations), `-shortest`, writes a YouTube description. It probes real per-chapter
  durations via ffprobe (`_with_real_durations`). Pure builders: `build_ass_subtitles`,
  `_escape_ass`, `_ass_time`. Uses `-tune stillimage` (static image → tiny video stream).
- `VideoWorker` (workers.py) + the "Xuất video" box (tab_audio.py) drive it.

## Verified environment facts

- This ffmpeg has the audio-visualizer filters: **`showfreqs`, `showcqt`, `showspectrum`
  (bar/spectrum styles), `showwaves` (waveform), `showvolume`, `avectorscope`**, plus the
  `fade` (video) filter; ASS supports `\fad(in_ms,out_ms)` for smooth text fades. libx264
  is available.

## Requirements

- Composite an audio-driven visualizer over the (blurred-fill) background image.
- Put the chapter name **just below** the visualizer; keep it timed to chapter boundaries.
- Fade the chapter name in/out on chapter change (ASS `\fad`, or a cross-fade).
- Keep it off-thread with progress/cancel and ffmpeg-gated, like 019. Additive — the
  YouTube description, chapter timing, and blurred-fill background stay.

## Open questions for /plan

- **Visualizer style:** "sound bar" most likely = **bars** (`showfreqs` or `showcqt`), vs
  a **waveform line** (`showwaves`) vs a **spectrum** (`showspectrum`). Recommend one (or
  a small choice), with a tasteful default color/size. Show a real render to pick.
- **Layout:** where the sound bar sits (centered? lower third?), the chapter name right
  below it, and what happens to the **persistent novel title** from 019 (keep it on top?).
- **Smooth transition:** ASS `\fad` on each chapter event (simplest, per-title fade) vs a
  video cross-fade. Recommend `\fad`. Fade duration (~300–500 ms?).
- **Performance / cost (important):** the visualizer makes the video **non-static**, so
  `-tune stillimage` no longer applies — the video stream is larger and the encode is
  slower/heavier than 019 (which was near-free video). Quantify and surface it; maybe
  cap the visualizer framerate, or keep batch mode as the steer.
- **The filtergraph:** how the `showfreqs`/`showwaves` output (A→V) is generated from the
  audio input and overlaid on the background, then the ASS burned on top — the exact
  `-filter_complex` (audio → visualizer → overlay on bg → subtitles). Keep the pure
  builders (ASS) testable without ffmpeg; the filtergraph is the one ffmpeg piece.
- **Color/theming:** a fixed nice default vs a small user choice. Recommend a default now.
- **CJK caveat:** same bundled-font limit as 019 — Vietnamese chapter names render,
  original-language would be tofu.
