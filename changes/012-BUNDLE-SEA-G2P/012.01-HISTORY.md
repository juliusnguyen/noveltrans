# 012 — Bundle sea_g2p.bin (frozen-app TTS fix) — History

## 2026-07-16 — Fix

**Bug:** in the packaged `.app`, TTS failed with "VieNeu-TTS lỗi khi đọc đoạn văn:
No such file or directory (os error 2)" during synthesis (model loaded fine; the error
was in `tts.infer`).

**Cause:** VieNeu's Vietnamese phonemizer `sea_g2p` loads its data file `sea_g2p.bin`
via `os.path.join(os.path.dirname(__file__), "sea_g2p.bin")` (sea_g2p/g2p.py:15). The
PyInstaller spec collected `vieneu` but not `sea_g2p`, so the `.bin` (a non-Python data
file PyInstaller won't grab on its own) was absent from the bundle → Rust ENOENT.

**Fix (`packaging/NovelTrans.spec`):** add `sea_g2p` to the `collect_all` loop so
`sea_g2p.bin` (and the Rust `.so`) ship in the app. Checked the other VieNeu deps: perth
is pure-Python; tokenizers/soxr ship only auto-collected extensions — sea_g2p.bin was the
sole missing data file.

**Verification:** dev synthesis works (model loads, 3.7s audio produced), confirming the
only frozen gap was the missing .bin. Rebuilt app; confirmed sea_g2p.bin is in the
bundle; reinstalled to /Applications.
