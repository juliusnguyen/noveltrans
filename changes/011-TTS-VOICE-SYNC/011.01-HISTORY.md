## 2026-07-16 00:57 — Implemented

Applied plan `011.02-INITIAL-PLAN.md`.

- `src/noveltrans/config.py`: `DEFAULT_TTS_VOICE` `"Ngọc Lan"` → `"Ngọc Linh"`.
- `src/noveltrans/tts/vieneu.py`:
  - Rewrote `PRESET_VOICES` to the 14 real voices (labels mirror the v3turbo
    `list_preset_voices()` output); updated the comment.
  - Added a module `logger`, a `self.voice_notice` attr, and a new `_resolve_voice()`
    called at the end of `load()`. It pins `self.voice` to a voice the loaded model
    actually offers: unknown voice → model `_default_voice` → first available → `""`.
    Logs + records a Vietnamese notice on substitution. Fully guarded so it can never
    make `load()` raise. `synthesize()` unchanged (voice now guaranteed valid/empty).
- `src/noveltrans/gui/workers.py` (`AudioWorker.run`): after `engine.load()`, adopt the
  resolved voice via `getattr(engine, "voice", self.voice)` (so filename slug,
  `pending_audio` dedup, and stored `audio_voice` reflect the voice actually spoken) and
  emit `engine.voice_notice` through the `progress` signal when set.
- `tests/test_tts.py`: updated 2 stale literals (`Ngọc Lan` → valid names), extended
  `_engine_with_mock` with `presets`/`default_voice` params, and added 5 tests
  (resolve-to-default, resolve-to-first, keep-valid, survive-list-error,
  default-constant-is-a-preset). `FakeTtsEngine` worker tests unchanged (the `getattr`
  guard preserves them).
