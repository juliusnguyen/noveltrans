# 011 — TTS Voice Sync

## Problem

Synthesis fails with:

```
VieNeu-TTS lỗi khi đọc đoạn văn: Voice 'Ngọc Lan' not found. Available: ['Minh Đức', 'Phạm Tuyên', 'Thái Sơn', 'Xuân Vĩnh', 'Thanh Bình', 'Trúc Ly', 'Ngọc Linh', 'Đoan Trang', 'Mai Anh', 'Thục Đoan', 'Minh Triết', 'Thùy Dung', 'Quang Sơn', 'Ngọc Trân']
```

The hardcoded `PRESET_VOICES` in `src/noveltrans/tts/vieneu.py` and the
`DEFAULT_TTS_VOICE = "Ngọc Lan"` in `src/noveltrans/config.py` are stale.
The installed VieNeu-TTS version no longer ships "Ngọc Lan" (and several
other preset names in the list), so when the saved/default voice isn't in
the model's actual voice set, `infer(text, voice=...)` raises.

## Goal

Make voice selection robust so a name mismatch never crashes synthesis:

- The preset/default voice must be one the model actually provides (or the
  code should adapt to whatever the installed version offers).
- If the configured voice isn't available at synth time, fall back
  gracefully (e.g. to the first available voice or the engine's own
  default) instead of raising — ideally with a warning, not a hard error.
- Keep the "shown before load" preset list accurate, or derive the list
  from the model where possible so it can't drift out of sync again.

## Notes

- Available voices per the error: Minh Đức, Phạm Tuyên, Thái Sơn, Xuân Vĩnh,
  Thanh Bình, Trúc Ly, Ngọc Linh, Đoan Trang, Mai Anh, Thục Đoan, Minh Triết,
  Thùy Dung, Quang Sơn, Ngọc Trân.
- Relevant files: `src/noveltrans/tts/vieneu.py`, `src/noveltrans/config.py`,
  and TTS-related tests (`tests/test_tts.py`).
