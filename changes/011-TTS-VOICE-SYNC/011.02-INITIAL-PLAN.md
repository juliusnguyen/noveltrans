# 011.02 — Initial Plan: TTS Voice Sync

Bugfix for VieNeu-TTS stale/hardcoded voice names causing a synthesis crash
(`Voice 'Ngọc Lan' not found`).

## Root cause

The installed vieneu build runs mode `v3turbo` (`VieneuV3Turbo`). Its
`infer(text, voice=<name>)` routes the string name through `get_preset_voice(name)`, which
raises `ValueError("Voice '{name}' not found. Available: [...]")` when the name is not one of
the 14 currently-shipped presets. The app hardcodes an old roster in two places:

- `src/noveltrans/config.py:24` — `DEFAULT_TTS_VOICE = "Ngọc Lan"` (no longer exists).
- `src/noveltrans/tts/vieneu.py:16-27` — `PRESET_VOICES` lists 6 dead names (`Ngọc Lan`,
  `Mỹ Duyên`, `Gia Bảo`, `Đức Trí`, `Trọng Hữu`, `Bình An`).

A saved QSettings `tts_voice` (or the stale default) flows unchanged into
`VieneuEngine.synthesize` → `infer(voice=...)` → crash. Passing a string voice IS the correct
API; the string just has to be a real name. Fix = keep the preset list accurate AND resolve any
unknown voice to a real one at load time (once), never crashing at synth time.

## Verified API facts (installed `vieneu`, mode `v3turbo`)

- `infer(text, voice=<str name>)` accepts a string name; unknown → the reported `ValueError`.
- `self._tts.list_preset_voices()` returns `[(label, voice_id), ...]`,
  `label = "{name} — {description}"`, `voice_id = name` — format-identical to the pre-load list.
- Model default voice = private attr `self._tts._default_voice` (currently `'Phạm Tuyên'`).
- Ground-truth roster (14): Minh Đức, Phạm Tuyên, Thái Sơn, Xuân Vĩnh, Thanh Bình, Trúc Ly,
  Ngọc Linh, Đoan Trang, Mai Anh, Thục Đoan, Minh Triết, Thùy Dung, Quang Sơn, Ngọc Trân.
  Full labels captured in the scratchpad snapshot. Survivors of the old preset list: Thái Sơn,
  Xuân Vĩnh, Trúc Ly, Ngọc Linh.

## Design decisions

1. **Resolve the voice ONCE in `load()`, not per-chunk.** `synthesize()` runs once per text
   chunk (many times per chapter), so validation belongs in `load()`. After `self._tts` is
   constructed, if `self.voice` is set but not among the model's real preset ids, fall back and
   rewrite `self.voice` in place.
2. **Fallback target order:** `getattr(self._tts, "_default_voice", None)` (currently
   `'Phạm Tuyên'`), then `available[0]`, then `""` (makes `synthesize` call `infer(text)` with no
   voice — the model's own default). Access `_default_voice` defensively (private attr).
3. **Store the resolved voice back on the engine so the worker records the truth.** `AudioWorker`
   uses `self.voice` for the on-disk filename slug (`workers.py:365`), for
   `pending_audio(self.voice, …)` (`349`), and `save_audio(…, self.voice, …)` (`382`). If the
   engine silently substitutes, the DB `audio_voice` and `pending_audio` dedup would record a
   voice never actually spoken. The worker re-reads the engine's resolved voice after `load()` so
   all three record the voice actually synthesized (and a fallback won't spuriously re-pend forever).
4. **Warn, don't crash.** Log a warning in the engine on substitution; `AudioWorker` emits a
   user-visible `progress` message (`Signal(int, int, str)` — verified) when the requested voice
   was substituted.
5. **New `DEFAULT_TTS_VOICE` — DECIDED: `"Ngọc Linh"`.** Old default was a female "dịu dàng"
   voice that no longer exists. Chosen replacement `Ngọc Linh` (Nữ · Bắc · Phong cách kể chuyện) —
   female (continuity with old UX), storytelling style (good fit for narrating novels), and it
   survived the old preset list. It is a member of the new `PRESET_VOICES`. (Note: this is distinct
   from the engine-side runtime fallback, which still uses the model's own `_default_voice`
   `Phạm Tuyên` — see decision 2.)

## Step-by-step changes

### 1. `src/noveltrans/config.py`
- Line 24: `DEFAULT_TTS_VOICE = "Ngọc Lan"` → `"Ngọc Linh"` (see decision 5).
- No change to the `tts_voice` property/setter (159-165). A stale persisted value is corrected two
  ways: (a) the combo repopulates and `findData(stale)` → `-1` → index 0
  (`tab_audio.py:283-284`), and (b) `load()` resolution neutralizes any stale value that reaches
  the engine. No migration code needed.

### 2. `src/noveltrans/tts/vieneu.py`
- **Rewrite `PRESET_VOICES` (16-27)** to the 14 real `("{name} — {description}", "{name}")` pairs
  using the roster (e.g. `("Phạm Tuyên — Nam · Bắc · Phong cách tự nhiên", "Phạm Tuyên")`,
  `("Trúc Ly — Nữ · Bắc · Phong cách tự nhiên", "Trúc Ly")`, …). Update the comment (14-15) to note
  mode `v3turbo` and that these mirror `list_preset_voices()`, reconciled by `load()`.
- **Add a module logger:** `import logging` + `logger = logging.getLogger(__name__)` (minimal,
  stdlib way to "warn, don't crash"; no logging exists in `src/noveltrans/` yet).
- **`__init__` (40-42):** add `self.voice_notice = ""` (human-readable substitution message).
- **Extend `load()` (44-52)** — after `self._tts = Vieneu()` succeeds, resolve the voice (guarded):
  - `available = [vid for _, vid in self.list_voices()]` (reuses `list_voices()`, which returns the
    loaded model's real pairs and already falls back safely on exception).
  - If `self.voice and available and self.voice not in available`:
    `fallback = getattr(self._tts, "_default_voice", None)`; if `fallback not in available` use
    `available[0]`; `logger.warning(...)`; set `self.voice_notice` + `self.voice = fallback`.
  - Wrap so a broken/absent voice list never makes `load()` raise.
- **Leave `synthesize()` unchanged (71-78):** `self.voice` is now guaranteed valid or empty, so
  `infer` cannot hit "not found". `TtsError` wrapper stays as a backstop.
- **Leave `list_voices()` unchanged (59-69):** `list_preset_voices()` already returns formatted
  pairs.

### 3. `src/noveltrans/gui/workers.py` (`AudioWorker.run`, 322-382)
- After successful `engine.load()` (330): `self.voice = getattr(engine, "voice", self.voice)`.
  The `getattr` default keeps `FakeTtsEngine`-based tests (no `.voice` attr) unchanged.
- If the resolved `self.voice` differs from the constructed value (or `engine.voice_notice` set),
  emit an informational `self.progress.emit(0, 0, <notice>)` so the substitution is visible.
- This runs **before** `pending_audio` (349), the filename slug (365), and `save_audio` (382), so
  all three record the voice actually synthesized — keeping dedup/resume logic
  (`project.py`) consistent.
- No change to `TtsVoicesWorker` (502-514): pre-load presets are now correct.

### 4. `src/noveltrans/gui/tab_audio.py`

## Sequencing
1. `config.py` constant (independent).
2. `vieneu.py` `PRESET_VOICES` + logger + `voice_notice` + `load()` resolution (core fix).
3. `workers.py` sync resolved voice (depends on 2).
4. Update/add tests (depends on 1-3), then `pytest tests/test_tts.py` (skip `@pytest.mark.live`,
   which needs the 334 MB model; run it manually if the package is installed to confirm the new
   constants resolve without fallback).

## Anticipated challenges
- `_engine_with_mock` must feed `load()` a real iterable of pairs + a `_default_voice`, or the
  resolution loop iterates a bare `MagicMock` — main test-plumbing task.
- `_default_voice` is private; the `getattr(..., None)` guard is essential so a future rename falls
  through to `available[0]`.
- Keep `load()` resolution failure-tolerant: querying the voice list must never raise out of
  `load()`, or we trade a synth-time crash for a load-time crash.
