# 004 — Discord auto-unlock for medoctruyen's 50/day cap

## Problem
When downloading from medoctruyen.vn, the site enforces a hard **50 chapters/day**
limit. On hitting it, medoctruyen returns a page with a single-use unlock code and
tells the reader to type `/mochuong <code>` in the `#mở-khoá` channel of its Discord
to unlock the next 50. Today the app stops the batch and shows those instructions;
the user does the Discord step **by hand**, then re-runs the download.

## Goal
Automate that Discord step so a batch resumes unattended.

## Chosen approach (decided with the user)
**Option A — browser automation, on a throwaway Discord account.** Considered and
rejected: a token-based self-bot (higher ban risk) and an OS keystroke-injection
approach (fragile, needs Accessibility permission). A one-click *assist* (copy code +
open channel, user presses Enter) was offered as the zero-ToS-risk fallback and is
retained as the manual fallback path.

- Playwright drives a **dedicated, persistent Chromium profile** (separate from the
  user's everyday browser) that a **throwaway** Discord account logs into once.
- On `DailyLimitError`, the app runs `/mochuong <code>` in the configured channel,
  then auto-resumes the download. Falls back to the manual dialog on any failure or
  if the profile isn't logged in.

## Risk note
Automating a Discord *user account* violates Discord's ToS regardless of method —
hence the throwaway-account guidance and low-frequency (once/day) usage. Nothing
uploads credentials; the login lives only in the local profile.
