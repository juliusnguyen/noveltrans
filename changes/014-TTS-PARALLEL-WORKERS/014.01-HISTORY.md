## [IMPL] Parallel TTS workers — 2026-07-16
Implemented per `014.01-INITIAL-PLAN.md`.

**Source changes:**
- `src/noveltrans/config.py` — added `DEFAULT_TTS_WORKERS = 1` and a `tts_workers`
  property/setter (reads with `type=int`, clamps to `max(1, …)` so a corrupt/0/negative
  stored value can never disable generation).
- `src/noveltrans/gui/settings_dialog.py` — imported `QSpinBox`; added a
  "Luồng tạo audio song song" spinbox (range 1–6) with a Vietnamese RAM/CPU-warning
  tooltip; persisted it in `accept()`.
- `src/noveltrans/gui/workers.py` — added module-level `_AudioResult` dataclass; new
  imports (`queue`, `threading`, `concurrent.futures.{FIRST_COMPLETED,ThreadPoolExecutor,wait}`,
  `dataclasses.dataclass`). `AudioWorker` gained a `workers: int = 1` param (clamped).
  `run()` now loads one "probe" engine (unchanged fail-fast + voice resolution), opens the
  project, computes `pending`, then dispatches: `workers == 1` → `_run_sequential`
  (the original loop, moved verbatim); `workers > 1` → `_run_parallel`. The parallel path
  drives a `ThreadPoolExecutor(min(N, #chapters))` where each thread gets its own engine via
  `_engine_for_thread` (`threading.local` + a seed `queue.Queue` holding the probe so its
  ~334 MB load is reused, not wasted); `_synth_one` synthesizes to a unique per-chapter file
  and returns an `_AudioResult` with **no sqlite access**. The orchestrator drains completed
  futures and performs ALL DB writes (`save_audio`/`mark_audio_error`), stale-file unlink, and
  signal emits on the connection-owning thread. Cancellation via the shared `_cancelled` flag;
  `pool.shutdown(wait=True)` joins in-flight work.
- `src/noveltrans/gui/tab_audio.py` — pass `workers=self.config.tts_workers` into
  `AudioWorker(...)` in `_start_generate`.

**Tests added (`tests/`):**
- `test_tts.py`: `TestAudioWorker.test_workers_one_loads_engine_once` (guards the sequential
  path builds only the probe); `TestAudioWorkerParallel` (generates-all, continues-on-error,
  stale-file-cleanup, engine-count-capped-by-chapters, using a per-call fresh-`FakeTtsEngine`
  factory); `TestConfigWorkers` (default 1, clamps ≤0 → 1, round-trips a valid value).
- `test_main_window.py`: `TestSettingsDialog.test_tts_workers_persists` (spinbox reflects
  default and `accept()` writes the value).

**Verification:** `ruff check src tests` clean; `pytest` full suite **256 passed, 5 deselected**
(the 5 deselected are `@pytest.mark.live` real-model tests). The parallel tests run a real
`ThreadPoolExecutor` with multiple threads, so the concurrency/orchestration logic is exercised
end-to-end (with fake engines). A real multi-engine run (RAM/throughput) is left as the manual
check in the plan §6.

**Not done (out of scope, per plan §7):** chunk-level intra-chapter parallelism; parallelizing
merge/translate/download; auto worker-count from RAM/CPU; ProcessPoolExecutor.

## [FIX] Theme: visible spinbox buttons + checkbox/radio indicators — 2026-07-16
User reported the spinbox increment/decrement buttons and the checkbox indicator were
hard to see on the dark theme. The QSS styled the spinbox *frame* but never its up/down
buttons or the check/radio indicators, so Fusion's low-contrast defaults showed through.
This affected every spinbox/checkbox in the app, not just the new setting.

- `src/noveltrans/gui/style.py`:
  - Added QSS for `QAbstractSpinBox::up-button`/`::down-button` — a distinct `SURFACE_HI`
    button column split top/bottom (canonical `top right`/`bottom right`, no `height: 50%`
    which renders unreliably), with accent hover/pressed and `padding-right` to reserve room.
  - Added QSS for `QCheckBox::indicator` / `QRadioButton::indicator` — bordered box/circle,
    accent fill when checked, accent-hi on hover, muted when disabled.
  - New `_make_icons()` renders the spinbox arrows (TEXT triangles), a white checkmark, and a
    white radio dot to PNGs in a temp dir at `apply_theme()` time (Qt QSS `url()` has no
    data-URI support, and generating at runtime avoids shipping image assets with the frozen
    app and keeps colours tied to the palette tokens). `_icon_qss()` appends the `image: url(...)`
    rules; `apply_theme` now sets `_QSS + _icon_qss(_make_icons())`.

Verified by rendering the Settings dialog + a standalone controls sample offscreen and
inspecting the glyphs. `ruff` clean; full suite still **256 passed, 5 deselected**.
