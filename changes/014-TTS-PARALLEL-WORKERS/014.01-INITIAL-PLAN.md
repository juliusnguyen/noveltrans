# Implementation Plan — 014 TTS Parallel Workers

## Overview

Add a configurable `tts_workers` setting (default 1) and teach `AudioWorker` to synthesize chapters concurrently. The `AudioWorker` QThread stays the single orchestrator: it always loads one "probe" `VieneuEngine` up front (unchanged fail-fast + voice resolution), owns the one `NovelProject` sqlite connection, and performs *all* DB writes. When `workers == 1` it runs the current sequential loop verbatim (byte-for-byte behavior). When `workers > 1` it drives a `ThreadPoolExecutor(max_workers=N)` whose pool threads each lazily build and reuse one engine (via `threading.local`, seeded with the already-loaded probe so no 334 MB load is wasted); pool threads only do CPU/IO synthesis into per-chapter unique filenames and return plain result objects, which the orchestrator drains and commits to sqlite. This keeps sqlite single-writer, preserves cancellation/progress, and only spins up `min(N, #chapters)` engines.

## 1. Config change — `src/noveltrans/config.py`

Add a module constant next to the other `DEFAULT_TTS_*` constants (after line 25):

```python
DEFAULT_TTS_WORKERS = 1  # sequential; >1 loads one ~334MB engine per worker
```

Add a property/setter pair mirroring `tts_format` / `keep_awake_enabled` style (place after the `tts_use_translation` property, ~line 182). Read with `type=int` and clamp to a floor of 1 so a corrupted/0/negative stored value can never disable generation:

```python
@property
def tts_workers(self) -> int:
    """Parallel TTS synthesis workers. Each worker loads its own ~334MB
    VieNeu engine, so N workers ≈ N×334MB RAM. 1 = sequential (default)."""
    return max(1, self._s.value("tts_workers", DEFAULT_TTS_WORKERS, type=int))

@tts_workers.setter
def tts_workers(self, value: int) -> None:
    self._s.setValue("tts_workers", max(1, int(value)))
```

### 3d. Parallel path

A small result carrier (module-level dataclass):

```python
@dataclass
class _AudioResult:
    index: int
    title: str
    status: str          # "ok" | "error" | "cancelled"
    rel_path: str = ""
    seconds: float = 0.0
    prev_audio_path: str = ""   # chapter.audio_path, for stale-file cleanup
    error: str = ""
```

Pool worker function — **no sqlite access**, only reads already-materialized `Chapter` fields and writes its own unique file:

```python
def _synth_one(self, chapter, source, slugify, project_path, audio_dir, tl, seed):
    if self._cancelled:
        return _AudioResult(chapter.index, "", "cancelled")
    engine = self._engine_for_thread(tl, seed)   # threading.local + seed queue
    if self.use_translation:
        title, text = chapter.translated_title or chapter.title, chapter.translated
    else:
        title, text = chapter.title, chapter.content
    name = f"{chapter.index + 1:04d}-{slugify(title)}-{slugify(self.voice)}.wav"
    out_path = audio_dir / name
    try:
        seconds = engine.synthesize_chapter(title, text, out_path,
                                            cancelled=lambda: self._cancelled)
        if self.out_format == "mp3":
            from noveltrans.tts.convert import convert_to_mp3
            out_path = convert_to_mp3(out_path)
        rel_path = str(out_path.relative_to(project_path))
        return _AudioResult(chapter.index, title, "ok", rel_path, seconds,
                            chapter.audio_path or "")
    except TtsError as exc:
        if self._cancelled:
            return _AudioResult(chapter.index, title, "cancelled")  # mid-chapter cancel, not an error
        return _AudioResult(chapter.index, title, "error", error=str(exc))
    except Exception as exc:
        return _AudioResult(chapter.index, title, "error", error=repr(exc))
```

Engine-per-thread with seed reuse:

```python
def _engine_for_thread(self, tl, seed):
    engine = getattr(tl, "engine", None)
    if engine is None:
        try:
            engine = seed.get_nowait()          # first thread reuses the loaded probe
        except queue.Empty:
            engine = get_tts_engine("vieneu", voice=self.voice)  # self.voice already resolved
            engine.load()                        # lazy: only when a new thread actually starts
        tl.engine = engine
    return engine
```

Because `self.voice` is already a real resolved voice, a freshly-loaded engine's `_resolve_voice()` is a no-op (no second substitution, no divergent voice, `voice_notice` ignored — already surfaced once).

Orchestrator drain loop (all DB writes + signals here, on the QThread):

```python
tl = threading.local()
seed = queue.Queue(); seed.put(probe)
done = errors = 0
total = len(pending)
pending_iter = iter(pending)
inflight = set()
pool = ThreadPoolExecutor(max_workers=min(self.workers, total))

def submit_next():
    if self._cancelled:
        return False
    try:
        ch = next(pending_iter)
    except StopIteration:
        return False
    inflight.add(pool.submit(self._synth_one, ch, source, slugify,
                             project.path, project.audio_dir, tl, seed))
    return True

try:
    for _ in range(min(self.workers, total)):
        if not submit_next():
            break
    while inflight:
        finished, inflight = wait(inflight, return_when=FIRST_COMPLETED)
        for fut in finished:
            r = fut.result()
            if r.status == "cancelled":
                continue                          # not counted, no write (matches sequential break)
            if r.status == "ok":
                if r.prev_audio_path and r.prev_audio_path != r.rel_path:
                    (project.path / r.prev_audio_path).unlink(missing_ok=True)  # stale file cleanup
                project.save_audio(r.index, r.rel_path, self.voice, r.seconds, source)
                self.chapter_done.emit(r.index)
            else:  # "error"
                errors += 1
                project.mark_audio_error(r.index, r.error)
                self.chapter_error.emit(r.index, r.error)
            done += 1
            self.progress.emit(done, total, r.title)
            submit_next()                         # backfill; no-op once cancelled/exhausted
finally:
    pool.shutdown(wait=True)                       # let in-flight chapters finish/cancel cleanly
self.progress.emit(done, total, "")
self.finished_ok.emit(done - errors, errors)
```

New imports at top of `workers.py`: `import queue`, `import threading`, `from concurrent.futures import FIRST_COMPLETED, ThreadPoolExecutor, wait`, `from dataclasses import dataclass`. `TtsError` / `slugify` / `get_tts_engine` continue to be imported lazily inside `run()` (so `patch("noveltrans.tts.get_tts_engine")` still works — the name is resolved at call time).

**GIL note**: `engine.synthesize()` → `vieneu.infer()` is ONNX Runtime native code that releases the GIL during inference, so the pool gives real wall-clock parallelism despite Python threads. This is why `ThreadPoolExecutor` (shared 334 MB model per thread, cheap result passing, trivial cancellation via the shared `self._cancelled` flag) is preferred over `ProcessPoolExecutor` (would need per-process model load + pickling and can't share the flag).

## 5. Edge cases & risks

- **N==1 stays identical**: separate `_run_sequential` method is the current loop verbatim, guarded by `if self.workers == 1`. No executor, no result objects, no threading.local — same call sequence, same signal timing, same `save_audio` argument order. Default `tts_workers == 1` and the `max(1, ...)` clamp make this the guaranteed fallback.
- **SQLite single-writer**: pool threads never touch `NovelProject`; they read only already-materialized `Chapter` dataclasses (`pending` is fully built before the pool starts) and write unique per-chapter files. Every `save_audio` / `mark_audio_error` runs on the orchestrator thread that owns the connection. Confirmed feasible: `save_audio(idx, rel_path, voice, seconds, source)` and `mark_audio_error(idx, msg)` take only plain data the result object carries — no rework of signatures needed. (Alternative — per-thread connections with `busy_timeout` — is rejected: it fights the "single writer at a time" WAL design in the module docstring and risks `database is locked` under contention for no benefit, since writes are microscopic vs. synthesis.)
- **Filename collisions**: none — `f"{index+1:04d}-{slug(title)}-{slug(voice)}.wav"` is unique per chapter index, so concurrent `save_wav`/`convert_to_mp3` writes never target the same path.
- **Stale-file "re-voiced/format changed → delete old" logic (current lines 387–389)**: preserved by carrying `chapter.audio_path` as `prev_audio_path` in the result and doing the `unlink(missing_ok=True)` on the orchestrator right before `save_audio`. Because the unlink now runs single-threaded and only for the just-committed chapter, it's actually safer than inline.
- **Cancellation races**: `self._cancelled` is a plain bool set from the GUI thread; workers only read it (mid-chapter via the `cancelled=` lambda into `synthesize_chapter`, and at task start). A task cancelled mid-chapter raises `TtsError`, is caught, and — because `self._cancelled` is set — returns `status="cancelled"` (not counted, no `mark_audio_error`), matching the sequential `break`-without-error behavior. `submit_next()` stops feeding new work once cancelled; `pool.shutdown(wait=True)` lets in-flight chapters unwind. `shutdown()`/`wait(120_000)` in tab_audio still joins the orchestrator, which now also joins the pool.
- **Partial-file cleanup on cancel**: unchanged and safe — `synthesize_chapter` writes the WAV only after concatenating all chunks, so a mid-chapter cancel raises *before* any file is written; there is no partial file to clean, exactly as today.
- **Mid-run errors**: one chapter's `TtsError`/`Exception` becomes a `status="error"` result; the orchestrator marks it and keeps draining/submitting, so the batch continues (same as the sequential `except` blocks). `finished_ok.emit(done - errors, errors)` reports the same counts.
- **Memory**: N workers ≈ N×334 MB resident model. Spinbox capped at 6 and tooltip warns. First-run model download happens per-engine but the file is cached by `vieneu`/HuggingFace, so only the first probe pays the download; extra engines load from disk.
- **Non-deterministic completion order**: acceptable per requirements; the progress bar counts completions, and `chapter_done`/`chapter_error` carry the index so `_on_chapter_updated` refreshes the right row regardless of order.

## 7. Out of scope

- Splitting a **single chapter** across workers (chunk-level parallelism). This plan parallelizes at chapter granularity only; `synthesize_chapter`'s internal chunk loop stays sequential per chapter.
- Parallelizing `MergeWorker`, `TranslateWorker`, or `DownloadWorker`.
- A dynamic/auto worker count based on detected RAM/CPU, or a per-project override — the setting is a single global int.
- `ProcessPoolExecutor` / multi-process isolation and any change to how `vieneu` loads/caches the ONNX model.
- Changing the progress UI to show per-worker status or reordering completion display.
